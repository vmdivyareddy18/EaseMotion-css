Issue #9882: Source Map Generation

Enabling CSS source maps in the production build pipeline

Before
📄 easemotion.min.css Present
❌ easemotion.min.css.map Missing
❌ Browser DevTools show only minified CSS
❌ No traceability to source files
❌ Error stacks point to line 1
After
📄 easemotion.min.css Present
🗺️ easemotion.min.css.map Present
✅ DevTools map to original source files
✅ Each rule traces to its source file
✅ Source map comment appended to minified CSS
/*# sourceMappingURL=easemotion.min.css.map */

How It Works

1
Build script uses csso with sourceMap: true
2
Generates easemotion.min.css.map alongside minified CSS
3
Source map comment appended to minified output
4
Validation scripts verify map integrity and pack inclusion

Affected Scripts

  • scripts/build-minified-css.mjs — Enable source map generation via csso
  • scripts/validate-bundle.mjs — Validate source map integrity
  • scripts/validate-pack.mjs — Include .map file in npm package
