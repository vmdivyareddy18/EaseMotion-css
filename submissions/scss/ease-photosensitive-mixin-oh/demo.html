Photosensitive-Safe Animation Mixin

SCSS mixin for seizure-safe animations with reduced motion support

Why Photosensitive-Safe Animations?

Some animations can trigger photosensitive reactions like seizures. This mixin ensures animations are safe by:

  • ⚠️ Preventing rapid flashing (max 3 flashes/second)
  • 🔒 Capping opacity changes (never fully transparent)
  • ⏱️ Setting minimum duration limits
  • ♿ Auto-supporting prefers-reduced-motion

SCSS Mixin Usage

Full SCSS Mixin 
// Photosensitive-Safe Pulse
@mixin flash-safe-pulse($duration: 2s, $min-opacity: 0.4, $max-opacity: 1) {
  $safe-min: if($min-opacity < 0.3, 0.3, $min-opacity);
  
  animation: pulse-safe #{$duration} ease-in-out infinite;
  
  @keyframes pulse-safe {
    0%, 100% { opacity: $safe-min; }
    50% { opacity: $max-opacity; }
  }
  
  @media (prefers-reduced-motion: reduce) {
    animation: none;
    opacity: $safe-min;
  }
}

// Usage
.safe-element {
  @include flash-safe-pulse(2s, 0.4, 1);
}

Safety Guidelines

  • Never animate from full opacity (0) to full opacity (1) rapidly
  • Limit animations to maximum 3 flashes per second
  • Always provide prefers-reduced-motion fallbacks
  • Test animations with users who have photosensitive conditions
  • Consider offering a manual toggle for animation preferences
