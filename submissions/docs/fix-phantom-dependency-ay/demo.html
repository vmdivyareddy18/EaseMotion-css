Phantom Runtime Dependency — Fix Reference

package.json contains an accidental "dependencies" block that installs an npm package named dependencies at runtime, breaking the zero-dependency promise of EaseMotion CSS.

🔴 CRITICAL — Remove before next release
✘ Current package.json (buggy) 
{
  "name": "easemotion-css",
  "description": "Zero dependencies.",
  ...
  "devDependencies": {
    "vitest": "^4.1.8",
    ...
  },
  "dependencies": {
    "dependencies": "^0.0.1"
  }
}
✔ Corrected package.json (after fix) 
{
  "name": "easemotion-css",
  "description": "Zero dependencies.",
  ...
  "devDependencies": {
    "vitest": "^4.1.8",
    ...
  }
  // "dependencies" key removed entirely
}

Impact of the Bug

  • Installs an unnecessary npm package named dependencies@^0.0.1 into every user's node_modules at runtime.
  • Directly contradicts the "Zero dependencies" claim in the package description and README.
  • May trigger npm audit warnings, blocking CI pipelines.
  • Inflates the published npm bundle and install footprint unnecessarily.
  • Erodes trust — users expect a CSS-only framework to have no runtime JS dependencies.
Proposed fix for the maintainer: Remove the entire "dependencies": { "dependencies": "^0.0.1" } block from package.json. No other keys need to change. After removal, run npm install to regenerate package-lock.json, then verify with npm run validate:manifest && npm run validate:bundle to confirm zero runtime dependencies remain.
