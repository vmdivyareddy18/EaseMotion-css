.ease-bounce Default Translation Fix

The ease-kf-bounce keyframe uses a hardcoded -20px translation. On a 16×16 icon that is 125% of its own height — flying completely out of its container. Replacing it with -0.5em makes the bounce distance proportional to the element's font-size.

Issue #39220
✘ Current — hardcoded -20px
16px icon — bounces 20px (125% height) 🚨
Button — bounces 20px (fixed)
Aa 32px heading — bounces 20px
✔ Fixed — -0.5em (scales with element)
16px icon — bounces ~8px (50% height) ✅
Button — bounces ~7px (proportionate)
Aa 32px heading — bounces ~16px (feels right)

Exact diff — two files, one line each

--- a/core/animations.css  (line 189)
-    transform: translate3d(0, -20px, 0);
+    transform: translate3d(0, -0.5em, 0);

--- a/easemotion/bounce.css  (line 11)
-    transform: translateY(-20px);
+    transform: translateY(-0.5em);
For the maintainer: Apply the two-line diff above. No class names change. No variables are removed. Existing users who override the transform in their own CSS are unaffected. Optionally, expose --ease-bounce-distance: -0.5em as a custom property on :root so power users can tune it without touching keyframes — the full annotated reference is in style.css.
