Delay Class Gap Finder

Need a 250ms stagger? Official utilities jump from ease-delay-200 to ease-delay-300. This lab maps the ladder, highlights the gaps, and shows safe --custom-delay / inline patterns — without editing core.

Official ease-delay-* ladder

Workaround patterns (no core edits)

Use a utility when it matches. For mid-values, set a local CSS variable or an inline delay. Both keep your project self-contained.

1. Utility class (preferred when it fits)

<div class="ease-fade-in ease-delay-200">…</div>
<div class="ease-fade-in ease-delay-300">…</div>

Use when 200ms or 300ms is close enough. Zero custom CSS.

2. Inline --custom-delay

<div
  class="ease-fade-in custom-delay"
  style="--custom-delay: 250ms"
>…</div>
/* in your project stylesheet — not core */
.custom-delay {
  animation-delay: var(--custom-delay, 0ms);
}

Best for one-off mid-values like 250ms, 350ms, or 900ms.

3. Inline style delay

<div
  class="ease-fade-in"
  style="animation-delay: 250ms"
>…</div>

Fastest for demos. Prefer the variable pattern in real apps so you can theme delays from one place.

4. When to use which

  • Official class — value exists on the ladder
  • --custom-delay — mid-gap values, reusable
  • Inline style — quick prototypes only
  • Do not — add ease-delay-250 to core for a single page
