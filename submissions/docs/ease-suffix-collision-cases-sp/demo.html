Contribution Suffix Collision When Two People Use Same Initials

Focus: collision cases & policy — not a suffix generator. Short initials like ak look fine until two people both invent ease-card-ak. Learn why clashes happen and which longer / dated / username suffixes stay unique.

Policy takeaway: A suffix must be unique in submissions/, not merely “your initials.” Prefer GitHub username, a date stamp, or a longer personal token over 2-letter initials alone. See also CONTRIBUTING.md examples: ease-hover-sap, ease-tabs-ak, ease-card-pr.

Case study: ease-card-ak clash

Two contributors — Ananya Kapoor and Amir Khan — both choose initials ak and both want a card demo. Same feature slug + same short suffix = identical folder path.

Collision

Same initials → same folder

Git cannot store two directories with one name. The second PR conflicts or overwrites intent.

submissions/examples/
├── ease-card-ak/   ← Ananya (PR #1)
└── ease-card-ak/   ← Amir (PR #2) CLASH
Safe

Longer / unique suffixes

Same idea, different unique tokens — both PRs can merge cleanly.

submissions/examples/
├── ease-card-ananyak/
├── ease-card-amirkhan/
└── ease-card-ak-20260315/

Why short initials are risky at scale

  • Common initials cluster. Many people share ak, as, js, sk, ap — especially in large programs like GSSoC.
  • Feature names also collide. Popular ideas reuse the same slug (card, navbar, modal). Suffix is often the only differentiator.
  • CONTRIBUTING examples use short forms as illustration (ease-tabs-ak is an example format — not a guarantee that ak is free in the tree.
  • Reviewers lose signal. Duplicate paths force renames mid-review and slow merges.
This page documents collision policy. For interactive name generation, see submissions/docs/ease-suffix-generator-sp/ — a different tool.

Recommended suffix policies

1. GitHub username

Best default. Already unique on GitHub and easy for reviewers to map to a person.

ease-card-suman20041

2. Longer personal token

First name + last initial, or 4–8 letter handle — much rarer than 2 letters.

ease-card-ananyak

3. Initials + date

Keep short initials only if you append a date or issue number for uniqueness.

ease-card-ak-20260315

Safe name examples by track

Copy a pattern, then swap in your username / date / feature.

Before you open a PR

  • Search GitHub for your exact folder name under submissions/.
  • Avoid lone 2-letter initials when the feature slug is a common word.
  • Prefer ease-{feature}-{github-username} for uniqueness.
  • If you keep initials, append -YYYYMMDD or an issue number.
  • Match the track folder: docs/, examples/, react/, or scss/.
  • Do not rename or delete someone else’s existing submission folder.
