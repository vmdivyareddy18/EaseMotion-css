CSS Easing Curve Explorer

An interactive guide to how CSS timing functions shape motion — see, compare, and understand easing before you use it.

An easing function describes how a value changes over the duration of an animation. Instead of moving at a constant speed, most real-world motion accelerates and decelerates — easing curves let you describe that acceleration in CSS with transition-timing-function or animation-timing-function.

Built-in Easing Functions

The five standard keyword timing functions defined by CSS.

linear

Constant speed from start to finish. No acceleration.

ease

Default in CSS. Starts fast, slows toward the end.

ease-in

Starts slow, accelerates toward the end.

ease-out

Starts fast, decelerates smoothly to a stop.

ease-in-out

Slow start, fast middle, slow finish. Symmetrical.

Anticipation Bounce

Pulls back before launching forward, then overshoots.

cubic-bezier(.68,-0.6,.32,1.6)

Elastic Pop

Overshoots the target before settling — great for pop-ins.

cubic-bezier(.34,1.56,.64,1)

When Should You Use It?

Practical guidance for choosing the right curve.

Linear

  • Continuous loaders and spinners
  • Progress bars
  • Marquees and infinite scrolling

Ease In

  • Elements exiting the viewport
  • Building momentum before a transition

Ease Out

  • Entrances and appearing elements
  • Modals, tooltips, dropdowns

Ease In Out

  • General-purpose UI transitions
  • Moving elements between two states

Motion Accessibility

Not everyone wants motion — and CSS lets you respect that.

Vestibular disorders and motion sensitivity can make animated interfaces uncomfortable or disorienting. The prefers-reduced-motion media query lets you detect a user's OS-level preference and scale back or remove non-essential motion accordingly.
