    Common mistakes

    • Claiming more than 2 issues without finishing current work
    • Going inactive for 24+ hours without opening a PR
    • Commenting "assign me" instead of exact /claim
    • Opening PRs that edit core/ during contribution freeze
    • Missing labels or wrong submission track folder
    GSSoC contribution etiquette for this repo
    • Read CONTRIBUTING.md before your first claim.
    • One focused feature per pull request.
    • Do not ping maintainers — reviews happen on a rolling basis.
    • Use /unclaim if you cannot finish — frees the issue for others.
    • Only the maintainer merges PRs; never self-merge.
