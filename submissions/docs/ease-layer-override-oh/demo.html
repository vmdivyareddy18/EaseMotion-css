Understanding EaseMotion Cascade Layers

EaseMotion CSS uses @layer to organize styles from lowest to highest priority:

@layer easemotion-utilities (lowest)
@layer easemotion-components ⬅️ Override here!
@layer easemotion-base (highest)

Styles inside @layer easemotion-components will override the framework without !important!

Best Practices

  • Always override inside @layer easemotion-components
  • Avoid !important — use layers instead
  • Keep overrides organized in a separate CSS file
  • Use CSS custom properties for easy theming
  • Test that overrides work after EaseMotion updates
  • Group related overrides together
