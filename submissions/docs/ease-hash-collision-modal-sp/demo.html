Hash Modal Open While URL Already Has Unrelated #id

Focus: collision between an in-page section hash (#features) and a modal hash (#hcm-modal-x) — including the stray hash clear when modal.js sets location.hash = '' on close. Not a general deep-link playground.

Short answer: The URL can only hold one hash. Opening a modal overwrites #section. Closing via Escape / backdrop clears the hash entirely — it does not restore the previous section id. Remember the prior hash (or avoid hash for modals) to keep deep-links intact.

Why the collision happens

  1. One hash slot. location.hash is a single string. #features and #hcm-modal-x cannot coexist.
  2. Modal open overwrites. Navigating to #hcm-modal-x replaces the section id. CSS :target and modal.js both key off that one value.
  3. Close clears to empty. From core/modal.js: Escape and backdrop set window.location.hash = ''. That closes the overlay but also drops the previous unrelated #id (stray clear).
  4. Scroll / :target side effects. Jumping hashes can scroll the page; losing #features means refresh/share no longer lands on that section. 
// core/modal.js (close paths)
if (e.target === overlay) {
  window.location.hash = ''; // Clear hash = close modal
}
if (e.key === 'Escape') {
  window.location.hash = ''; // triggers hashchange → close
}

Safer — remember previous hash, restore on close
Copied 
let previousHash = '';

function openModalRememberingHash(modalHash) {
  previousHash = window.location.hash; // e.g. "#features"
  window.location.hash = modalHash;    // e.g. "#hcm-modal-x"
}

function closeModalRestoringHash() {
  window.location.hash = previousHash || '';
  previousHash = '';
}
Safer — keep section hash; toggle modal with is-active only
Copied 
// Do not change location.hash for the modal at all
const overlay = document.querySelector('#hcm-modal-x');
openBtn.addEventListener('click', () => {
  overlay.classList.add('is-active');
  document.body.style.overflow = 'hidden';
});
closeBtn.addEventListener('click', () => {
  overlay.classList.remove('is-active');
  document.body.style.overflow = '';
});
