The Browser Rendering Pipeline

Every frame the browser renders goes through up to three distinct phases. To animate smoothly, we want to bypass as many phases as possible.

1. Layout (Reflow)

Calculates shapes and geometry. Changing dimensions or placement forces layout updates for the entire page.

width, height, top, left, margin
→

2. Paint

Fills in pixels (colors, backgrounds, shadows). Painting is slow and happens on CPU threads.

color, background, box-shadow
→

3. Composite

Draws pre-painted layers onto the screen. Offloaded to the GPU hardware for instant rendering.

transform, opacity

Composite Layers

Animating compositor-only properties allows the browser to create a separate layer for that element, avoiding main thread calculations.

  • GPU Accelerated
  • Keeps main thread responsive
  • Zero layout recalculations

Will-Change Property

Use `will-change` to hint to the browser that an element will be animated, enabling optimization in advance.

  • Only use on active animators
  • Don't overuse (wastes GPU memory)
  • Apply directly in CSS declaration

Reduced Motion

Always respect user system settings for motion control to ensure maximum accessibility for vestibular-sensitive users.

  • Disable fast animations
  • Use simple opacity transitions
  • Check prefers-reduced-motion
