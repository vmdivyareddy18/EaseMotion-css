Ping/Pulse Scale Overflow Creating Horizontal Scrollbar

Edge-of-viewport .ease-ping scales to scale(2). That paint box can escape the layout width and force unwanted overflow-x — even when your content is not wider than the screen.

Short answer: Clip the ping with a small containment wrapper (overflow: hidden) or hide overflow on the edge section. Do not remove the animation — contain its paint so it cannot widen scrollWidth.

What .ease-ping does

From core/animations.css: the ping keyframe scales the element up while fading out. Transforms expand the visual box; without a clipping ancestor, that expansion can increase scrollable overflow. 

@keyframes ease-kf-ping {
  75%, 100% {
    transform: scale(2);
    opacity: 0;
  }
}

.ease-ping {
  animation: ease-kf-ping 1s … infinite;
}

Why scale creates horizontal scroll

  1. Transforms expand paint, not layout flow — but overflow still counts. scale(2) does not change the element’s layout size in the flex/grid sense, yet browsers still include the transformed ink in overflow calculations for ancestors with visible overflow.
  2. Edge placement removes the safety margin. Centered pings usually bloom inside the content box. Right/left-edge badges bloom past the viewport.
  3. Body / main rarely clips by default. Without overflow-x: hidden (or a local clip wrapper), the document grows a horizontal scrollbar.
  4. Pulse can contribute too. Soft scale pulses near edges are milder than ping’s scale(2), but stacked effects + shadows can still tip overflow.

Pattern comparison

Pattern Result Verdict
Edge .ease-ping on a visible-overflow ancestor scrollWidth > clientWidth → horizontal scrollbar Bug
Local wrapper overflow: hidden around the ping stack Ping animates; overflow stays local Fix
Section / card overflow-x: hidden Clips all edge effects in that region Fix
Global body { overflow-x: hidden } only Hides symptom site-wide; can mask real layout bugs Last resort
Move indicator inward (≥ half of max scale radius) Bloom stays inside the content box Fix

Containment recipes

1. Ping clip wrapper

Best default. Size the wrapper to the max bloom (~2× the dot) and set overflow: hidden.

2. Card / section clip

Use when many badges share an edge. Prefer overflow-x: hidden on the section, not the whole page.

3. Inset the indicator

Add padding/margin so scale(2) never reaches the viewport edge. Combine with a clip wrapper for safety.

Debug checklist

  • Measure document.documentElement.scrollWidth vs clientWidth while ping is running.
  • Temporarily remove ease-ping / ease-pulse — if the scrollbar vanishes, motion overflow is the cause.
  • Prefer a local clip wrapper over global body overflow hacks.
  • Keep decorative pings aria-hidden="true"; they are visual only.
  • Respect prefers-reduced-motion for infinite pings.
