ease-btn-hover vs ease-hover-grow

Both names sound like “make this react on hover.” They do different things. Hover each button below, then use the decision table to pick the right class — API literacy, not another hover effect dump.

Issue #46181
Quick truth: ease-btn-hover is a button modifier (lift + shadow/glow). ease-hover-grow is a generic utility (scale up). Same intent category — different API surface.

Live lab: same button, each class

Identical base markup (ease-btn ease-btn-primary ease-btn-pill). Only the hover helper changes. Hover (or tab-focus if your pointer is coarse) to feel the difference.

ease-btn-hover

Button modifier

On hover: lifts (~3px) and grows shadow / glow. Keeps button footprint size.

ease-hover-grow

Generic utility

On hover: scales to ~1.06. Works on buttons, cards, icons — not button-only.

API overlap explained

ease-btn-hover components/buttons.css

  • Designed to pair with ease-btn
  • transform: translate3d(0, -3px, 0)
  • Adds elevated box-shadow + button glow token
  • Guarded by hover: hover + fine pointer media
  • Reads as “this is a clickable CTA lift”

ease-hover-grow core/animations.css

  • Generic hover utility (any element)
  • transform: scale(1.06)
  • No shadow — size change only
  • Great for chips, cards, icons, avatar tiles
  • Reads as “this thing gets slightly bigger”

Decision table — which one when

Use this when two names feel interchangeable. Prefer the class whose effect matches the UX you want — not which name sounds closer to “hover button.”

You want… On… Pick Why
CTA lifts with depth / glow Primary / outline buttons ease-btn-hover Button-specific lift + shadow API
Subtle enlarge on hover Button, card, icon, badge ease-hover-grow Generic scale utility
Card feels tappable without looking “pressed up” Cards / media tiles ease-hover-grow Scale reads as focus/attention
Marketing hero CTA with elevation ease-btn only ease-btn-hover Matches button language in docs/README
Icon button that shouldn’t cast a big glow Small controls ease-hover-grow Scale is quieter than lift+shadow
Both lift and scale Button (rare) Stack carefully Both set transform — last/higher specificity wins

Stacking both classes

Usually avoid putting ease-btn-hover and ease-hover-grow on the same element. Both animate transform. You don’t get lift and scale — one declaration fights the other. Prefer one clear motion job per control.

Copy-paste usage

Button lift ease-btn-hover 
<button class="ease-btn ease-btn-primary ease-btn-pill ease-btn-hover">
  Get Started →
</button>
Generic scale ease-hover-grow 
<button class="ease-btn ease-btn-primary ease-btn-pill ease-hover-grow">
  Get Started →
</button>

<!-- Also fine on non-buttons -->
<div class="ease-card ease-hover-grow">…</div>
