Copy-ready override snippets

Button theme

@layer easemotion-components {
  .ease-btn-primary {
    background-color: #0d9488;
    border-color: #0d9488;
  }
  .ease-btn-primary:hover {
    background-color: #0f766e;
    border-color: #0f766e;
  }
}

Card border

@layer easemotion-components {
  .ease-card {
    border: 2px solid #8b5cf6;
    box-shadow: 0 8px 24px rgba(139, 92, 246, 0.15);
  }
}

Tab underline

@layer easemotion-components {
  .ease-tabs-nav {
    border-bottom-color: #e2e8f0;
  }
  .ease-tab-input:nth-of-type(1):checked
    ~ .ease-tabs-nav .ease-tab-label:nth-of-type(1) {
    color: #6c63ff;
    box-shadow: inset 0 -3px 0 #6c63ff;
  }
}
Why not use !important?
  • Layer overrides keep specificity flat and maintainable.
  • !important fights the framework and breaks composability.
  • Put theme rules in @layer easemotion-components or a layer declared after EaseMotion layers.
  • Load your override stylesheet after easemotion.css.
