Bug #45826: Without the spacing fix, switching to Light Theme repaints the navbar but does not push the hero section down — causing the hero content to sit behind or overlap the fixed navbar.

Hero Section Spacing Fix

The hero always starts below the fixed navbar — in both Dark and Light themes — because the offset is driven by the --navbar-height CSS variable, not a hardcoded pixel value.

See the fix ↓

Current theme: Dark

How the Fix Works

❌ The Bug

The hero section used a static margin-top or padding-top that matched the navbar height only in one theme. When the theme switched, the navbar background and layout could change, but the hero offset remained hardcoded — causing overlap. 

/* ❌ Broken: hardcoded offset */
.hero {
  padding-top: 64px; /* only works in dark theme */
}

✅ The Fix

The navbar height is defined once as a CSS variable --navbar-height on :root. Both <main> (or <body>) and .hero reference this single source of truth. Changing the theme never breaks the spacing. 

/* ✅ Fixed: CSS variable drives spacing */
:root {
  --navbar-height: 64px;
}

body {
  padding-top: var(--navbar-height);
}

.hero {
  /* body padding already creates the gap;
     hero padding-top is an optional extra guard */
  padding-top: var(--navbar-height);
}

📱 Responsive

The variable can be updated at each breakpoint inside a single @media rule. Every element that consumes --navbar-height updates automatically — no duplicated breakpoint overrides needed. 

/* Responsive update — one place only */
@media (max-width: 768px) {
  :root {
    --navbar-height: 56px;
  }
}

Key Takeaways

🎯

Single Source of Truth

Defining the navbar height once in a CSS variable eliminates the class of bugs where spacing is maintained in multiple places and they drift out of sync.

🎨

Theme-Agnostic

The spacing rule is not tied to any theme class or selector. Dark, Light, or any custom theme can change colours and fonts without touching the layout offset.

♿

Accessible

Content is never obscured by the navbar. Skip-to-main-content links, focus management, and :focus-visible outlines all work correctly because the layout offset is consistent.
