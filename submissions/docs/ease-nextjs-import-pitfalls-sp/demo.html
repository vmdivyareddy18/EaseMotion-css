Next.js App Router Import Placement Pitfalls

Compare wrong vs correct EaseMotion CSS import placement in App Router projects. Learn why global styles belong in layout.js, how FOUC happens, and why Server Components work without hydration issues.

Golden rule: Import easemotion-css/easemotion.min.css once in app/layout.js. Use className utilities in page.js and Server Components — do not re-import the stylesheet per page.

FOUC: Flash of Unstyled Content

When CSS loads per-page instead of globally in the root layout, users briefly see unstyled HTML before animations and tokens apply. Click the button to simulate the flash.

Welcome to EaseMotion

Styles should load before first paint — not after navigation.

How to prevent FOUC

  • Import EaseMotion once in app/layout.js, not in individual pages.
  • Keep the import at the top of the layout file, before the component export.
  • Avoid dynamic import() for global CSS unless you understand the tradeoff.
  • Do not duplicate the import in both layout and page files.
  • Use the minified bundle (easemotion.min.css) for faster first paint.

Server Component notes

From the EaseMotion README — CSS-only integration with no runtime JavaScript.

✓ Works in Server Components

EaseMotion is plain CSS. Use className="ease-fade-in" directly in Server Components — no 'use client' wrapper needed for styling.

✓ No hydration issues

Unlike JS animation libraries, EaseMotion adds no client-side runtime for core classes. Styles apply through the global stylesheet imported in layout.

⚠ Optional engine is separate

The em="" Motion Engine is opt-in JS. Only import easemotion-css/engine in Client Components when you need the DSL.

Recommended production snippets

Copy these patterns directly into a new Next.js App Router project.

app/layout.js

Import here 
// app/layout.js — import once globally
import 'easemotion-css/easemotion.min.css';

export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  );
}

app/page.js

Classes only 
// app/page.js — use class names, no import
export default function Page() {
  return (
    <main className="ease-center ease-fade-in">
      <h1 className="ease-slide-up">Welcome</h1>
      <button className="ease-btn ease-btn-primary ease-hover-grow">
        Get Started
      </button>
    </main>
  );
}

Pre-ship pitfall checklist

  • Global import: EaseMotion CSS is imported exactly once in app/layout.js.
  • No page imports: Individual page.js files do not re-import the stylesheet.
  • No duplicates: Layout and pages do not both import the same CSS file.
  • Import order: The CSS import appears before the layout component export.
  • Server Components: Pages use className only — no client wrapper for basic utilities.
  • Motion engine: If using em="", import the engine only in Client Components.
  • FOUC test: Hard-refresh and navigate between routes — styles should never flash unstyled.
