Bug Showcase: Case-Sensitivity Validation Mismatch

Core Tooling Bug Fix

❌ The Issue

The manifest validation script (validate-package.mjs) fails locally and in CI build pipelines due to a case-sensitive check on the repository URL. It looks for "SAPTARSHI-coder/EaseMotion-css" with exact casing, but standard packages and git checkouts may list it as lowercase: "saptarshi-coder/easemotion-css".

🔗 Affected File

File path: scripts/validate-package.mjs

🛠️ Proposed Fix

Normalize both strings to lowercase before checking. Here is the suggested diff:

- if (!manifest.repository?.url?.includes("SAPTARSHI-coder/EaseMotion-css")) {
+ if (!manifest.repository?.url?.toLowerCase().includes("saptarshi-coder/easemotion-css")) {
    fail("repository.url must point to SAPTARSHI-coder/EaseMotion-css");
  }
