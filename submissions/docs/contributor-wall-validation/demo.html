Contributor Wall Builder Validation

Demonstration of HTML escaping to prevent XSS vulnerability and input validation to detect missing contributor data.

🔒

HTML Escaping & XSS Protection

Usernames (logins) loaded from API are now fully HTML-escaped. This prevents malicious scripts or elements injected through GitHub profiles from breaking the contributor wall layout or executing arbitrary code.

Before (Vulnerable) alt="{login}"
After (Secured) alt="{html.escape(login)}"
⚠️

Contributor Count Validation

Added verification check to compare parsed contributor count against expected headcount. If the data is truncated or missing, it alerts the maintainers on standard error rather than silently failing.

Before (Silent Failure) contributors = contributors[:551]
After (Warns & Validates) if len(contributors) < EXPECTED_COUNT:
    print(warning, file=sys.stderr)
