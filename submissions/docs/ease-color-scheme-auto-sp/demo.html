color-scheme: light dark Auto Token Behavior

EaseMotion sets color-scheme: light dark on :root. That is not the same as a full dark-token theme switcher. Toggle modes below and compare browser auto UI vs a full remapped surface.

Issue #46189
Different from a dark-mode switcher: color-scheme mainly tells the browser how to paint native controls (inputs, scrollbars, form widgets). A full theme remap changes EaseMotion tokens like --ease-color-bg. This lab separates those ideas.

Framework already supports limited remaps via prefers-color-scheme and [data-theme] — still not every component token is a complete design-system dark theme.

What changes / what doesn’t

  • ✓ Changes with color-scheme: many native controls, scrollbars, form widgets, selection in some UAs.
  • ~ Partial with preferences: EaseMotion remaps some tokens under prefers-color-scheme: dark / [data-theme="dark"] (bg, surface, text, shadows, glass) — not every accent/component recipe.
  • ✕ Does not mean: “all UI auto becomes a curated dark theme” from color-scheme: light dark alone.

Why this distinction matters

  • Browser auto behavior UA

    color-scheme is a hint to the user agent. It’s cheap and important for forms/scrollbars, but it isn’t a design-token engine.

  • Token remap is intentional CSS Tokens

    Flipping --ease-color-bg / surface / text is a theme layer. That’s what [data-theme="dark"] (and some preference media) provide — still not a full component-by-component dark system.

  • Avoid false bug reports Docs

    If OS is dark but a card still looks light, check whether you only have auto color-scheme vs an actual token remap.

Quick reference

Approach What it does Use when
color-scheme: light dark UA may theme native chrome automatically Default / baseline (EaseMotion :root)
color-scheme: dark only Native dark widgets; tokens may stay light Teaching / isolating UA behavior
[data-theme="dark"] Remaps key EaseMotion surface tokens Explicit app theme toggle
Full design-system dark Every component/recipe curated for dark Future / product-level theming work

Copy-paste guidance

Baseline What EaseMotion already ships 
:root {
  color-scheme: light dark; /* UA-aware native controls */
}

/* Explicit theme when you need token remap (framework supports this) */
[data-theme="dark"] {
  color-scheme: dark;
  /* --ease-color-bg / surface / text remapped by EaseMotion */
}

/* Remember: color-scheme alone ≠ complete dark design system */
