Fix: Missing Vendor Prefixes

Issue #53769 — The build pipeline has no PostCSS/Autoprefixer step, so vendor prefixes like -webkit-appearance and -webkit-user-select are never emitted into easemotion.min.css.

1. appearance: none on .ease-select

Without -webkit-appearance: none, iOS Safari still renders the native dropdown arrow, overriding the custom SVG chevron.

.ease-select { appearance: none; } .ease-select { -webkit-appearance: none; /* Safari / iOS WebKit */ appearance: none; }
✕ Before — Safari shows native arrow ✓ After — custom chevron visible everywhere

2. user-select: none on .ease-toggle

Without -webkit-user-select: none, older WebKit builds allow text selection on the toggle label during rapid clicks, which looks broken on mobile Safari.

.ease-toggle { user-select: none; } .ease-toggle { -webkit-user-select: none; /* Safari / iOS WebKit */ user-select: none; }
✕ Before — label selectable on old Safari ✓ After — no accidental text selection

3. Long-term: Autoprefixer in the build pipeline

Manually adding prefixes doesn't scale. The recommended fix is to integrate PostCSS + autoprefixer into scripts/build-minified-css.mjs with a .browserslistrc config so all future properties are prefixed automatically.

// scripts/build-minified-css.mjs (proposed addition) import postcss from 'postcss'; import autoprefixer from 'autoprefixer'; // After bundling, before minifying: const prefixed = await postcss([autoprefixer]).process(bundledCss, { from: undefined, }); const minifiedCss = minifyCss(prefixed.css);
// .browserslistrc (new file at repo root) > 0.5% last 2 versions not dead iOS >= 12 Safari >= 12
✓ Zero-maintenance vendor prefixes for all future CSS
