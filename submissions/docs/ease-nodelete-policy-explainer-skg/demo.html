Why PRs Get Auto-Closed

The bot checks every PR automatically. Any of these patterns causes an immediate close — no feedback, no second chance.

🚫

Edited a core file

Any change inside core/, components/, workflows, or root config files is blocked.

📂

Wrong directory

Files added directly under submissions/ root (not inside one of the 4 allowed subdirs) are rejected.

📄

Missing required files

Standard track needs all three: demo.html, style.css, README.md.

📏

Under line-count threshold

Total lines across all submitted files must reach ≥ 250. Thin stubs get closed immediately.

Policy Rules at a Glance

These are the exact checks the bot runs on every PR.

Rule Threshold What happens if violated
No core edits 0 files outside submissions/ Auto-closed immediately
Correct subdirectory Must be inside examples/, docs/, react/, or scss/ Auto-closed immediately
3-file requirement demo.html + style.css + README.md (Standard/Docs track) Auto-closed immediately
Line-count ≥ 250 Total added lines across all files in the PR Auto-closed immediately
Unique folder name Append your initials or short ID Merge conflict / may be closed
No self-merge Only the maintainer merges Immediately reverted
📏 Line-count ≥ 250 explained

The bot sums all added lines (+ lines in the diff) across every file in your PR. Comments, blank lines, and HTML boilerplate count. A typical passing submission looks like: 85 lines of HTML + 120 lines of CSS + 60 lines of README = 265 total.

📄 3-file rule explained

Standard track (submissions/examples/) and Docs track (submissions/docs/) both require exactly these three files. React track replaces demo.html with a .jsx file. SCSS track only needs _mixin.scss + README.md.

Standard Track

submissions/examples/your-feature-initials/
├── demo.html    ← self-contained, opens directly in browser
├── style.css    ← all keyframes + utility classes
└── README.md    ← what / how / why (≥ 3 sections)

Target issues: General animations, UI components, hover effects.

Docs Track

submissions/docs/your-guide-initials/
├── demo.html    ← documentation page or showcase
├── style.css    ← styles for the doc page
└── README.md    ← what / how / why

Target issues: Policy guides, engine documentation, integration walkthroughs.

React Track

submissions/react/your-component-initials/
├── YourComponent.jsx   ← uses EaseMotion CSS classes
├── README.md           ← description + props table + usage
└── style.css           ← (optional) scoped styles

Target issues: Labeled react. Component must use EaseMotion utility classes.

SCSS Track

submissions/scss/your-mixin-initials/
├── _your-mixin.scss    ← mixin / variables / functions
└── README.md           ← params table + @include example

Target issues: Labeled scss. Must build on EaseMotion token variables or keyframes.
