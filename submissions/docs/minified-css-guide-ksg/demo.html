Using the Minified Build in Production

EaseMotion CSS ships two files. This guide explains when and how to use easemotion.min.css — the single-file production bundle — instead of the dev entry point.

File Size Comparison

Development build
easemotion.css
~120 KB
14 separate source files — each generates an HTTP request. Comments and whitespace preserved for readability.
Production build (minified)
easemotion.min.css
~71 KB   (~18 KB gzip)
All 14 files inlined into one. Comments stripped, whitespace collapsed. ~41% smaller, 1 HTTP request.

What npm run build Does

  • 1
    Reads the entry point

    Starts from easemotion.css, which contains only @import declarations pointing at core/, components/, and easemotion/ sub-files.

  • 2
    Inlines all local imports

    Each local @import is recursively resolved and its content is spliced in-place. External @import URLs (e.g. Google Fonts) are preserved at the top of the output.

  • 3
    Strips comments

    All /* … */ block comments are removed. String literals inside rules are left untouched.

  • 4
    Collapses whitespace

    Multiple spaces and newlines are reduced to single spaces. Whitespace around { } : ; , > is removed. Trailing semicolons before } are dropped.

  • 5
    Writes easemotion.min.css

    Outputs the single-file minified bundle to the repo root, ready to ship via npm or CDN.

How to Reference the Minified Build

<!-- ✅ Production: single minified file, one HTTP request -->
<link rel="stylesheet"
      href="easemotion.min.css">

<!-- 🚧 Development only: entry point with @import chain -->
<link rel="stylesheet"
      href="easemotion.css">
/* In your main CSS or JS entry point */

/* ✅ Production bundle (recommended for most bundlers) */
@import 'easemotion-css/easemotion.min.css';

/* Or in JavaScript */
import 'easemotion-css/easemotion.min.css';

/* 🚧 Dev build (keeps source maps and readable output) */
import 'easemotion-css/easemotion.css';
<!-- Via jsDelivr (replace x.x.x with the version you need) -->
<link rel="stylesheet"
      href="https://cdn.jsdelivr.net/npm/easemotion-css@x.x.x/easemotion.min.css">

<!-- Via unpkg -->
<link rel="stylesheet"
      href="https://unpkg.com/easemotion-css/easemotion.min.css">

Do / Don't

✓ Do
<!-- Production HTML -->
<head>
  <link
    rel="stylesheet"
    href="easemotion.min.css"
  >
</head>

Single file, no @import chains, browser-render-blocking is minimised.

✗ Don't
<!-- ❌ Dev entry in production -->
<head>
  <link
    rel="stylesheet"
    href="easemotion.css"
  >
</head>

Triggers 14 extra HTTP requests for @imported files and ships comments + whitespace.

Install & Rebuild

npm install
# Install the package
npm install easemotion-css

# Rebuild the minified bundle
# (maintainers only — after editing source)
npm run build

The prepublishOnly hook runs npm run build automatically before every npm publish, so the minified file is always up to date on npm.
