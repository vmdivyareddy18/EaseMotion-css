01 · Rationale

Why Audit Motion?

Motion drifts quietly as a codebase grows. Regular audits catch that drift before it becomes visible inconsistency across the product.

Consistency Across Components

Auditing surfaces components that have quietly diverged from the shared motion language.

Better User Experience

Consistent, well-timed motion reduces friction and makes interfaces feel more predictable.

Accessibility Compliance

Audits confirm reduced-motion support and safe defaults are actually implemented, not assumed.

Predictable Interactions

Users build expectations from repeated patterns; audits keep those patterns intact.

Easier Maintenance

Catching drift early is far cheaper than untangling inconsistent motion across dozens of components later.

Design System Health

Motion consistency is a measurable signal of how well a design system is actually being followed.

03 · Categories

Audit Categories

Eight categories cover the areas where motion consistency most commonly breaks down.

Timing Consistency

What to Review
Durations used across similar interactions and components.
Common Problems
Nearly identical interactions animating at noticeably different speeds.
Recommended Checks
Compare durations against the shared timing tokens.

Easing Consistency

What to Review
The easing curves applied to comparable transitions.
Common Problems
Mixed linear, ease, and custom curves used interchangeably.
Recommended Checks
Confirm each transition references a shared easing token.

Motion Hierarchy

What to Review
Whether primary actions receive more motion emphasis than secondary ones.
Common Problems
Minor UI elements animating as prominently as primary actions.
Recommended Checks
Rank components by importance and compare against motion weight.

Transition Sequencing

What to Review
The order and delay of multi-step or multi-element transitions.
Common Problems
Sequences that feel arbitrary rather than logically ordered.
Recommended Checks
Verify sequencing follows a consistent, documented pattern.

Interaction Feedback

What to Review
Hover, press, and focus motion across interactive elements.
Common Problems
Some components respond instantly while similar ones animate.
Recommended Checks
Confirm feedback motion is applied uniformly to comparable elements.

Accessibility

What to Review
Reduced-motion support and focus visibility during transitions.
Common Problems
Reduced-motion styles that exist but are never actually verified.
Recommended Checks
Test every animated component with reduced motion enabled.

Browser Consistency

What to Review
How motion renders across the supported browser matrix.
Common Problems
Subtle timing or rendering differences that go unnoticed in single-browser testing.
Recommended Checks
Re-verify key components across each supported browser.

Responsive Behaviour

What to Review
How motion adapts across breakpoints and screen sizes.
Common Problems
Animations tuned only for desktop that clip or misbehave on mobile.
Recommended Checks
Test each animated component at every defined breakpoint.

04 · Metrics

Motion Consistency Metrics

Quantifying consistency turns a subjective impression into something a team can track and improve over time.

Metric Description Recommended Goal Example
Duration Consistency How closely similar interactions share the same animation duration Comparable interactions within one shared duration token All dropdowns opening in 180ms
Shared Easing The proportion of transitions using an approved easing curve All transitions reference a documented easing token Every modal using the same standard easing curve
Animation Reuse How often components reuse existing motion patterns versus inventing new ones New one-off patterns treated as the exception, not the norm A new component reusing the existing slide-in pattern
Token Usage The share of motion values sourced from shared tokens rather than hard-coded Hard-coded duration and easing values approaching zero Duration values referencing --duration-base instead of 300ms
Accessibility Compliance Whether reduced-motion alternatives exist and are verified Every non-essential animation has a tested reduced-motion path A fade-only fallback for a component that normally slides and scales
Motion Hierarchy Whether primary actions receive proportionally more motion emphasis Motion weight scales with the importance of the action A primary CTA scaling slightly more than a secondary link
Interaction Feedback Consistency of hover, focus, and press feedback across components Comparable interactive elements share the same feedback pattern Every button applying the same hover lift and shadow

06 · Reporting

Reporting Recommendations

A useful audit report gives every finding enough structure to be triaged, assigned, and verified later.

Severity Levels

Classify each finding as critical, moderate, or minor based on its user-facing impact.

Root Cause

Explain why the inconsistency exists, not just where it was observed.

Affected Components

List every component or flow impacted so fixes can be scoped accurately.

Suggested Improvements

Recommend a specific, actionable fix rather than a general observation.

Priority

Rank the finding relative to others so teams know what to address first.

Verification Status

Track whether a finding is open, in progress, fixed, or re-verified.

AUDIT-014 Moderate

Inconsistent dropdown open duration

Root Cause
Two dropdown variants were built independently and never reconciled.
Affected Components
Navigation dropdown, filter dropdown
Suggested Improvement
Migrate both to the shared --duration-fast token.
Priority
High
Verification Status
In progress

07 · Continuous Improvement

Continuous Improvement Process

A single audit only captures a moment in time. Treating auditing as a repeating loop is what keeps a design system's motion coherent as it grows.

  1. 1

    Audit

    Run a scoped review against the current audit categories and metrics.

  2. 2

    Fix

    Address findings in priority order, starting with the highest severity.

  3. 3

    Review

    Have a second engineer or designer confirm each fix matches the intended pattern.

  4. 4

    Validate

    Re-test the fixed components against the original findings that flagged them.

  5. 5

    Document

    Update the audit report and shared documentation with the resolved status.

  6. 6

    Repeat

    Schedule the next audit rather than treating this pass as a one-time effort.

Continuous auditing catches drift while it is still cheap to fix, keeps the design system's motion language trustworthy, and turns consistency into an ongoing practice instead of a one-time cleanup.

08 · Best Practices

Best Practices

A short set of habits that make motion audits sustainable over time.

Audit Regularly

Schedule audits on a recurring cadence rather than only after something looks wrong.

Standardize Timing

Consolidate durations into a small, shared set of tokens everyone can reference.

Reuse Motion Patterns

Prefer an existing pattern over inventing a new one for a similar interaction.

Keep Accessibility First

Treat reduced-motion support as part of the definition of done, not an afterthought.

Measure Before Changing

Base fixes on documented findings rather than individual impressions.

Document Findings

Keep a written record of what was found and fixed so the next audit builds on it.

Motion consistency is not achieved once, it is maintained through repeated auditing. Treat every pass as an opportunity to catch drift early, document what was found, and keep the design system's motion language trustworthy.
