Ghost Button — Hover Contrast Fix

Demonstrates the white-on-white invisibility bug in the Ghost button variant and three approaches to fix it while maintaining WCAG AA contrast.

The Bug — hover to see the invisible button

The Ghost button variant uses white text on a dark background. On hover, the background fills to solid white — but the color property is never updated, leaving white text on a white background: a 1:1 contrast ratio. The label vanishes completely.

❌ Buggy — hover over the button
✗ 1:1 contrast on hover Text becomes invisible — WCAG 1.4.3 violation 
/* Ghost button — buggy hover */
.ghost-btn {
  background-color: transparent;
  color: #ffffff;           /* white text on dark bg ✓ */
  border: 2px solid #ffffff;
}

.ghost-btn:hover {
  background-color: #ffffff;  /* bg → white */
  /* color: ???  ← MISSING: still #ffffff = invisible! */
}

Fix A — Invert text color to dark on hover

The simplest and most compatible fix. When the background fills to white, switch the text to a near-black value. Contrast goes from 1:1 (invisible) to 15.7:1 — exceeding even WCAG AAA (7:1).

✅ Fix A — hover to confirm text is visible
✓ 15.7:1 on hover White (#fff) on dark (#0f172a) — WCAG AAA 
.ghost-btn:hover {
  background-color: #ffffff;
  color: #0f172a;          /* ✅ dark text on white bg — 15.7:1 contrast */
  border-color: #ffffff;
}

Fix B — Semi-transparent overlay (no solid fill)

Instead of filling to solid white, use a translucent white overlay (rgba(255,255,255,0.12)). The dark background remains visible through the overlay, so white text retains excellent contrast. This approach keeps the ghost aesthetic on hover.

✅ Fix B — hover: subtle overlay, text stays white
✓ ~12:1 on hover White text on dark bg through 12% white overlay 
.ghost-btn:hover {
  background-color: rgba(255, 255, 255, 0.12);  /* ✅ translucent — bg shows through */
  border-color: #ffffff;
  /* text remains #ffffff — dark bg still visible underneath */
}

Fix C — color-mix() auto-derived text (modern)

A CSS-native approach for modern browsers (Chrome 111+, Firefox 113+, Safari 16.2+). color-mix() derives the hover text color from the border token — no hard-coded dark value needed. The ghost border color and the "inverted" text always stay in sync automatically.

💡 Fix C — color-mix() — hover to test
✓ ~15:1 on hover Text derived via color-mix(in srgb, #fff, #000 85%) 
.ghost-btn {
  --ghost-border: #ffffff;
  border: 2px solid var(--ghost-border);
  color: var(--ghost-border);
}

.ghost-btn:hover {
  background-color: var(--ghost-border);
  color: color-mix(in srgb, var(--ghost-border), #000000 85%);
  /* Result: #262626 — no hard-coded value needed */
}

WCAG 1.4.3 requires a minimum contrast ratio of 4.5:1 for normal text (AA) and 7:1 for AAA. All three fixes exceed the AA threshold by a wide margin.
