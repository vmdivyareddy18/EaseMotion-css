# Overview

A static documentation page for the EaseMotion CSS repository titled
**"Motion Architecture Decision Records (ADR) Guide"**. It explains how
teams document motion-related architecture decisions throughout a
project's lifecycle, in the style of engineering handbooks like Microsoft
Learn or GitHub Docs.

## Features

- Fully static HTML and CSS, no JavaScript
- Sticky quick-navigation bar linking to every section
- Vertical timeline for the ADR lifecycle
- Structured ADR template reference
- Expandable example ADR cards using native `<details>`
- Horizontal process diagram for the review workflow
- Callout cards for best practices and common mistakes

## Sections Included

- Hero introduction
- Quick Navigation
- What is a Motion ADR?
- Why Teams Need Motion ADRs
- Motion ADR Lifecycle (timeline)
- Standard ADR Template
- Example Motion ADRs (ADR-001 through ADR-005)
- ADR Review Workflow (process diagram)
- Best Practices
- Common Mistakes
- Footer
