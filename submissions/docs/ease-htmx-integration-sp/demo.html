Integration guidance


      

        
Why retrigger animations?

        

          CSS animations run once when an element is first painted. After an
          HTMX swap, the DOM is new but the browser may not replay
          @keyframes unless you reset animation.
        

      


      

        
Reusable pattern

        
    
          
  • Add entrance classes to swapped HTML (ease-fade-in).
    • 
          
  • Listen for htmx:afterSwap on document.body.
    • 
          
  • Reset animation on animated children of evt.detail.target.
    • 
          
  • Use hx-indicator for loading feedback during requests.
    • 
        

      


      

        
Production tips

        
    
          
  • Prefer innerHTML swap for partial updates.
    • 
          
  • Stagger with ease-delay-100 … ease-delay-300.
    • 
          
  • Handle htmx:responseError for failed requests.
    • 
          
  • Respect prefers-reduced-motion — EaseMotion does by default.
    •
