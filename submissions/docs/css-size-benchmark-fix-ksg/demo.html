CSS Size Benchmark

Demonstrates the corrected label mapping between easemotion.css and easemotion.min.css.

⚠️ The Bug (Before Fix)

The benchmark script had the two EaseMotion file paths assigned to the wrong labels, producing a nonsensical result:

Label File actually read Raw size Gzip size
EaseMotion CSS (min) easemotion.css ← wrong file 2.8 KB 858 B
EaseMotion CSS (full) easemotion.min.css ← wrong file 174.1 KB 28.2 KB

Result: the "minified" version appeared 62× larger than the "full" version — the opposite of reality.

✅ After Fix — Correct Mapping

Swapping the file paths to match their correct labels gives the expected result: the minified bundle is always larger than the source entry file (which only contains @import statements).

Label File read Raw size Gzip size
EaseMotion CSS (min) easemotion.min.css ✓ ~71 KB ~14 KB
EaseMotion CSS (full) easemotion.css ✓ ~1.9 KB ~612 B

The minified bundle is larger because it is the fully inlined and minified output of all source files. The "full" entry file only holds @import declarations.

The One-Line Fix

In the benchmark script, two file path constants were simply swapped: 

// BEFORE (buggy)
{ library: "EaseMotion CSS (min)",  filePath: "easemotion.css"     }
{ library: "EaseMotion CSS (full)", filePath: "easemotion.min.css" }

// AFTER (fixed)
{ library: "EaseMotion CSS (min)",  filePath: "easemotion.min.css" }
{ library: "EaseMotion CSS (full)", filePath: "easemotion.css"     }
