Inter Font FOIT + Entrance Animation Flash

The README encourages Inter for typography. When that font finishes loading while ease-fade-in / ease-slide-up are running, text can jump or flash — looking like a motion bug even when both systems work. This lab reproduces the glitch and shows practical fixes.

Issue #46176
What you’re seeing: FOIT/FOUT happens when a webfont swaps in after first paint. EaseMotion entrance utilities animate opacity/transform at the same time — so a late metric change feels like a “flash” or jump mid-fade/slide.

Why this happens

  • Font metrics change mid-animation Jump

    Inter and Georgia (or system UI) have different widths/heights. When the stack swaps during ease-slide-up, glyphs reflow — it reads as a glitch, not a font load.

  • FOIT vs FOUT display

    font-display: block can hide text briefly (FOIT). swap shows fallback quickly (FOUT) but may still jump. optional can skip late swaps on slow networks — often best when entrance motion is critical above the fold.

  • README order matters preconnect

    Without early preconnect to fonts.googleapis.com and fonts.gstatic.com, Inter arrives later — right in the middle of EaseMotion entrance timings (~150–300ms).

font-display cheat sheet

Value Behavior With entrance motion
block Invisible text wait (FOIT), then swap High flash risk mid-fade/slide
swap Fallback immediately, then Inter OK for most pages; pair with metrics
optional May keep fallback if font is late Best when motion must stay stable
fallback Short block, then swap window Compromise between block and swap

Copy-paste snippets

Avoid late-loading Inter during first paint + entrance. Prefer the green setup that matches the README’s preconnect order and adds metrics.

Avoid Late / no preconnect 
<!-- ❌ No preconnect · display=block · loads late -->
<link
  href="https://fonts.googleapis.com/css2?family=Inter:wght@400;600;700&display=block"
  rel="stylesheet"
/>
<h1 class="ease-fade-in">Build faster.</h1>
<p class="ease-slide-up">Animation-first CSS for humans.</p>
Use this Mitigated (recommended) 
<link rel="preconnect" href="https://fonts.googleapis.com" />
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
<link
  href="https://fonts.googleapis.com/css2?family=Inter:wght@400;600;700&display=swap"
  rel="stylesheet"
/>
<style>
  @font-face {
    font-family: "Inter Fallback";
    src: local("Arial");
    size-adjust: 107%;
    ascent-override: 90%;
    descent-override: 22%;
    line-gap-override: 0%;
  }
  body {
    font-family: Inter, "Inter Fallback", system-ui, sans-serif;
  }
</style>

Production checklist

  • ✓ Keep README order: preconnect Google Fonts → preconnect gstatic (crossorigin) → Inter stylesheet.
  • ✓ Prefer display=swap (README default) or optional for hero + entrance motion.
  • ✓ Add fallback metrics (size-adjust / ascent/descent overrides) so a late swap barely jumps.
  • ✕ Don’t inject Inter midway through ease-fade-in / ease-slide-up.
  • ✕ Don’t blame EaseMotion first — check font timing when text “flashes.”
