Flip / 3D backface-visibility Safari Glitch Doc

Safari can flicker, flash the reverse face, or briefly show both sides during ease-flip / flip-x / flip-y motion when backface-visibility, preserve-3d, and perspective are stacked incorrectly. This lab reproduces the glitch and documents a Safari-safe stack — 3D-specific, not iOS 100vh / navbar.

Why Safari flickers

  • Safari is stricter about compositor layers for 3D transforms. Missing -webkit-backface-visibility: hidden often leaves the reverse face briefly visible.
  • perspective on the wrong node (animated child instead of parent) fights transform-style: preserve-3d.
  • Without a tiny translateZ(0.1px) (or similar), faces can z-fight at the midpoint of a flip.
Tip: Test flip cards in Safari / iOS Safari. Chrome may look fine with an incomplete stack — that is the trap.

Hover or focus

/* Safari-safe stack */
.scene {
  perspective: 1000px;
}
.inner {
  position: relative;
  transform-style: preserve-3d;
  -webkit-transform-style: preserve-3d;
  transition: transform 0.6s ease;
}
.face {
  position: absolute; inset: 0;
  backface-visibility: hidden;
  -webkit-backface-visibility: hidden;
  transform: translateZ(0.1px);
}
.back { transform: rotateY(180deg) translateZ(0.1px); }

Safari-safe property checklist

  1. Put perspective on the scene / parent, not only on the rotating face.
  2. Set transform-style: preserve-3d (+ -webkit-transform-style) on the inner.
  3. Set backface-visibility: hidden (+ -webkit-backface-visibility) on both faces.
  4. Offset faces with a tiny translateZ(0.1px) to avoid mid-flip z-fighting.
  5. Prefer EaseMotion ease-flip-3d / ease-flip-3d-face as the base, then harden with webkit prefixes in project CSS.

Copy-friendly patterns

1. Core entrance (fine alone)

<div class="ease-flip">Hello</div>

ease-flip already sets perspective + backface-visibility. Glitches show up when you nest custom double-sided faces without the full stack.

2. Card flip (use ease-flip-3d)

<div class="ease-flip-3d">
  <div class="ease-flip-3d-inner">
    <div class="ease-flip-3d-face">Front</div>
    <div class="ease-flip-3d-face ease-flip-3d-back">Back</div>
  </div>
</div>

3. Harden for Safari (project CSS)

.ease-flip-3d-inner {
  -webkit-transform-style: preserve-3d;
  transform-style: preserve-3d;
}
.ease-flip-3d-face {
  -webkit-backface-visibility: hidden;
  backface-visibility: hidden;
  transform: translateZ(0.1px);
}
.ease-flip-3d-back {
  transform: rotateY(180deg) translateZ(0.1px);
}

4. When to use which

  • ease-flip — single-face entrance
  • ease-flip-3d — double-sided hover cards
  • Project harden — always add webkit + translateZ
  • Do not — put perspective only on the face
