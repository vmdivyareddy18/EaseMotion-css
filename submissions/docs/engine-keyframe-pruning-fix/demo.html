Root cause

      

        usedKeyframes is built only from
        ease-*-prefixed classes via a naive
        ease-* → ease-kf-* string replace.
        em="..." attributes compile to hashed
        _em_xxxxx class names, which never match that
        pattern — so the keyframe an em-generated rule actually needs
        is never registered before pruning runs, and gets stripped.
      

      
Fix

      

        Export the compiler's existing KEYFRAME_MAP and,
        while processing each parsed em AST, register its
        mapped keyframe into usedKeyframes before pruning
        executes. See linked PR for the full engine-file diff.
