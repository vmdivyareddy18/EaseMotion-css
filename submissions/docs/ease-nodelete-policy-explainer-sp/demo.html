Golden rule: Your PR must only add files inside one of the four allowed tracks: submissions/examples/, submissions/docs/, submissions/react/, or submissions/scss/. Never edit core/, components/, or delete existing code.

📏 Line-count ≥ 250 rule

Submissions should meet 250+ total lines across all new files, or include every required file for the track with meaningful content.

  • Thin 20-line demos get flagged as low-effort
  • Count includes HTML, CSS, README, and JSX/SCSS
  • Padding with blank lines does not count — write real content

📁 3-file rule (docs / examples track)

Standard HTML/CSS and docs tracks require exactly three files:

  • demo.html — self-contained browser demo
  • style.css — your styles
  • README.md — what / how / why

🚫 No-delete policy

Any diff that removes lines from existing framework files triggers auto-close. Your PR should show only green additions in paths you own.

📂 Four allowed tracks only

Never create folders directly under submissions/ root. Always nest inside one of the four track subdirectories.

Pre-PR compliance checklist

  • All files are inside submissions/examples/, docs/, react/, or scss/
  • No edits to core/, components/, root configs, or workflows
  • No deletions of existing repository files
  • Folder name includes your unique suffix (e.g. -sp)
  • All required track files are present (3-file rule for docs/examples)
  • Total contribution is 250+ lines OR fully complete with meaningful content
  • One feature per PR — squash commits before review
  • PR template checklist is fully addressed
