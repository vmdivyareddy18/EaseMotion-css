Bug Fix: validate-package.mjs URL Case-Sensitivity

This submission fixes a bug that causes npm run release:check to fail on fresh clones because the repository.url check is strictly case-sensitive.

Before (Fails)

if (!manifest.repository?.url?.includes("SAPTARSHI-coder/EaseMotion-css")) {
  fail("repository.url must point to SAPTARSHI-coder/EaseMotion-css");
}

After (Fix)

if (!manifest.repository?.url?.toLowerCase().includes("saptarshi-coder/easemotion-css")) {
  fail("repository.url must point to SAPTARSHI-coder/EaseMotion-css");
}
