Why this breaks
  • .ease-reveal starts at opacity: 0 but still occupies zero image height.
  • reveal.js adds .ease-reveal-active on intersect — before the image loads.
  • When the image arrives, layout height changes → cumulative layout shift.

Fixed — CLS-safe patterns

Low CLS risk

Uses aspect-ratio, skeleton placeholder, and optional delayed reveal until load event.

Fix patterns used
  • aspect-ratio on wrapper (16 / 9)
  • width / height attributes on <img>
  • Skeleton placeholder until image loads
  • Delayed reveal — wait for load before adding active class

Best practices for ease-reveal + media
  • Always reserve space for images, video, and embeds before reveal animates.
  • Prefer aspect-ratio + object-fit: cover for responsive galleries.
  • Use ease-skeleton components while async content loads.
  • For heavy pages, delay .ease-reveal-active until critical images fire load.
  • Test with throttled network in DevTools → Performance → Experience.
