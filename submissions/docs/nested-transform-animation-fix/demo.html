Nested Transform Conflict — Fix Demo

When a parent and child both use transform-based animations, the child's transform overrides the parent's. This page shows the bug and the fix side-by-side.

❌ The Bug — Nested transforms cancel each other

The parent floats using transform: translateY(). The child scales using transform: scale(). Because CSS transform is a single property, the child's value completely replaces the parent's — the float is cancelled.

Parent: ease-float (translateY)
Child: ease-scale (scale)
← Float is cancelled here

✅ The Fix — Use individual transform properties

Modern CSS supports translate, scale, and rotate as independent properties. They compose on top of each other correctly — the parent's float and the child's scale both play simultaneously without conflict.

Parent: float using translate property
Child: scale using scale property
✅ Both animations compose!

🔧 Why this happens & how to fix it

❌ Buggy — single transform property

/* Parent keyframe */
@keyframes float {
  0%, 100% { transform: translateY(0); }
  50%       { transform: translateY(-10px); }
}

/* Child keyframe — OVERRIDES parent's transform! */
@keyframes pulse-scale {
  0%, 100% { transform: scale(1); }
  50%       { transform: scale(1.08); }
}

✅ Fixed — individual transform properties

/* Parent: uses standalone `translate` property */
@keyframes float-fixed {
  0%, 100% { translate: 0 0; }
  50%       { translate: 0 -10px; }
}

/* Child: uses standalone `scale` property — COMPOSES! */
@keyframes pulse-scale-fixed {
  0%, 100% { scale: 1; }
  50%       { scale: 1.08; }
}

Root cause: CSS transform is a single shorthand property. When a child element sets transform, it completely replaces any transform inherited or applied by the parent's animation on its own stacking context. There is no "merging" of transform values across parent/child boundaries.

The fix: CSS now supports translate, scale, and rotate as independent longhand properties (Chrome 104+, Firefox 72+, Safari 14.1+). Each is applied separately and they compose naturally — the parent's translate and the child's scale both apply without conflict.
