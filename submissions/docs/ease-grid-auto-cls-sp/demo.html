Grid Auto-Fit + Entrance Animations CLS Mini-Lab

ease-grid-auto cards with ease-zoom-in can cause visible reflow and Cumulative Layout Shift. This lab reproduces the combo and documents containment patterns that keep the grid stable — distinct from reveal-CLS or loop-CPU issues.

Why this combo shifts layout

  1. ease-zoom-in starts at scale(0.85) and uses bounce easing that can overshoot past scale(1).
  2. ease-grid-auto uses repeat(auto-fit, minmax(...)) — track sizes react to content and overflow.
  3. When zoom is applied on the grid item itself, overshoot + overflow can expand rows, nudge neighbors, or flash scrollbars — registered as layout shift.

Containment patterns (copy-friendly)

Keep motion. Isolate it from the auto-fit grid item. No core edits required.

1. Broken (avoid)

<div class="ease-grid ease-grid-auto ease-gap">
  <article class="card ease-zoom-in">…</article>
  <article class="card ease-zoom-in ease-delay-200">…</article>
</div>

Animation target = grid child. Overshoot can disturb tracks.

2. Shell + inner zoom (preferred)

<div class="ease-grid ease-grid-auto ease-gap">
  <article class="card-shell">
    <div class="card-inner ease-zoom-in">…</div>
  </article>
</div>
.card-shell {
  min-height: 8rem;      /* reserve space */
  overflow: hidden;      /* clip overshoot */
  contain: layout;       /* isolate reflow */
}
.card-inner { height: 100%; }

3. Prefer fade/slide on grid items

<!-- Safer on the grid child itself -->
<article class="card ease-fade-in ease-delay-200">…</article>

Opacity-only entrances do not scale the box. Save zoom for inner media or hero elements.

4. When to use which

  • Shell + inner — you want zoom on auto-fit cards
  • Fade / slide — simplest CLS-safe entrance
  • Avoid — ease-zoom-in directly on ease-grid-auto children
  • Do not — change core utilities for a page-level fix
