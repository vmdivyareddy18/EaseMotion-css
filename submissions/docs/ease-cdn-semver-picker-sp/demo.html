✓ Pin a version (production)

              
    
                
  • URL always serves the exact same bytes — reproducible builds.
    • 
                
  • No surprise breaking changes when the repo updates.
    • 
                
  • jsDelivr caches pinned versions permanently — fastest response.
    • 
                
  • Upgrade deliberately: change the URL, test, then ship.
    • 
              

            

            

              
⚠ Track @main (prototypes only)

              
    
                
  • Content changes whenever the default branch changes.
    • 
                
  • A refactor or class rename upstream can break your live site.
    • 
                
  • jsDelivr caches branch URLs up to 12 hours — updates are delayed and inconsistent across regions.
    • 
                
  • Fine for demos, CodePens, and issue reproductions — never production.
    •

URL formats per provider

        
    
          
  • jsDelivr (GitHub): cdn.jsdelivr.net/gh/user/repo@version/file
    • 
          
  • jsDelivr (npm): cdn.jsdelivr.net/npm/easemotion-css@version/file
    • 
          
  • unpkg (npm): unpkg.com/easemotion-css@version/file
    • 
          
  • unpkg has no @main — it serves npm releases only.
    • 
        

      


      

        
Cache behavior notes

        
    
          
  • Pinned version URLs are cached permanently at the edge.
    • 
          
  • Branch URLs (@main) refresh within ~12 hours, not instantly.
    • 
          
  • Purging a branch URL propagates slowly across regions.
    • 
          
  • If styles look stale after a push, the branch cache is the usual cause.
