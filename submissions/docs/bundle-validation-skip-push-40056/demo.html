🐛 Bundle Validation Skip on Push Events

Issue #40056: Stale Minified CSS Slips Through to Main

The Problem

❌ Current Behavior (Skips Validation)

1. Push Event to main git push origin main Push event triggered
⬇
2. GitHub Actions Checks GITHUB_BASE_REF = undefined Not set on push events
⬇
3. Script Falls Back to main git diff --name-only origin/main After checkout, origin/main = current commit
⬇
4. Diff is Empty No files changed ⚠️ origin/main already = HEAD
⬇
5. Validation Skipped "No core, component, or entrypoint files modified. Skipping bundle validation." 🔴 CRITICAL: Stale CSS Slips Through!

⚠️ Risk Assessment

Stale Minified CSS

Outdated easemotion.min.css can be deployed to production without validation

Silent Failure

No error reported — users won't know CSS is out of sync

Production Impact

Styles may break or behave differently than intended

Automation Bypass

Direct pushes or automation can bypass validation entirely

Root Cause Analysis

Why Does This Happen?

1. Environment Variable Mismatch

Pull Request Events: GITHUB_BASE_REF is set (target branch)

Push Events: GITHUB_BASE_REF is undefined (no "base" for direct push)

2. Fallback Strategy Problem

When GITHUB_BASE_REF is undefined, script defaults to comparing against origin/main

But on a push to main, origin/main already points to the newly pushed commit!

3. Git Diff Empty Result

# After push to main and checkout: git diff --name-only origin/main # origin/main = HEAD # Result: (empty) # Reason: Comparing HEAD against itself = no changes

4. Validation Skipped

Empty diff triggers "no modifications" logic, skipping all bundle validation

Stale easemotion.min.css is never verified

The Solution

✅ Differentiate Between Push and Pull Request Events

❌ Before (Fails on Push)

// scripts/validate-bundle.mjs const baseRef = process.env.GITHUB_BASE_REF || 'main'; const compareRef = 'origin/' + baseRef; // Problem: On push to main, origin/main = HEAD const diff = execSync( `git diff --name-only ${compareRef}` ).toString().split('\n').filter(Boolean); if (diff.length === 0) { console.log('Skipping bundle validation...'); process.exit(0); }

✅ After (Works for Push & PR)

// scripts/validate-bundle.mjs const isPullRequest = !!process.env.GITHUB_BASE_REF; const eventName = process.env.GITHUB_EVENT_NAME; let compareRef; if (isPullRequest) { // PR: Compare against target branch compareRef = 'origin/' + process.env.GITHUB_BASE_REF; } else if (eventName === 'push') { // Push: Use before/after SHAs const { before, after } = JSON.parse( fs.readFileSync(process.env.GITHUB_EVENT_PATH, 'utf8') ).push; compareRef = before; } else { // Fallback: Compare against main compareRef = 'origin/main'; } const diff = execSync( `git diff --name-only ${compareRef}` ).toString().split('\n').filter(Boolean); if (diff.length === 0) { console.log('No relevant files changed.'); process.exit(0); }

Implementation Approach:

  1. Detect workflow event type (GITHUB_EVENT_NAME)
  2. For pull_request events: Use GITHUB_BASE_REF
  3. For push events: Extract before/after SHAs from GitHub event JSON
  4. For other events: Fall back to origin/main
  5. Compare against the appropriate reference to detect changes
  6. Test both workflows to ensure validation runs correctly

Technical Details

Environment Variables

GITHUB_BASE_REF: Set only on pull_request events (target branch)

GITHUB_EVENT_NAME: Type of event (push, pull_request, etc.)

GITHUB_EVENT_PATH: Path to webhook payload JSON

Push Event Payload

before: Previous commit SHA (parent of push)

after: New commit SHA (pushed commit)

Allows precise diff calculation for push events

Git Diff Behavior

Comparing HEAD against itself = empty diff

After push, origin/main = HEAD automatically

Must use before SHA for accurate change detection

Best Practice

Always differentiate event types in CI/CD

Push and PR workflows behave differently

Adjust comparison refs based on event context
