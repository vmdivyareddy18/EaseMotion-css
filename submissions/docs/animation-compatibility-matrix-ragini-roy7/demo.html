Animation Compatibility Matrix

Which EaseMotion utilities combine safely — and which silently override each other.

✅ Safe combo

fade-in only

ease-fade-in alone — animates opacity + transform: scale() cleanly.

❌ Conflicting combo

fade-in + slide-up together

Both classes set the animation property — only ease-slide-up plays; the fade-in keyframe never runs.

Entrance animations

One-shot, animation: ... both — only one can play per element.

AnimationCompatibleAvoidWhy
ease-fade-in ✓ OK hover utilities ✗ Conflict other entrance classes Only one animation shorthand wins
ease-slide-up ✓ OK hover utilities ✗ Conflict other entrance classes Same animation collision
ease-zoom-in / ease-zoom-out ✓ OK hover utilities ✗ Conflict other entrance, ease-flip, ease-mask-reveal Same animation collision
ease-bounce-in ✓ OK hover utilities ✗ Conflict other entrance classes Same animation collision
ease-flip / ease-flip-3d ✓ OK hover utilities, ease-card-lift ✗ Conflict other entrance, ease-rotate Both animate transform: rotate*
ease-mask-reveal (+ variants) ⚠ Partial ease-fade-in (different properties, verify timing) ✗ Conflict other mask-reveal variants, ease-slide-* clip-path vs transform overlap depends on variant

Continuous / looping animations

animation-iteration-count: infinite — meant to run indefinitely.

AnimationCompatibleAvoidWhy
ease-bounce ✓ OK ease-hover-glow, ease-pulse ✗ Conflict ease-rotate, ease-wave, ease-shake, ease-float All animate transform continuously
ease-rotate ✓ OK ease-pulse, ease-hover-glow ✗ Conflict ease-bounce, ease-wave, ease-shake, ease-flip-3d transform collision
ease-pulse / ease-ping ✓ OK almost everything (isolated opacity/scale) ✗ Conflict each other Both use animation — same-property rule still applies
ease-wave / ease-shake / ease-float ✓ OK ease-hover-glow, ease-pulse ✗ Conflict ease-bounce, ease-rotate, each other transform collision
ease-gradient-rotation ✓ OK almost everything — Animates background-position — no overlap with anything else
ease-shimmer-sweep / ease-skeleton-shimmer ✓ OK almost everything ✗ Conflict each other Both animate background

Delay & duration modifiers

Utility familyCompatibleNotes
ease-delay-* (75–1000ms) ✓ OK any animation class Only adjusts animation-delay
ease-transition-delay-* ✓ OK any hover/transition utility Only adjusts transition-delay
ease-duration-fast/medium/slow ✓ OK any animation class Only adjusts animation-duration
ease-reveal-delay-1…-5 ✓ OK sibling elements, for staggering Purpose-built for staggered reveals
The one rule to remember: if two classes animate the same CSS property (transform, animation, filter, background), only one wins. Classes touching different properties compose safely. When unsure, check Computed Styles in dev tools for a property collision.
