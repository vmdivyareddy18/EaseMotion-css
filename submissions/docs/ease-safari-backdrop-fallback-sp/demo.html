Safari -webkit-backdrop-filter Glass Fallback

The README browser table says Safari needs -webkit-backdrop-filter for glass effects. This lab shows cards with / without the prefix, and a mimic-unsupported mode for the solid fallback path.

README note: Safari 9.1+ supports backdrop-filter with a -webkit-backdrop-filter fallback. EaseMotion glass (e.g. ease-navbar-glass) ships both on purpose.

Why Safari needs the prefix

  • WebKit shipped the prefixed name first History

    Older Safari builds recognize -webkit-backdrop-filter even when the unprefixed property alone isn’t enough for your target matrix.

  • Missing prefix → “glass is broken” Bug lookalike

    Authors often report broken EaseMotion glass on Safari when they copy only backdrop-filter into custom CSS.

  • Always pair with a solid fallback Resilience

    Use @supports not ((backdrop-filter) or (-webkit-backdrop-filter)) (or mimic mode above) so unsupported engines still get readable UI.

Copy-paste snippets

Avoid Unprefixed only 
.glass {
  background: rgba(255, 255, 255, 0.22);
  backdrop-filter: blur(16px);
  /* ❌ No -webkit-backdrop-filter — fragile on Safari */
}
Use this Safari-safe + @supports fallback 
.glass {
  background: rgba(255, 255, 255, 0.22);
  border: 1px solid rgba(255, 255, 255, 0.35);
  backdrop-filter: blur(16px);
  -webkit-backdrop-filter: blur(16px); /* Safari / WebKit */
}

@supports not ((backdrop-filter: blur(0)) or (-webkit-backdrop-filter: blur(0))) {
  .glass {
    background: rgba(255, 255, 255, 0.94);
    border-color: rgba(15, 23, 42, 0.12);
  }
}
