Neumorphic + Glass Combo Contrast Fail Board

Stacking ease-card-neumorphic, ease-card-glass, and glow hover on a light soft-UI background can wash out text and controls. This board measures those fails and documents safer combos — different from dark glass + glow issues.

What goes wrong on light soft-UI

  • Neumorphic uses a soft page-tint background (--ease-color-bg) with low-contrast dual shadows.
  • Glass adds translucent white (rgba(255,255,255,0.35) + blur — text sits on a washed surface.
  • Glow hover (ease-card-glow / ease-hover-glow) blooms primary light over the mix and further softens perceived contrast.

WCAG AA needs ≥ 4.5:1 for normal text and ≥ 3:1 for large text / UI components.

Avoid (fail stack)

  ease-card-glass
  ease-card-glow">
  <p class="muted">…</p>

Three soft layers + muted ink ≈ contrast fail on light UI.

Prefer (one surface)

  <p>Body uses default dark text tokens.</p>
</article>

/* project CSS — not core */

When to use which

  • Neumorphic alone — soft-UI cards with dark text
  • Glass alone — decorative shell + opaque ink panel
  • Glow — border/shadow accent, never light text color
  • Do not — stack neuro + glass + glow on light stages
