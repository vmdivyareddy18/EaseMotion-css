CSS Custom Properties & Shadow DOM

Verifying that --css-vars pierce through simulated shadow DOM boundaries

How CSS Custom Properties Cross Boundaries

CSS custom properties (CSS variables) are inherited by design — they flow through the DOM tree, including shadow DOM boundaries. Unlike many other CSS properties that are scoped to shadow DOM, custom properties pierce the shadow boundary. This demo simulates nested shadow DOM contexts using regular HTML containers with scoped styles.

Limitations & Workarounds

Scenario Inherits? Notes
Normal custom property ✅ Yes Inherited by default across shadow boundaries
:host context ✅ Yes Custom properties on host element are inherited by shadow tree
:part() styling ⚠️ Partial Only exposes specific parts; exportparts attribute needed
Scoped CSS (@scope) ✅ Yes @scope respects inheritance boundaries
Regular CSS properties (e.g., color) ❌ No Regular properties do NOT pierce shadow DOM — only custom properties do

Workaround: :host and exportparts

When styling inside a shadow tree, use :host to expose customization points:

/* Inside shadow DOM — expose via :host */
:host {
  --component-bg: var(--theme-bg, #1a1a1a);
  --component-text: var(--theme-text, #d1d5db);
}

/* Consumer sets variables on the host element */
my-component {
  --theme-bg: #2a2a2a;
  --theme-text: #ffffff;
}

Using ::part() for Deep Styling

/* Component exposes a part */
<my-component>
  #shadow-root
    <div part="header">...</div>
</my-component>

/* Consumer styles that part */
my-component::part(header) {
  background: var(--header-bg, #1a1a1a);
}

How It Works

CSS custom properties (--* variables) are the only CSS properties that naturally cross shadow DOM boundaries. This is by specification (CSS Custom Properties for Cascading Variables Level 1). When a custom property is set on a host element, its value is inherited by all elements within the shadow tree.

For deeper customization, use :host to expose internal variables and ::part() selectors to target specific elements. The exportparts attribute on nested shadow hosts exposes their parts to outer consumers.
