Layout Correction Fix

Masonry Card Spacing Alignment

Issue Fixed: Previously, direct child cards inside a masonry grid container were assigned margin-bottom: inherit;. Because parent masonry containers rarely carry a margin-bottom, the cards resolved this value to 0, removing their vertical spacing and causing them to touch and overlap.

We resolved this by replacing the inheritance rule with an explicit margin variable mapping, preserving spacing alignment.

Card 1: Dashboard Analytics

This is a short card containing card statistics and summaries. Observe the gap underneath this card—it keeps a clean vertical space separating it from Card 4.

Card 2: Activity Feed

This card is slightly taller. It lists user operations, background jobs, system logs, and commits. The spacing below is maintained correctly to prevent visual collision.

Card 3: Settings

Short dashboard settings configuration options.

Card 4: Detailed Report

A detailed analysis report. In Pinterest-style columns, items slide up to fill gaps. Prior to this fix, this card would press tightly against Card 1 above it with zero gap.
