Tailwind + EaseMotion — Compatibility Fix

Fixes #8404 — replaces animation shorthand with longhand properties so EaseMotion and Tailwind animations stack on the same element instead of overriding each other.

Before vs After

ScenarioBefore (shorthand)After (longhand)

✅ Both Together — EaseMotion + Tailwind

Both libraries' animations now stack on the same element. EaseMotion loads after Tailwind in this demo — before the fix, EaseMotion would have overridden everything.

Fades in AND spins .ease-fade-in.animate-spin

EaseMotion entry + Tailwind continuous

💜 Slides up AND pulses .ease-slide-up.animate-pulse

EaseMotion entry + Tailwind continuous

⬆️ Zooms in AND bounces .ease-zoom-in.animate-bounce

EaseMotion entry + Tailwind continuous

Why longhand works: The CSS animation shorthand resets ALL animation sub-properties. Two classes using shorthand on the same element conflict. Longhand properties (animation-name, animation-duration etc.) are independent — each class sets only what it owns, and the browser merges them correctly.
