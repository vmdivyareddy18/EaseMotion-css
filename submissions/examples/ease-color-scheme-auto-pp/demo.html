Theme Diagnostic Checklist

Know what to expect from native browser schemes vs explicit token rules.

What Changes Automatically:

  • ✓
    Browser scrollbars (track and thumb color)
  • ✓
    Native text input field backgrounds & borders
  • ✓
    Select dropdown lists and option menu containers

What Requires Manual Variables:

  • !
    Custom card background classes (e.g. .pp-card)
  • !
    App-level layout borders and custom divide colors
  • !
    Linear gradient texts and logo graphic assets

Implementation Snippets

Use these boilerplate templates to configure theme logic in your stylesheets.

Step 1: Declare color-scheme 
/* Declare on root so scrollbars and inputs darken automatically */
:root {
  color-scheme: light dark;
}
Step 2: Remap custom variables 
/* Explicitly override variables on prefers-color-scheme */
@media (prefers-color-scheme: dark) {
  :root {
    --bg-card: #0f172a;
    --border-color: #1e293b;
    --text-primary: #f8fafc;
  }
}
