Keyboard Behavior (CSS-limited)

Key Expected CSS Support
Tab Move focus to / within dropdown ✅ :focus-within
Enter / Space Activate focused item ⚠️ Native browser behavior
↑ / ↓ Navigate between items ❌ Requires JavaScript (role="menuitem" with native Tab works)
Escape Close dropdown, return focus to trigger ❌ Requires JavaScript

How It Works

This dropdown uses the :focus-within pseudo-class to show the menu items when the container or any of its children has focus. The dropdown opens when you Tab into it and closes when you Tab away.

/* Show menu when container or child has focus */
.dropdown:focus-within .dropdown-menu {
  display: block;
  opacity: 1;
  transform: translateY(0);
}

/* Ensure menu items are in tab order */
.dropdown-menu [role="menuitem"] {
  tabindex: 0; /* implicitly via tabindex in HTML */
}

/* Hide menu by default */
.dropdown-menu {
  display: none;
  opacity: 0;
  transform: translateY(-4px);
}

Note: Pure CSS cannot detect arrow keys or Escape. For full keyboard operability (arrow navigation, Escape to close), JavaScript is needed. This demo shows what CSS can achieve with :focus-within and proper tabindex attributes, which provides basic open/close on focus/blur.
