2. Recommended Fix — ease-tooltip Keyboard Support

Add :focus and :focus-visible triggers alongside :hover.

/* Fix: tooltip keyboard accessibility */
[data-tip]:hover::after,
[data-tip]:focus::after,
[data-tip]:focus-visible::after {
  opacity: 1;
  transform: translateX(-50%) scale(1);
}

3. Recommended Fix — ease-progress ARIA Pattern

Add role="progressbar" with value attributes for screen readers.

<div class="ease-progress"
     role="progressbar"
     aria-valuenow="75"
     aria-valuemin="0"
     aria-valuemax="100"
     aria-label="Loading progress">
  <div class="ease-progress-bar" style="--ease-progress: 75%;"></div>
</div>
