Customization

Tune it with five CSS variables

Every modal is driven entirely by custom properties — override them globally on :root, or per-modal on the #modal-id selector, as shown in the variations above.

Variable Purpose
--modal-duration Speed of the fade/scale transition (e.g. 0.35s).
--modal-radius Corner rounding of the modal card.
--modal-overlay Background color of the soft, blurred overlay backdrop.
--modal-shadow Elevation shadow applied to the modal card.
--modal-scale Starting scale value the card animates up from (e.g. 0.94).
/* Override per modal instance */ #custom-modal { --modal-duration: 0.35s; --modal-radius: 18px; --modal-overlay: rgba(15,23,42,.5); --modal-shadow: 0 30px 60px rgba(5,150,105,.25); --modal-scale: 0.94; }
Accessibility

Checkout that works for everyone

A broken checkout modal costs sales. This component is built to stay usable for keyboard and screen-reader shoppers.

⌨ Keyboard Accessible

Every trigger, close control, and shipping option is operable with Tab and Enter.

🔆 Visible Focus States

A high-contrast focus ring appears on every interactive element via :focus-visible.

🏷 Semantic HTML

role="dialog", aria-modal, and aria-labelledby give assistive tech a clear landmark.

🎯 Large Touch Targets

Buttons and shipping option rows meet a 44px+ minimum hit area for confident mobile tapping.

🌓 High Contrast

Text, prices, and CTA colors are tuned to meet WCAG AA contrast against their backgrounds.

📱 Responsive Layout

Product rows, buttons, and forms reflow gracefully from desktop down to small mobile screens.
