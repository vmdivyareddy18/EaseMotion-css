CSS :target-current Navigation Utilities

Automatically highlight the currently active navigation item in tables of contents and documentation sites — without JavaScript. Uses the upcoming :target-current pseudo-class with a robust :target fallback.

🎯 :target-current 📖 Table of Contents ⬇️ :target fallback

💡 What is :target-current? It's an upcoming CSS pseudo-class that matches a navigation link (<a href="#section">) when the element it points to is currently the scroll target — i.e., visible at the top of the viewport. Unlike :target which only matches when the hash is in the URL, :target-current tracks the user's scroll position in real time. Until browser support arrives, we use :target as the working fallback.

Introduction

Documentation websites and long-form content often require active section highlighting. The :target-current pseudo-class provides a future-friendly, declarative CSS solution aligned with the evolution of the CSS specification.

Click any link in the sidebar to navigate to that section. Notice how the active link is highlighted — no JavaScript required. This is powered by the CSS :target pseudo-class as a working fallback for the upcoming :target-current.

Usage

Apply .ease-toc to your navigation container. Each anchor link inside will automatically receive the active style when its target section is in the URL hash. 

<nav class="ease-toc">
  <a href="#intro">Introduction</a>
  <a href="#usage">Usage</a>
</nav>

<section id="intro">...</section>
<section id="usage">...</section>

The CSS then uses :target-current (future) and :target (present fallback) to highlight the correct link.

Variants

Three visual variants are included to suit different design contexts:

  • .ease-toc — Default: purple highlight with left border accent
  • .ease-toc-pill — Pill-style background highlight
  • .ease-toc-underline — Subtle underline indicator

All variants work with the same HTML structure — just swap the class on the <nav> element.

Fallback Strategy

Since :target-current is not yet natively supported in any browser, the implementation uses :target as a working fallback combined with @supports selector() to forward-progressively adopt :target-current when browsers implement it. 

/* Working today — :target fallback */
.ease-toc a:target {
  color: #6c63ff;
  font-weight: 600;
}

/* Future — :target-current (upcoming spec) */
@supports selector(a:target-current) {
  .ease-toc a:target-current {
    color: #6c63ff;
    font-weight: 600;
  }
}

Browser Support

:target is supported in all browsers. :target-current is a proposed addition to the CSS specification and is not yet available in any stable browser release. The utility is built to progressively enhance — working today with :target and automatically improving as :target-current lands.

  color: #6c63ff;
  font-weight: 600;

/* ── Working :target fallback (used today) ─────────────── */
/* Targets the SECTION with the matching hash in the URL.  */
/* We use the :target pseudo-class on sections and then    */
/* style links via a sibling/general approach in JS-free   */
/* way using :target on the anchor directly.               */
.ease-toc a:target {
  color: #6c63ff;
  font-weight: 600;
  border-left: 3px solid #6c63ff;
  padding-left: 0.75rem;
}

/* ── Future progressive enhancement ────────────────────── */
@supports selector(a:target-current) {
  .ease-toc a:target-current {
    color: #6c63ff;
    font-weight: 600;
    border-left: 3px solid #6c63ff;
    padding-left: 0.75rem;
  }
}
