# Service Ownership Catalog Component

## What does this do?

Provides a centralized interface for tracking service ownership, team responsibilities, on-call assignments, SLAs, dependencies, and operational health.

## How is it used?

```html
<div class="service-card healthy">
    <h3>Customer Profile Service</h3>
</div>
```

## Why is it useful?

Modern platform engineering teams need clear ownership and accountability for services. This component provides a reusable service catalog interface that helps teams manage ownership, governance, reliability, and operational responsibilities.
