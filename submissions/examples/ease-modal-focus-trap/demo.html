Focus Trap Verification

Behavior Expected CSS :target
Tab cycles within modal Focus stays inside modal content ✅ Tab naturally wraps within modal's focusable elements
Escape closes modal Modal closes, focus returns to trigger ⚠️ :target deactivates via URL change; Escape requires JS focus() or a close link
Click outside closes Clicking backdrop dismisses modal ✅ Overlay wraps content; clicking overlay changes URL
Focus returns to trigger After close, focus goes back to open button ⚠️ Browser restores focus to last focused element (a[href="#demo-modal"])

How It Works

This modal uses the CSS :target pseudo-class, which activates when the URL hash matches the element's id.

/* Hide modal by default */
.modal-overlay {
  display: none;
}

/* Show when hash matches */
.modal-overlay:target {
  display: flex;
}

/* Close link deactivates target by changing hash */
<a href="#" class="modal-close">Close ✕</a>

The focus trap works naturally because the modal is the only thing visible and all focusable elements are within it. Browsers will cycle Tab through the modal's buttons. For a true focus trap that prevents Tab from reaching browser chrome, JavaScript is required (cycling focus through the first/last elements).

Escape: In a :target-based modal, pressing Escape doesn't automatically close it. A close link or JavaScript window.location.hash is needed. The open button retains focus after closing because the <a> element returns to the last focused location.
