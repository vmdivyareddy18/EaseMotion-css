Specificity Resolution via Cascade Layers

This page demonstrates how wrapping the framework in @layer solves specificity wars.

The Button Test

The button below has the framework class .ease-btn (which sets it to blue in the framework layer).

However, the developer's custom stylesheet targets a simple button tag (which sets it to red).

Because the developer's style is unlayered, it effortlessly overrides the framework's layered class, even though a class selector (.ease-btn) technically has higher specificity than a tag selector (button).
