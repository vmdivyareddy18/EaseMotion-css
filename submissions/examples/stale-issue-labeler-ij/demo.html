Stale Issue/PR Labeler

GitHub Actions workflow to automatically mark and close stale issues and PRs

This GitHub Actions workflow automatically labels issues and pull requests that have been inactive for 60 days with a "stale" label. If no activity occurs for another 14 days, the issue or PR is automatically closed. This helps keep the EaseMotion repository maintainable and focused.

name: Mark stale issues and PRs
on:
  schedule:
    - cron: "0 0 * * *"

jobs:
  stale:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/stale@v9
        with:
          repo-token: ${{ secrets.GITHUB_TOKEN }}
          stale-issue-message: >
            This issue has been automatically marked as stale because it has not
            had recent activity. It will be closed if no further activity occurs
            within 14 days.
          stale-pr-message: >
            This PR has been automatically marked as stale because it has not
            had recent activity. It will be closed if no further activity occurs
            within 14 days.
          days-before-stale: 60
          days-before-close: 14
          stale-issue-label: "stale"
          stale-pr-label: "stale"
          exempt-issue-labels: "pinned,security"
          exempt-pr-labels: "pinned,security"

How it works: The workflow runs daily via a cron schedule. It checks for issues and PRs with no activity in 60 days, labels them as "stale", and leaves a comment. If 14 more days pass without activity, they are automatically closed. Labels like "pinned" and "security" are exempt from this process.
