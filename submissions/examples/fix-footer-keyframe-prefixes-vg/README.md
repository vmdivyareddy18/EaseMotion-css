# Footer Keyframe Namespace Collision Showcase

1. What does this do? Demonstrates keyframe name collisions in the footer component with generic `float` and `heartbeat` names, and showcases how framework-prefixed names (`ease-kf-footer-float` and `ease-kf-footer-heartbeat`) isolate animations.
2. How is it used? It serves as a visual guide and regression test for the framework's namespace isolation patterns.
3. Why is it useful? It ensures EaseMotion components do not collide with consumer CSS or other libraries by maintaining a clean, prefixed global keyframe namespace.
