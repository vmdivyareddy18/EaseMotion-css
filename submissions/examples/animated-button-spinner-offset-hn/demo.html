Button Spinner Centering Fix

Issue Fixed: In the loading button component, the spinner pseudo-element (::after) is centered using the modern CSS property translate: -50% -50%. However, the rotation keyframes also applied transform: translate(-50%, -50%). Since these are separate CSS properties, both were being calculated, doubling the shift and dragging the spinner off-center.

We resolved this by removing the redundant translation rules from the keyframe definitions, letting the spinner rotate cleanly in place.
