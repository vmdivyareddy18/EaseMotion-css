Technical Deep-Dive

Underlying mechanics, cross-browser hanging fallbacks, and layout rules.

1. Target Elements

The text-indent property only applies to block-level containers (like <p>, <div>, or <li>) or elements styled with display: block or display: inline-block. It does not affect standard inline spans (like <span> or <a>) unless their display properties are modified.

2. Cross-Browser Hanging Fallback

Standard CSS specifies a `hanging` keyword: text-indent: 2rem hanging;. However, browser support for this keyword is inconsistent. To guarantee a perfect hanging indent across Chrome, Firefox, Safari, and Edge, the industry standard relies on a combined margin/padding trick: 

.indent-hanging-4 {
  padding-left: 2.5rem !important; /* Pushes the entire block right */
  text-indent: -2.5rem !important;  /* Pulls the first line back left */
}

This combination ensures the first line aligns perfectly with the left border, while all trailing lines indent by exactly 2.5rem.
