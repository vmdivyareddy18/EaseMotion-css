✧ @starting-style Animations

Entry animations for elements on first render — dialog, popover, and dynamic content

@starting-style lets you define the initial state before an element's first style update, enabling smooth entry animations for elements that are dynamically added, toggled via display, or shown as dialogs/popovers.

CSS Syntax

.entry-item {
  animation: fadeInUp 0.4s ease;

  @starting-style {
    opacity: 0;
    transform: translateY(20px);
  }
}

@keyframes fadeInUp {
  from {
    opacity: 0;
    transform: translateY(20px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}
