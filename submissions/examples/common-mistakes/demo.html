Common Mistakes with EaseMotion

Avoid these frequent anti‑patterns to keep your animations smooth and predictable.

❌ Animating left / top

Moving an element with left or margin-left forces layout recalculations, causing jank. Use transform: translateX() instead.

Bad Good

❌ Missing animation-fill-mode

Without forwards, an element snaps back to its original state after the animation.

Snaps back Stays in place

❌ Hard‑coding colors instead of using --ease-*

Directly overwriting component styles makes theming impossible. Use --ease-highlight-color to keep customisation safe.

Hard‑coded Themable

❌ Stacking ease-fade-in + ease-slide-up

Both set opacity and transform, causing one to override the other. Use a single combined animation class or custom keyframe.

Conflicting Single class
