Customization

Tune it with five CSS variables

Every modal is driven entirely by custom properties — override them globally on :root, or per-modal on the #modal-id selector, as shown in the variations above.

Variable Purpose
--modal-skew Starting skew angle the card animates from (e.g. -2deg).
--modal-duration Speed of the skew/fade/translate transition (e.g. 0.35s).
--modal-radius Corner rounding of the modal card.
--modal-shadow Elevation shadow applied to the modal card.
--modal-overlay Background color of the soft, blurred overlay backdrop.
/* Override per modal instance */ #custom-modal { --modal-skew: -2deg; --modal-duration: 0.35s; --modal-radius: 16px; --modal-shadow: 0 30px 60px rgba(15,23,42,.25); --modal-overlay: rgba(15,23,42,.5); }
Accessibility

Dashboards that work for everyone

SaaS admin portals are used all day, every day — accessibility here isn't optional. This component is built to stay usable for keyboard and screen-reader users.

⌨ Keyboard Accessible

Every trigger, form field, and close control is operable with Tab and Enter.

🔆 Visible Focus Indicators

A high-contrast focus ring appears on every interactive element via :focus-visible.

🏷 Semantic HTML

role="dialog", aria-modal, and aria-labelledby give assistive tech a clear landmark.

🎯 Large Touch Targets

Buttons, plan cards, and form controls meet a 44px+ minimum hit area.

🌓 High Color Contrast

Text and CTA colors are tuned to meet WCAG AA contrast against their backgrounds.

🧘 Reduced Motion

prefers-reduced-motion instantly disables skew, fade, and translate transitions.
