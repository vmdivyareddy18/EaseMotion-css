Hero That Fills the Screen

Scroll down — notice there's no awkward line, gap, or peek of the next section. The hero properly covers the full viewport height on both desktop and mobile.

See the Fix →

Buttons Section (cleanly below)

The Bug vs The Fix

❌ The Bug (Before)

.hero {
  height: 80vh;       /* doesn't fill viewport */
  padding-bottom: 0;  /* leaves a visible gap   */
}

/* Result:
   - Visible line/break before hero ends
   - Next section peeks above the fold
   - Mobile address bar causes overflow
*/

✅ The Fix (After)

.hero {
  min-height: 100vh;  /* full viewport fallback */
  min-height: 100dvh; /* mobile-safe dynamic    */
  display: flex;
  flex-direction: column;
  justify-content: center;
  align-items: center;
}

/* Result:
   - Hero fills the entire screen
   - Next section is cleanly below the fold
   - Works on iOS Safari, Chrome mobile, etc.
*/
