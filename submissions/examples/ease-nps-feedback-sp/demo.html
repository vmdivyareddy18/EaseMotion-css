Staggered score buttons

Each button uses ease-fade-in with ease-delay-* utilities for a cascading entrance, plus ease-hover-grow on hover.

Selected state

Active score gets .is-selected styling — primary fill, scale emphasis, and aria-checked="true" for screen readers.

Post-submit motion

Success checkmark uses ease-zoom-in; thank-you panel uses ease-slide-up + delayed ease-fade-in.

Copy-ready HTML snippet

<div class="nps-widget ease-card ease-card-shadow">
  <p class="nps-question">How likely are you to recommend us?</p>
  <div class="nps-scale-labels" aria-hidden="true">
    <span>Not at all likely</span>
    <span>Extremely likely</span>
  </div>
  <div class="nps-score-row" role="radiogroup" aria-label="Score 0 to 10">
    <button type="button" class="nps-score-btn ease-hover-grow ease-fade-in ease-delay-75"
            role="radio" data-score="0">0</button>
    <!-- …buttons 1–10 with increasing ease-delay-* … -->
  </div>
  <button type="button" class="nps-submit-btn ease-btn ease-btn-primary">
    Submit feedback
  </button>
</div>

<!-- Thank-you state (shown after submit) -->
<div class="nps-state-thanks">
  <h3 class="ease-fade-in ease-delay-150">Thank you!</h3>
  <p class="ease-fade-in ease-delay-300">Your feedback matters.</p>
</div>

Feedback widget guidance

Keyboard access

  • Arrow keys move between score buttons in the radiogroup.
  • Enter / Space selects the focused score.
  • Submit button activates with Enter when focused.

NPS vs CSAT labels

  • NPS: “Not at all likely” → “Extremely likely”.
  • CSAT: “Very dissatisfied” → “Very satisfied”.
  • Same 0–10 scale — swap copy for context.

Motion best practices

  • Keep submit success brief — avoid infinite loops.
  • Pair with prefers-reduced-motion resets in CSS.
  • Use finite iterations for attention animations.
