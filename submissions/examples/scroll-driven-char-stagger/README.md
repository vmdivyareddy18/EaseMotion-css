# Scroll-Driven Character Stagger

## What does this do?

Reveals each character of a headline individually as the user scrolls, driven entirely by `animation-timeline: scroll(root block)` — no JavaScript or Intersection Observer.

## How is it used?

```html
<h2 class="reveal-heading">
  <span class="char" style="--i: 1">H</span>
  <span class="char" style="--i: 2">e</span>
  <span class="char" style="--i: 3">l</span>
  <span class="char" style="--i: 4">l</span>
  <span class="char" style="--i: 5">o</span>
</h2>
```

Set `--i` on each `span.char` to control its scroll offset. In production, this is typically generated by a one-liner JS loop: `el.textContent.split('').forEach((c, i) => ...)`.

## Why is it useful?

Scroll-synced text reveals are among the most-searched CSS animation patterns. All existing scroll-driven submissions in EaseMotion animate block-level elements. This is the first to demonstrate character-level granularity using `animation-timeline: scroll()` — replacing a common JS-heavy Intersection Observer pattern with pure CSS. Includes `@supports not` fallback for unsupported browsers and full `prefers-reduced-motion` compliance. Chrome 115+, Firefox 114+ (flag).
