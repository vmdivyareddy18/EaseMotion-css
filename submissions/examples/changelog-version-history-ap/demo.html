Keep a Changelog Format Standards

        

          

            
              

                Category Header
                Content Type Guideline
                EaseMotion v1.0.0 Example
              

            
            
              

                ### Added
                For clean new features additions.
                Added entrance animations classes (e.g. .ease-fade-in).
              

              

                ### Changed
                For structural updates or deprecations.
                Refactored variable files to use nested CSS variables.
              

              

                ### Fixed
                For bugfixes and patches releases.
                Fixed z-index alignment conflicts on nested dropdown cards.
              

            
          

        

      


      
      
Automating Release Changelogs

      

        

          
1. Keep a Changelog Format

          

            Organize updates into logical categories. Avoid committing raw commit logs directly.
            Keep explanations readable for developers integration upgrades.
          

        


        

          
2. GitHub Actions Integration

          

            Configure an auto-release action trigger.
            Automatically parse PR titles and tags, generating release drafts appended to CHANGELOG.md on version releases.
