Default Scrollbar .scrollbar-auto

Standard Browser Scrollbar: This container is styled with the .scrollbar-auto utility class. It utilizes the default scrollbar width, which varies by operating system and device.

On desktop Windows systems, this is typically 12px wide. On macOS, scrollbars are usually overlay-based and thin by default.

Scrollbar width controls are crucial when dealing with sidebars, modal content, and code block containers where standard scrollbars might look too bulky.

Try scrolling in this card to view the default behavior. You can use standard mouse wheel, trackpad gestures, or drag the scroll track directly.

Thin Scrollbar .scrollbar-thin

Compact scrollbar style: This container uses .scrollbar-thin. It forces a compact, 6px wide scrollbar designed for elegant dashboard panels, popovers, and side sheets.

Firefox renders this natively using the W3C CSS Scrollbars Module Level 1 standard property scrollbar-width: thin;.

For Webkit/Blink browsers (Chrome, Edge, Safari), it applies custom styles via pseudo-elements to force a match.

Hovering over the thumb will change its background color to the brand primary teal accent, offering direct visual feedback to the user.

Hidden Scrollbar .scrollbar-none

Invisible scroll track: This container uses .scrollbar-none. The scrollbar is entirely hidden, but the viewport remains fully scrollable by trackpad, mouse wheel, or touch swiping.

This is commonly used in minimal card lists, full-screen slider carousels, mobile views, or distraction-free layouts.

Be careful when using this class on non-touch desktop screens, as users may not realize a container is scrollable if there are no visual indicators or shadows.

Technical Deep-Dive

Learn about browser specifications, layout shifts, and scroll optimization tips.

1. Cross-Browser Specifications

Currently, scrollbars are styled differently depending on the layout engine:

  • Gecko (Firefox): Uses standard CSS scrollbar properties. scrollbar-width: auto | thin | none controls the width, and scrollbar-color: <thumb-color> <track-color> controls the colors.
  • WebKit/Blink (Chrome, Edge, Safari): Relies on vendor-prefixed pseudo-elements. The width is controlled by setting width (for vertical) or height (for horizontal) on the ::-webkit-scrollbar selector. Colors are configured on ::-webkit-scrollbar-thumb and ::-webkit-scrollbar-track.

2. Layout Shifts & scrollbar-gutter

A common issue with dynamically showing or hiding scrollbars is Layout Shift. When a page gets enough content to overflow, the scrollbar appears, pushing content to the left.

To avoid this, you can combine these utilities with the scrollbar-gutter: stable; property, which reserves space for the scrollbar beforehand.
