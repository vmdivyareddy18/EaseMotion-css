# Business Landing — API Gateway Infrastructure SaaS (`saas-landing-api-gateway-21723`)

This proposal provides a complete, copy-paste ready B2B SaaS landing page template specifically designed for developer infrastructure, API gateways, and edge computing platforms (like Cloudflare, Kong, or Vercel), using pure HTML/CSS and EaseMotion utility classes.

## 📌 Feature Overview

An infrastructure SaaS landing page needs to project performance, security, and developer experience. This template uses a high-contrast, developer-centric aesthetic with monospace typography for technical data and a minimalist, robust color palette.

The layout includes:
- A hero section featuring an animated CSS-built API console/terminal mockup showing real-time metrics and logs
- A logo banner for technical engineering customer teams
- A dark-mode code snippet section demonstrating YAML configuration with file tabs
- A 6-card platform feature grid focusing on infrastructure use cases (Rate Limiting, Global Caching, Wasm)
- A request-volume based tiered pricing layout
- A dark, high-contrast footer CTA

### Showcased Classes
The template leverages the following EaseMotion animation and interaction utilities:
- `.ease-fade-in` / `.ease-fade-up` (For smooth text entrances and cascading log lines in the terminal mockup)
- `.ease-slide-up` (For structural layout animations in pricing and features, as well as console stats)
- `.ease-zoom-in` (For drawing attention to the primary terminal mockup and code editor window)
- `.ease-bounce-in` (For primary deployment CTAs)
- `.ease-hover-lift` / `.ease-hover-scale` / `.ease-hover-zoom` (For interactive buttons, feature cards, and mockups)

## ⚙️ How to Use

To view this full-page template locally, simply open the `demo.html` file in your web browser. 

The CSS handles the structural layout, styling, and syntax highlighting colors, while standard `ease-*` interaction classes are attached directly to HTML elements to provide the animations without needing custom JavaScript or specific CSS keyframes.

*Note: As per the contributing guidelines, this proposal is implemented inside a unique `submissions/examples/saas-landing-api-gateway-21723/` directory to avoid modifying core files directly and to ensure zero deletions.*
