mask vs clip-path

Side-by-side comparison of CSS mask and clip-path — two ways to hide/reveal parts of an element.

clip-path

Crops the element to a defined shape. The element is literally cut — clicks, layout, and events only work inside the visible area.

circle()
polygon()
ellipse()
inset()

mask

Uses an image (or gradient) as an alpha mask. Pixels remain in full (events and layout unaffected) — only visibility is controlled.

linear-gradient()
radial-gradient()
fade edges
luminance

Key Differences

Featureclip-pathmask
Trims boxYes — layout unaffectedNo — full box remains
TransitionsAnimated (same shape type)Animated via mask-image
Events outside shapeBlockedStill received
Gradient supportNoYes
PerformanceGPU-acceleratedGPU-accelerated

Use clip-path when you want to visually crop an element. Use mask when you need smooth fades or gradient-based transparency.
