Animation Delay Specificity

Understand why animation delays fail when combined with component shorthand classes and how to enforce delay rules.

⚠️ Overwritten Delays (Bug)

If a component class (such as buttons or cards) is declared after utility files in the CSS cascade, its shorthand animation property overrides the utility delay. In Firefox 106+, this shorthand reset renders all elements simultaneously with no delay:

🛠️ Specificity Override Solution

To prevent custom components from resetting delays, declare your delay utility helpers with !important values to guarantee specificity priority:

/* Enforces delay rules over shorthand property resets */
.ease-delay-100 { animation-delay: 100ms !important; }
.ease-delay-200 { animation-delay: 200ms !important; }
.ease-delay-300 { animation-delay: 300ms !important; }
.ease-delay-500 { animation-delay: 500ms !important; }
