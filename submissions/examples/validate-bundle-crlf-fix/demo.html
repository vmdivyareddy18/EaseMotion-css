CRLF Line Ending Fix — Issue #6694

❌ Before Fix Windows · core.autocrlf=true

Raw string comparison fails because Git converts LF → CRLF on checkout, but the minifier always outputs LF.

// ❌ Fails on Windows — \r\n vs \n mismatch
if (originalBundle !== rebuiltBundle) {
  console.error('❌ Bundle is stale.');
  process.exit(1);
}
$ npm run validate:bundle
❌ Bundle is stale. Run npm run build first.
(but bundle IS up to date — false positive)
✅ After Fix Windows · Linux · macOS

Normalize both strings to LF before comparing — OS-agnostic, no false failures.

// ✅ Normalize \r\n → \n before comparing
const normalize = (str) => str.replace(/\r\n/g, '\n');

if (normalize(originalBundle) !== normalize(rebuiltBundle)) {
  console.error('❌ Bundle is stale. Run npm run build first.');
  process.exit(1);
}
$ npm run validate:bundle
✅ Bundle is up to date (line endings normalised for cross-platform).
