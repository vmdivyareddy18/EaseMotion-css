:user-valid & :user-invalid

Form validation that activates after user interaction — not immediately on page load.

Comparison

:valid / :invalid

Shows feedback immediately — fields may appear invalid before the user interacts.

Required, must be a valid email
Required, must be a valid URL
Must be between 1 and 100

:user-valid / :user-invalid

Shows feedback only after the user has interacted with each field.

Required, must be a valid email
Required, must be a valid URL
Must be between 1 and 100

Key Insight

On page load, the :invalid form shows all fields with error styling immediately. The :user-invalid form shows no errors until you type in a field — providing a calmer initial experience.

Tab into a field, type something, then clear it to trigger :user-invalid. Compare with :invalid which fires immediately.
