font-display: swap — FOIT Fix

Fixes #4255 — adds &display=swap to the Google Fonts link in README and README.bn.md.

Quick Start Snippet

❌ Before — causes FOIT on slow connections:

<link
  href="https://fonts.googleapis.com/css2?family=Inter:wght@300;400;500;600;700"
  rel="stylesheet"
/>

✅ After — FOIT-safe with preconnect hints:

<link rel="preconnect" href="https://fonts.googleapis.com" />
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
<link
  href="https://fonts.googleapis.com/css2?family=Inter:wght@300;400;500;600;700&display=swap"
  rel="stylesheet"
/>

Comparison

Scenario Without display=swap With display=swap
Fast connection No visible difference No visible difference
Slow / mobile Text invisible up to 3s (FOIT) Fallback font shown instantly
Font fails to load Text stays invisible Fallback font used permanently

Why display=swap matters: Google Fonts injects a @font-face rule with font-display: swap when the parameter is present. Without it, font-display: block is used — the browser hides text for up to 3 seconds waiting for the font to download. With swap, a system fallback renders immediately and Inter swaps in once ready.

The two rel="preconnect" hints also reduce DNS + TLS handshake time for Google Fonts domains, further improving load speed on all connection types.
