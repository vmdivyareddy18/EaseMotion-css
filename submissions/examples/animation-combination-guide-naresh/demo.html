Combining CSS Animations

Understand the CSS animation overwrite limitation and learn how to combine EaseMotion animation classes effectively.

⚠️ The CSS Overwrite Limitation

When applying two shorthand animation classes (like .ease-fade-in and .ease-slide-up) to the same element, only the last declared class will play. This is because the CSS cascade replaces the animation property with the new value rather than combining them.

Overwritten (Incorrect)

Applied together on one element:
class="anim-fade-in anim-slide-up"

❌

🧩 Workaround 1: Nested HTML Wrappers

The cleanest work-around that requires no custom CSS is nesting HTML elements. Apply the fade animation to the parent element, and the slide/position animation to the child element.

Resolved (Nesting)

Parent has fade, child has slide:
<div class="parent"> <div class="child">

<!-- Applying multiple animations via nested wrappers -->
<div class="ease-fade-in">
  <div class="ease-slide-up">
    <!-- Your content here -->
  </div>
</div>

✒️ Workaround 2: Custom Merged Keyframes

If you prefer to keep your HTML flat, you can write a short custom CSS class that merges the keyframe properties of both animations into a single timeline.

Resolved (Combined Keyframes)

Merged animation class:
class="ease-fade-slide-up"

✅
/* Define custom keyframes that combine both properties */
@keyframes easeFadeSlideUpCombined {
  from {
    opacity: 0;
    transform: translateY(30px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

/* Apply the custom combined animation class */
.ease-fade-slide-up {
  animation: easeFadeSlideUpCombined 1s cubic-bezier(0.4, 0, 0.2, 1) forwards;
}
