@starting-style Entry Animations

Animate elements when they first appear in the DOM — dialogs, toasts, tooltips, popovers. Chrome 117+

⚡ Requires Chrome 117+. The demo still works in other browsers (no entry animation — element appears instantly).

Without @starting-style

Modal appears instantly — no entry animation on first render.

Plain Modal

This modal has transition on opacity and transform, but when first opened via display: flex, the browser cannot interpolate from the initial state.

The result: no entry animation — it just pops in.

With @starting-style

Modal fades and scales in smoothly from the first render.

Starting-Style Modal

This modal uses @starting-style to define the initial state before the element renders.

The browser knows the starting values: opacity: 0, transform: scale(0.95) — so the transition animates correctly on first entry.

Toast with @starting-style

Each toast animates in when created (use the button to spawn new toasts).

Implementation

Modal with @starting-style
.modal-overlay {
  display: none;
  opacity: 0;
  transition: opacity 0.3s, transform 0.3s;
}

.modal-overlay.active {
  display: flex;
  opacity: 1;
}

/* Defines the state BEFORE first render */
@starting-style {
  .modal-overlay.active {
    opacity: 0;
    transform: scale(0.95);
  }
}
@supports guard for progressive enhancement
@supports (at-rule(@starting-style)) {
  @starting-style {
    .ease-modal-active {
      opacity: 0;
      transform: scale(0.95);
    }
    .ease-toast-enter {
      opacity: 0;
      transform: translateX(100%);
    }
  }
}
