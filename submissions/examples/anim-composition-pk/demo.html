animation-composition Utilities

Control how multiple animations stacking on the same property combine — replace (default), add, or accumulate. Chrome 115+

⚡ Requires Chrome 115+. In unsupported browsers, all three behave as replace.

replace (default)

Last animation wins — rotate replaces zoom-in.

replace
ease-kf-zoom-in + ease-kf-rotate

add

Transforms are appended — rotate adds to zoom-in.

add
ease-kf-zoom-in + ease-kf-rotate

accumulate

Values are added together — 0.5 + 0.5 = 1.0 scale, 0 + 360 = 360deg.

accumulate
ease-kf-zoom-in + ease-kf-rotate

How It Works

When two animations target the same property (e.g., transform):

ModeBehaviorExample
replace Last declared animation's value wins rotate(360deg) replaces scale(0→1) → only rotation visible
add Transform functions are appended in order scale(0→1) then rotate(360deg) → both visible
accumulate Numeric values are summed scale(0.5) + scale(0.5) = scale(1.0)

Proposed Utility Classes

In core/utilities.css:

.ease-compose-replace {
  animation-composition: replace;
}
.ease-compose-add {
  animation-composition: add;
}
.ease-compose-accumulate {
  animation-composition: accumulate;
}
