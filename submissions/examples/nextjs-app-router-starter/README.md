# Next.js App Router Starter

## Overview

This example demonstrates a lightweight integration of **EaseMotion CSS** with the **Next.js App Router** architecture. It serves as a reference for structuring a modern Next.js project while using EaseMotion CSS animation utilities in a clean and maintainable way.

## Project Structure

The example is designed around the standard App Router layout:

```text
app/
├── layout.tsx
├── page.tsx
└── globals.css
```

- `app/layout.tsx` acts as the application's root layout and global entry point.
- `app/page.tsx` demonstrates how EaseMotion CSS utility classes can be applied to page content.
- `globals.css` (or the project's global stylesheet) is where EaseMotion CSS can be imported for application-wide usage.

## Usage

Import EaseMotion CSS into your global stylesheet or root layout, then use the framework's existing utility classes directly inside your App Router components.

This example focuses on the recommended integration pattern rather than introducing new framework features or custom utilities.

## Why is this useful?

- Demonstrates a clean integration with the Next.js App Router.
- Provides a simple reference structure for new projects.
- Encourages consistent use of existing EaseMotion CSS utilities.
- Aligns with EaseMotion CSS's animation-first and human-readable philosophy.
- Serves as a lightweight starter reference for future Next.js integrations.

## Notes

This submission is intended as an integration example and uses existing EaseMotion CSS utilities without introducing new components or framework APIs.
