Contain Values Compared

Each card uses a different contain value. Open DevTools to inspect containment behavior.

contain: paint

paint

Clips overflow without requiring overflow: hidden. The browser knows no child can paint outside this box, enabling paint-time optimizations.

→ This content is wider than its parent, but contained by paint.
contain: paint

contain: layout

layout

Creates a new formatting context. Floats, absolute positioning, and margin collapsing are scoped inside this element.

float

This float is contained — it won't escape into neighboring cards.

contain: layout

contain: size

size

The element's size is determined solely by itself, not its children. This allows the browser to skip layout calculations when children change.

Resize this card's content — the container dimensions remain stable.

contain: size

contain: strict

strict

Shorthand for contain: size layout paint. Maximum isolation — the element and its subtree are fully independent from the rest of the page.

Use for truly self-contained components like modals, tooltips, and embeds.

contain: strict ≡ size layout paint

contain: content

content

Shorthand for contain: layout paint. Excludes size containment for elements whose size depends on content — the most practical default.

Safe to add to most components without breaking layouts.

contain: content ≡ layout paint

Overflow Clipping Without overflow: hidden

contain: paint clips overflow while keeping scroll behavior usable.

overflow

With contain: paint, the browser can optimize painting because it knows absolutely nothing will paint outside this element's bounds. Unlike overflow: hidden, it does not change scroll behavior or create a scroll container — it only clips at the paint level.

Containment Reference

  • contain: paint — Clip to box, no descendant paints outside
  • contain: layout — Internal layout isolated, no external effects
  • contain: size — Size independent of child content
  • contain: style — Scoped counters and quotes (limited support)
  • contain: strict — All containment types active
  • contain: content — All except size containment
