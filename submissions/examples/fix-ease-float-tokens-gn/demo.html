Fix #5757: .ease-float ignores design tokens

.ease-float currently hardcodes 3s ease-in-out infinite, so it can't be customized via --ease-speed-*, --ease-ease, or --ease-animation-iterations like sibling classes (e.g. .ease-bounce).

Before (hardcoded)

3s, fixed

After (token-driven, default)

var(--ease-speed-slow, 3s)

After + custom override (1s)

--ease-speed-slow: 1s

Proposed core/animations.css diff

.ease-float {
-  animation: ease-float 3s ease-in-out infinite;
+  animation: ease-float
+    var(--ease-speed-slow, 3s)
+    var(--ease-ease, ease-in-out)
+    var(--ease-animation-iterations, infinite);
}
