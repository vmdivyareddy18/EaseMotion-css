🐛 Button Hover States Bug Fix

EaseMotion CSS Issue #44507 - Low Severity

❌ Current Behavior (Broken)

💡 Test Now: Hover your mouse over the buttons below. Notice they do NOT change color on hover—the background stays static.
Primary (Works ✓)
Success (Broken ✗)
Danger (Broken ✗)
Warning (Broken ✗)
Root Cause: In core/variables.css, the base and -dark color tokens are identical:
  • --ease-color-success = --ease-color-success-dark = #15803d
  • --ease-color-danger = --ease-color-danger-dark = #b91c1c
  • --ease-color-warning = --ease-color-warning-dark = #b45309
Since components/buttons.css uses the -dark variant for :hover, the color never changes.

✅ Fixed Behavior (Using Custom Override)

💡 Test Now: Hover your mouse over the buttons below. Notice they smoothly transition to a darker shade on hover.
Primary (Reference)
Success (Fixed ✓)
Danger (Fixed ✓)
Warning (Fixed ✓)
Solution: Update the -dark color tokens in core/variables.css to properly darker shades:
  • --ease-color-success-dark: #105e2d (was #15803d)
  • --ease-color-danger-dark: #8b1515 (was #b91c1c)
  • --ease-color-warning-dark: #873e07 (was #b45309)

📋 Summary

This demonstration shows that the Success, Danger, and Warning button variants have non-functional hover states because their -dark color tokens are identical to their base values. The fix requires updating three color variables in core/variables.css to darker, properly contrasting shades. The corrected values follow the same luminosity reduction pattern as the already-functional Primary button variant (Primary: #6c63ff → Primary-Dark: #4b44cc).
