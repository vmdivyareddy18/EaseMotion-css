Bounce Retina Text-Blur Fix

On high-DPI (retina) screens, CSS translateY animations promote elements to GPU compositor layers mid-animation. If the layer is created at that moment the browser re-rasterises text at the wrong pixel density, producing blurry, pixelated glyphs. This fix promotes the element to a GPU layer before the first animation frame so rasterisation always happens at full retina resolution.

Property What it does Why it fixes blur
translateZ(0) Forces GPU compositor layer creation Layer created at full retina DPI before first frame — no mid-animation re-rasterisation
backface-visibility: hidden Locks rasterisation mode Prevents the browser switching CPU↔GPU paths during transform changes
-webkit-font-smoothing: antialiased Selects greyscale antialiasing Subpixel antialiasing is disabled on composited layers; this picks the correct fallback path
-moz-osx-font-smoothing: grayscale Same as above for Firefox/macOS Ensures parity on Firefox on macOS retina screens
will-change: transform Signals animation intent to browser GPU texture allocated before first frame, eliminating first-frame rasterise cost
translateZ(0) in keyframes Keeps GPU layer alive at every stop Without this the compositor may drop the layer between keyframes and re-create it on the wrong path

🎨 The CSS Fix

Apply to the bouncing element. All properties work together; removing any one can re-introduce blur on specific browsers/OS combinations.

/* Retina-safe bounce class */
.bounce-crisp {
  /* ① GPU layer at full retina DPI from the start */
  -webkit-transform: translateZ(0);
  transform: translateZ(0);

  /* ② Lock rasterisation mode through the animation */
  -webkit-backface-visibility: hidden;
  backface-visibility: hidden;

  /* ③ Greyscale antialiasing (safe on composited layers) */
  -webkit-font-smoothing: antialiased;
  -moz-osx-font-smoothing: grayscale;

  /* ④ Pre-allocate GPU texture before first frame */
  will-change: transform;

  animation: bounce-crisp 1.8s cubic-bezier(0.8,0,1,1) infinite;
}

@keyframes bounce-crisp {
  0%, 100% {
    /* Keep translateZ(0) at every stop */
    transform: translateY(0) translateZ(0);
    animation-timing-function: cubic-bezier(0.8, 0, 1, 1);
  }
  50% {
    transform: translateY(-22px) translateZ(0);
    animation-timing-function: cubic-bezier(0, 0, 0.2, 1);
  }
}

📐 HTML

One class applied to the bouncing wrapper. Children (text, images, etc.) inherit the crisp rendering automatically.

<div class="bounce-crisp">
  <p>Sharp text during bounce!</p>
</div>

♿ Reduced-motion

The animation is stopped entirely for users who prefer reduced motion — bouncing at any speed can be uncomfortable for people with vestibular disorders.

@media (prefers-reduced-motion: reduce) {
  .bounce-crisp {
    animation: none !important;
    transform: translateZ(0) !important; /* preserve GPU layer */
    will-change: auto;
  }
}
