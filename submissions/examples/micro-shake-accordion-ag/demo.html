CSS Micro Shake Accordion

Pure CSS · Semantic HTML · Keyboard accessible · No JavaScript

What is the micro-shake animation? How it works under the hood

When an accordion item is opened, the icon plays a small shake animation to provide a subtle visual cue. The effect is lightweight and keeps the interaction feeling smooth.

Everything is implemented using semantic HTML and CSS, without relying on JavaScript.

How is keyboard accessibility handled? Tab, Space, Enter — all work natively

The <details> and <summary> elements carry built-in keyboard semantics. Browsers expose summary as a button in the accessibility tree — Tab focuses it, and Enter or Space toggles the panel.

Focus styles are provided via :focus-visible so the ring appears only for keyboard users, not on mouse click.

Why use details/summary over radio inputs? Native semantics vs. the CSS state hack

The hidden-radio technique works but repurposes form controls for a non-form interaction. <details> is the HTML element designed exactly for this: a disclosure widget. Screen readers announce it correctly out of the box.

Adding the name attribute to each <details> creates a mutually exclusive group — only one panel stays open at a time, mirroring the radio-group behaviour without any JavaScript.

What customization options are available? CSS custom properties you can override

Override any of these custom properties on :root or a wrapper class to adapt the component to your design system:

--shake-duration, --shake-angle, --expand-duration, --color-accent, --color-bg, --card-radius.

The palette ships in teal by default — swapping --color-accent and its dependants is enough to match any brand color.
