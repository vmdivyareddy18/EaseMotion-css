Visual Regression Testing Setup

Instructions and examples for integrating visual regression testing with EaseMotion CSS.

Why Visual Regression?

When animations or styles change unexpectedly, visual regression tests catch the difference before merging. This tooling helps you set up a basic screenshot comparison using Puppeteer or integrate with Percy/Chromatic.

Method 1: Puppeteer Screenshot Capture

// capture.js
const puppeteer = require('puppeteer');
(async () => {
  const browser = await puppeteer.launch();
  const page = await browser.newPage();
  await page.goto('http://localhost:3000/demo.html');
  await page.screenshot({ path: 'screenshots/home.png', fullPage: true });
  await browser.close();
})();

Run this script before and after changes, then diff the images.

Method 2: Percy Integration (GitHub Actions)

# .github/workflows/percy.yml
name: Percy Visual Tests
on: [pull_request]
jobs:
  percy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
      - run: npm install
      - run: npx percy snapshot submissions/examples/

Percy will compare snapshots and report visual diffs in the PR.

All scripts are self‑contained examples; adapt them to your CI pipeline.
