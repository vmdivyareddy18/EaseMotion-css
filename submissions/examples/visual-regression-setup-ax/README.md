# Visual Regression Testing Setup

A collection of instructions and code snippets for integrating visual regression testing into the EaseMotion CSS workflow. Helps catch unintended animation or layout changes automatically.

## EaseMotion classes used in demo
- **Layout:** ease-container, ease-flex, ease-items-center, ease-justify-center, ease-min-h-screen, ease-mx-auto, ease-py-16
- **Background:** ease-bg-gray-50, ease-bg-white
- **Typography:** ease-text-4xl, ease-font-bold, ease-text-lg, ease-text-gray-500, ease-text-2xl, ease-font-semibold, ease-text-gray-600, ease-text-sm, ease-text-gray-400
- **Spacing:** ease-mb-4, ease-mb-8, ease-mt-4, ease-mt-12, ease-p-6
- **Components:** ease-card
- **Animation:** ease-fade-in, ease-delay-200, ease-delay-500

## Contents
- Puppeteer screenshot capture script
- GitHub Actions workflow example with Percy
- Explanation of visual regression workflow

## How to use
1. Copy the relevant snippets into your project.
2. Install dependencies (puppeteer, percy) if needed.
3. Run the scripts locally or configure them in your CI pipeline.
