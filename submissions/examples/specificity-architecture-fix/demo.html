CSS Architecture Fix: Specificity Wars

This submission provides the architectural requirement to ban the !important flag, removing a catastrophic modularity blocker present across nearly 900 components.

Incorrect Architecture

The framework forces !important on the button background. As an end-user, my custom class .btn-broken-override cannot change the color to red, completely breaking my ability to theme the framework.

Correct Architecture

The framework uses CSS @layer or simple specificity. As an end-user, my custom class .btn-fixed-override smoothly overrides the default blue color to purple without resorting to specificity hacks.
