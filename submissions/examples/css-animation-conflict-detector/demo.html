💡 Under the Hood

Why do Conflicts Occur?

In standard CSS, only one animation can control a specific property (like transform) at a time on an element.

  • When multiple animations modify transform, the one declared last in the animation rule overrides previous ones.
  • This prevents animations (e.g. Spin + Pulse) from running concurrently, causing the overridden animation to be ignored.

Compatible vs. Resolved

Compatible: Animations affecting different properties (e.g. transform and opacity) run concurrently without conflict.

Resolved: Modern CSS introduces animation-composition: add which forces the browser to combine (multiply/add) transforms rather than replace them, enabling composite effects.
