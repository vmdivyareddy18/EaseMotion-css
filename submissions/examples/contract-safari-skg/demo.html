Minimal reproduction for the Contract animation stutter reported on Safari. Use the controls below to trigger each variant and compare behaviour across browsers.

ease-contract-bg-exit

Scales from 1 → 0.85 and fades out. Uses transform: scale() + opacity. On Safari, simultaneous opacity + scale can trigger a compositing layer split, causing a visible stutter at the start of the animation.

📦 Baseline No Safari fix
📦 GPU hint translateZ(0)
📦 will-change transform, opacity
📦 backface-visibility hidden

ease-contract-image-entrance

Scales from 1.25 → 1 and fades in. The scale-down entrance is where Safari may briefly pause ("freeze frame") at the start before the keyframe progresses, particularly when the element contains sub-pixel content or is nested in a flex container.

🖼️ Baseline No Safari fix
🖼️ GPU hint translateZ(0)
🖼️ will-change transform, opacity
🖼️ isolation isolate

ease-contract-shadow-emphasis

Pulses scale 1 → 0.96 → 1 while also animating box-shadow. Animating box-shadow is the highest-risk property for Safari stutter because it forces a repaint on every frame. This is the most likely source of the reported frame drops.

✨ Baseline box-shadow animated
✨ GPU hint translateZ(0)
✨ Workaround filter: drop-shadow()
🔎
Finding: The "Workaround" card replaces box-shadow with filter: drop-shadow(), which is GPU-composited on Safari and avoids the repaint path. Compare smoothness with the baseline on Safari.
