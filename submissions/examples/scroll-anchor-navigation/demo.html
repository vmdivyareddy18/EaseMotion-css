Smooth anchor navigation with sticky header offsets

Modern documentation layouts rely on sticky headers and in-page anchor links. Without offset handling, section titles scroll beneath the header and break the reading experience. This demo shows how scroll-behavior: smooth and scroll-margin-top work together for polished navigation.

scroll-behavior

Animates movement between anchor targets instead of instant jumps.

scroll-margin-top

Reserves space above each target so content clears sticky chrome.

:target styling

Highlights the active section and sidebar link without JavaScript.

Smooth scrolling behavior

Setting scroll-behavior: smooth on the root element (or a scroll container) tells the browser to animate scroll position changes — including anchor navigation triggered by link clicks and URL hashes.

CSS 
html {
  scroll-behavior: smooth;
}

Use the sidebar or header links on this page to feel the difference. Smooth scrolling improves perceived quality in long documentation pages without any JavaScript scroll libraries.

Accessibility note

Respect prefers-reduced-motion: reduce by resetting scroll-behavior: auto so users who disable motion still get instant, predictable jumps.

scroll-margin-top in practice

Every major section on this page uses scroll-margin-top: var(--san-header-h) so in-page links account for the sticky site header. Match the margin to your actual header height — fixed, sticky, or dynamic.

CSS 
:root {
  --san-header-h: 72px;
}

.san-section {
  scroll-margin-top: var(--san-header-h);
}

Unlike padding, scroll margin does not affect layout flow. It only adjusts where the browser considers the "scrolled into view" position for anchor targets.

Alternative: scroll-padding-top

Instead of per-section margins, you can apply scroll-padding-top on the scrolling element (usually html) to offset all anchor targets globally. The mini-panel on the right also demonstrates this on its viewport.

CSS 
html {
  scroll-padding-top: var(--san-header-h);
}

/* Or on a nested scroll container */
.docs-viewport {
  scroll-padding-top: 56px;
}

Implementation checklist

  1. Measure sticky header height and store it in a CSS variable.
  2. Enable scroll-behavior: smooth on the scroll root.
  3. Apply scroll-margin-top to section anchors or scroll-padding-top on the scroller.
  4. Style :target sections and use :has() to highlight matching nav links.
  5. Reset smooth scrolling when reduced motion is preferred.

Guide · Design tokens

Centralize header height, sidebar width, and scroll offsets in custom properties so sticky components and scroll margins stay in sync when the layout changes across breakpoints.

Provide skip links, maintain visible focus rings on anchor controls, and never rely on motion alone to communicate state. Keyboard users benefit equally from correct scroll offsets.
