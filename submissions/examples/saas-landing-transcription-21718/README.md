# Audio/Video Transcription SaaS Landing (`saas-landing-transcription-21718`)

This proposal provides a complete, copy-paste ready B2B SaaS landing page template designed specifically for AI Transcription software (similar to Otter.ai or Rev), utilizing pure HTML/CSS and standard EaseMotion utility classes.

## 📌 Feature Overview

A modern AI transcription tool needs to visually communicate speed, accuracy, and audio processing capabilities. This template achieves an AI-focused aesthetic using a clean indigo color palette, crisp sans-serif typography, and custom CSS animations that simulate audio waveforms and real-time text highlighting.

The layout includes:
- A hero section featuring an animated CSS-built transcription player mockup with moving audio waves and staggered word highlighting.
- A supported formats logo strip.
- A high-impact statistics section with simulated data loading.
- A 4-item feature grid with subtle rotation hover states.
- A clear, per-minute pricing structure with an integrated enterprise upsell.
- Trust-building customer testimonials and a comprehensive footer.

### Showcased Classes
The template leverages the following EaseMotion animation and interaction utilities:
- `.ease-wave` (Custom keyframe simulating audio equalizer bars in the player mockup)
- `.ease-highlight` (Applied sequentially to words in the transcript mockup to simulate real-time AI transcription reading)
- `.ease-pulse` (Used on the play button to draw attention)
- `.ease-slide-up` / `.ease-fade-up` (For smooth, staggered text and UI entrances across the page)
- `.ease-bounce-in` / `.ease-zoom-in` (Used on decorative floating badges and mockups to create depth)
- `.ease-count-up` (Class placeholder for numerical stat animation entrances)
- `.ease-hover-lift` (Standard interaction for primary buttons and cards)

## ⚙️ How to Use

To view this full-page template locally, simply open the `demo.html` file in your web browser. 

The CSS handles the layout, typography, and styling, while standard `ease-*` interaction classes are attached directly to HTML elements to provide the animations. The audio player mockup, complete with moving waveforms and sequentially highlighted transcript text, is built entirely with structural HTML and CSS, requiring zero JavaScript.
