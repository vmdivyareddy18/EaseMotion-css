content-visibility: auto

Skip rendering off-screen elements. Improves FCP, LCP, and scroll performance on pages with many animated components. Chrome 85+

⚡ Requires Chrome 85+. Unsupported browsers ignore the property — content remains visible and functional.

Without content-visibility

All 200 cards render immediately — even off-screen. Higher paint cost, lower FPS.

📊 Render time: —

With content-visibility

Off-screen cards are skipped. Browser only renders visible ones. ~10x faster initial paint.

📊 Render time: —

What It Does

  • ✅ Skips layout, paint, and compositing for off-screen elements
  • ✅ Reduces FCP (First Contentful Paint) significantly
  • ✅ Improves scroll performance on long pages
  • ✅ Preserves element dimensions via contain-intrinsic-size
  • ⚠️ Animations on off-screen elements don't tick until scrolled into view

Best For

  • 📋 Long lists of animated cards
  • 🖼️ Image galleries with lazy animations
  • 📄 Blog / feed pages with entries below the fold
  • 📊 Dashboard grids with many widgets
  • 🎬 Components whose entry animation triggers on scroll (via view-timeline)

Proposed Utility

core/utilities.css
.ease-content-visibility-auto {
  content-visibility: auto;
  contain-intrinsic-size: 0 500px;
}

Usage with Animated Components

<div class="ease-content-visibility-auto">
  <div class="ease-card ease-anim-fade-in">
    Only rendered when visible
  </div>
</div>

/* For animation cards in a grid */
.ease-card-grid > * {
  content-visibility: auto;
  contain-intrinsic-size: 0 240px;
}

Caveats

  • Animations on off-screen elements are paused — the browser stops ticking animation frames until the element scrolls into view. This is usually fine (why animate something no one sees?).
  • Set contain-intrinsic-size — without it, the browser reserves 0px height, causing scrollbar jumps when elements render. Approximate the height of your component.
  • Not a replacement for lazy loading — images inside still need loading="lazy".
