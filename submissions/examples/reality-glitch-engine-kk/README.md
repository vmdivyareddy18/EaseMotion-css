# Reality Glitch Engine 🌀

An immersive, pure-CSS cinematic "system failure" visual experience for EaseMotion CSS.

## 🌟 What does this showcase?
The Reality Glitch Engine demonstrates the absolute boundaries of pure CSS animations. It creates a terrifyingly beautiful digital collapse, fracturing the DOM into multiple dimension layers without a single line of JavaScript.

## 🚀 How is it used?
Simply open `demo.html` in any modern web browser. 
Hover your cursor over the center panel to trigger the **Reality Repair Sequence**—watch the fractured DOM reconstruct itself instantly.

## 💡 Why is it useful?
This showcase highlights advanced CSS techniques including:
- Complex `clip-path` fragmentation
- Chained `backdrop-filter` glassmorphism
- Advanced pseudo-element glitching
- CSS `:has()` selector for state management without JS
- Custom timeline pacing with `@keyframes` "time-freezing"

## ✨ Features
- **100% Pure CSS**: Zero JavaScript, no SVG graphics (pure DOM/CSS), no external assets.
- **6-Layer Parallax**: Depth achieved through layered compositing.
- **23 Custom `@keyframes`**: Ranging from macro-fractures to micro-pixel dissolution.
- **Hover-Triggered State Reversal**: The environment completely repairs itself instantly upon interaction.

## 🎬 Animation Breakdown
1. **REALITY FRACTURE**: 9 unique glass polygon fragments drifting apart.
2. **GLITCH DISTORTION**: RGB splitting and sudden clip-path tears.
3. **PIXEL CORRUPTION**: Individual block elements that spawn, blink, and scale up/down.
4. **SCANLINE SWEEP**: An infinite CRT-style scanline overlay.
5. **ENERGY CRACKS**: Glowing seams beneath the broken surface.
6. **DIMENSION REVEAL**: A secondary glowing hue that pulsates when the primary layer fractures.
7. **CHROMATIC ABERRATION**: Pseudo-element drop shadows acting as RGB splits.
8. **TIME FREEZE**: Intentional static periods in keyframes to mimic system locking.
9. **REALITY REPAIR**: Reconnecting fragments, fading glitch layers, and stabilizing energy when hovering the center panel.
10. **SYSTEM FAILURE LOOP**: An infinite 16-second cycle of decay and reconstruction.

## 🌐 Browser Support
- Works best on Chromium-based browsers (Chrome, Edge).
- Requires support for CSS `:has()` selector for the interactive repair sequence.
- Requires `backdrop-filter` for glassmorphism effects.

## 🏃 How to Run
No build steps required. Double-click `demo.html` to launch the experience.
