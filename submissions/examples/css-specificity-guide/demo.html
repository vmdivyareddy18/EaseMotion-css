CSS Specificity & Override Guide

This guide explains how to customize EaseMotion CSS safely using CSS custom properties instead of relying on !important.

1. CSS Custom Property Override Pattern

Recommended approach:

:root {
    --ease-duration: 2s;
    --ease-delay: 0.5s;
}

Custom properties make it easy to update animations globally without modifying framework code.

2. Scoped Override Example

Global override:

:root {
    --ease-duration: 2s;
}

Component-specific override:

.hero-section {
    --ease-duration: 4s;
}

Use scoped variables when only a specific section requires customization.

3. Common Override Mistakes

Avoid Recommended 
.ease-bounce {
    animation-duration: 5s !important;
}

.hero-section {
    --ease-duration: 5s;
}
High-specificity selectors Simple scoped selectors
Editing framework classes directly Using CSS variables

4. Why Specificity Matters

Excessive selector specificity can make styles difficult to override and maintain. Scoped variables provide a cleaner and more predictable customization strategy.

5. Benefits of the Recommended Approach

  • Cleaner code
  • Improved maintainability
  • No !important conflicts
  • Better team collaboration
  • Easier future updates
