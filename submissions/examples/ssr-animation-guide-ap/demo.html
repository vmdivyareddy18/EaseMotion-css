SSR Animation Core Concepts

      

        

          
Pure CSS animations

          

            Base transitions using standard static class names (e.g. .ease-fade-in) execute instantly
            without requiring JavaScript hydration. They are fully SSR-compatible.
          

        


        

          
JS-Dependent triggers

          

            Animations relying on viewport tracking, click-based class-toggles, or staggered index delays require
            hydration, which can delay execution and cause visual flashes if not gated properly.
          

        


        

          
FOUC Prevention

          

            Prevent Flash of Unanimated Content (FOUC) by applying initial states (e.g. opacity: 0)
            directly in the CSS. Do not rely on JavaScript to hide elements before triggering animations.
