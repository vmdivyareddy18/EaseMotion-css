# Invoice Status Timeline

1. What does this do? Adds a responsive invoice status timeline with animated rails, pulsing active checkpoints, status tags, and hover or focus feedback.
2. How is it used? Apply `.status-timeline` to an ordered list and use `.status-step` with state classes like `.completed`, `.active`, or `.waiting` for each billing step.
3. Why is it useful? It makes billing progress easier to scan with purposeful CSS-only motion and keyboard-friendly focus treatment.
