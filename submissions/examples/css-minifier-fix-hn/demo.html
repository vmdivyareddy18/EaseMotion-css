CSS Minifier String Corruption Fix

Issue Fixed: In the build script (`build-minified-css.mjs`), whitespace stripping around structural CSS tokens (such as colons, braces, and greater-than arrows) was executed globally using `replace(/\s*([{}:;,>])\s*/g, "$1")`. This corrupted text values inside quotes (e.g. `content: "A > B"`) and SVG tags embedded as background image data URIs (breaking their tag structure).

Input CSS (contains quotes and SVG data URI)

.minifier-test-card {
  content: "User Feedback: Success > Failure, Next Steps";
  background-image: url("data:image/svg+xml;charset=utf-8,%3Csvg xmlns='http://www.w3.org/2000/svg' fill='none' viewBox='0 0 20 20'%3E%3Cpath stroke='%236c63ff' stroke-linecap='round' stroke-linejoin='round' stroke-width='1.5' d='M6 8l4 4 4-4'/%3E%3C/svg%3E");
}
