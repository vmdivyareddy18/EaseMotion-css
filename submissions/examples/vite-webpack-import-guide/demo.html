Import Guide

How to use EaseMotion CSS in Vite, webpack, and CDN projects

Step 1 — Install

bash 
npm install easemotion-css

Step 2 — Import in your entry file

Add one line to your main.js (or main.ts). Vite handles CSS imports natively — no extra config needed.

js 
// main.js or main.ts
import 'easemotion-css';

Granular imports (optional)

Import only the modules you need for a smaller bundle:

js 
// Core (required, in this order)
import 'easemotion-css/core/variables.css';
import 'easemotion-css/core/base.css';
import 'easemotion-css/core/animations.css';
import 'easemotion-css/core/utilities.css';

// Components (add as needed)
import 'easemotion-css/components/buttons.css';
import 'easemotion-css/components/cards.css';
💡
Vite supports CSS imports out of the box. No loaders or plugins required.

Framework examples

React + Vite:

jsx 
// src/main.jsx
import 'easemotion-css';
import React from 'react';
import ReactDOM from 'react-dom/client';
import App from './App';

ReactDOM.createRoot(document.getElementById('root')).render(<App />);

Vue + Vite:

js 
// src/main.js
import 'easemotion-css';
import { createApp } from 'vue';
import App from './App.vue';

createApp(App).mount('#app');

Svelte + Vite:

js 
// src/main.js
import 'easemotion-css';
import App from './App.svelte';

new App({ target: document.getElementById('app') });

Step 1 — Install

bash 
npm install easemotion-css

Step 2 — Configure CSS loaders

webpack needs css-loader to handle CSS imports. Add this to your config:

js 
// webpack.config.js
module.exports = {
  module: {
    rules: [
      {
        test: /\.css$/,
        use: ['style-loader', 'css-loader'],
      },
    ],
  },
};
⚠️
If you don't have css-loader installed: npm install -D css-loader style-loader

Step 3 — Import

js 
// index.js or app.js
import 'easemotion-css';

Granular imports

js 
import 'easemotion-css/core/variables.css';
import 'easemotion-css/core/base.css';
import 'easemotion-css/core/animations.css';
import 'easemotion-css/core/utilities.css';
💡
For production, consider using MiniCssExtractPlugin instead of style-loader to extract CSS into separate files.

Full bundle (recommended)

One line in your HTML head. No npm, no build step, no config.

html 
<link rel="stylesheet"
      href="https://cdn.jsdelivr.net/npm/easemotion-css/easemotion.css" />

Individual modules

Load only what you need:

html 
<!-- Core (required, in this order) -->
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/easemotion-css/core/variables.css" />
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/easemotion-css/core/base.css" />
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/easemotion-css/core/animations.css" />
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/easemotion-css/core/utilities.css" />

<!-- Components (add as needed) -->
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/easemotion-css/components/buttons.css" />
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/easemotion-css/components/cards.css" />

unpkg alternative

html 
<link rel="stylesheet"
      href="https://unpkg.com/easemotion-css/easemotion.css" />
⚠️
Always load variables.css first — all other modules depend on the CSS custom properties it defines.
