EaseMotion CSS Core Concepts

Introduction

Welcome to the EaseMotion CSS documentation. EaseMotion CSS is a curated, animation-first CSS framework designed for developers who want expressive motion and clean HTML structure without complex build steps.

Our core philosophy states: "If you can say it in plain English, you should be able to write it as a class name." We prioritize developer readability and design consistency above all else.

Design Tokens

At the foundation of the framework is our design token system. Tokens define standard values for colors, transition speeds, easing curves, spacing scales, and border radii.

CSS Variables

All tokens are defined as standard CSS variables within core/variables.css. This ensures that changing a single token value updates the entire framework consistently.

Animations

Our expressive animation utilities provide standard CSS keyframes for entrances, exits, and loop indicators. Standard classes include ease-fade-in, ease-slide-up, and ease-bounce.

Keyframe Declarations

Animation keyframes are stored in core/animations.css. They utilize modern web specifications and run directly on the compositor thread for optimal performance.

Utilities

EaseMotion CSS includes layout utilities such as ease-center, ease-flex, and ease-grid, which map layout requirements to CSS Flexbox and Grid.

Accessibility

Accessibility is a first-class citizen in EaseMotion CSS. Every animation class in the framework has built-in support for prefers-reduced-motion media queries to prevent dizziness or vestibular discomfort.
