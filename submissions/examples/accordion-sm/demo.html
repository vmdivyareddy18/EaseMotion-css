CSS-only Accordion

Built on native <details> / <summary>. No JavaScript — smooth open/close, keyboard accessible out of the box.

What is EaseMotion CSS?

A zero-dependency, animation-first CSS framework with readable class names like ease-fade-in and ease-hover-grow. No build step, no config.

Does it need JavaScript?

No. This accordion runs entirely on native <details>/<summary> behavior with CSS-driven open/close transitions.

Is it accessible?

Yes — keyboard navigation, focus states, and screen-reader semantics all come for free from the native <details> element.

Glass variant, inside .ease-card

Glass header style

The glass variant applies a frosted, translucent header while keeping the same CSS-only transition logic as the base accordion.

Works inside cards

Drop it inside .ease-card and it inherits the surrounding surface without extra markup.
