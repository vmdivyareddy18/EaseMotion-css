Use EaseMotion CSS with React, Vue, and Astro — installation, SSR, scoped styles, and examples.

Using EaseMotion CSS with React

Installation

npm install easemotion-css

Global Import

Import the stylesheet in your entry point:

// app.jsx or main.jsx
import 'easemotion-css/easemotion.min.css';

Next.js App Router — import in app/layout.js:

// app/layout.js
import 'easemotion-css/easemotion.min.css';

export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  );
}

Next.js Pages Router — import in pages/_app.js:

// pages/_app.js
import 'easemotion-css/easemotion.min.css';

function MyApp({ Component, pageProps }) {
  return <Component {...pageProps} />;
}
export default MyApp;

Component Example

function HeroSection() {
  return (
    <section className="ease-center ease-padding-16">
      <h1 className="ease-fade-in">Build faster.</h1>
      <button className="ease-btn ease-btn-primary ease-hover-grow">
        Get Started
      </button>
    </section>
  );
}

Card Component

function ProfileCard({ name, role }) {
  return (
    <div className="ease-card ease-hover-lift" style={{ maxWidth: 320 }}>
      <h2 className="ease-text-lg">{name}</h2>
      <p className="ease-text-sm ease-muted">{role}</p>
      <button className="ease-btn ease-btn-outline ease-btn-sm">
        View Profile
      </button>
    </div>
  );
}

SSR / Next.js Notes

  • Import in app/layout.js (App Router) or pages/_app.js (Pages Router) for SSR.
  • EaseMotion CSS uses CSS custom properties — no FOUC on SSR.
  • No tree-shaking needed; unused classes add ~5 KB gzipped.

Scoped Styles / CSS Modules

import styles from './MyComponent.module.css';

function MyComponent() {
  return (
    <div className={`ease-card ${styles.wrapper}`}>
      {/* scoped + global work together */}
    </div>
  );
}

Using EaseMotion CSS with Vue

Installation

npm install easemotion-css

Global Import

// main.js
import { createApp } from 'vue';
import App from './App.vue';
import 'easemotion-css/easemotion.min.css';

createApp(App).mount('#app');

Component Example

<template>
  <section class="ease-center ease-padding-16">
    <h1 class="ease-fade-in">Build faster.</h1>
    <button class="ease-btn ease-btn-primary ease-hover-grow">
      Get Started
    </button>
  </section>
</template>

Card Component

<template>
  <div class="ease-card ease-hover-lift" style="max-width: 320px">
    <h2 class="ease-text-lg">{{ name }}</h2>
    <p class="ease-text-sm ease-muted">{{ role }}</p>
    <button class="ease-btn ease-btn-outline ease-btn-sm">
      View Profile
    </button>
  </div>
</template>

<script setup>
defineProps({ name: String, role: String });
</script>

Nuxt / SSR

For Nuxt 3, import via nuxt.config.js:

// nuxt.config.js
export default defineNuxtConfig({
  css: ['easemotion-css/easemotion.min.css'],
});

Scoped Styles

<template>
  <div class="ease-card custom-wrapper">
    <slot />
  </div>
</template>

<style scoped>
.custom-wrapper {
  background: var(--ease-color-surface);
}
</style>

Using EaseMotion CSS with Astro

Installation

npm install easemotion-css

Global Import

Import in your layout frontmatter:

---
// src/layouts/BaseLayout.astro
import 'easemotion-css/easemotion.min.css';
---
<html lang="en">
  <head><title>My Site</title></head>
  <body>
    <slot />
  </body>
</html>

Component Example

---
import BaseLayout from '../layouts/BaseLayout.astro';
---
<BaseLayout>
  <section class="ease-center ease-padding-16">
    <h1 class="ease-fade-in">Build faster.</h1>
    <button class="ease-btn ease-btn-primary ease-hover-grow">
      Get Started
    </button>
  </section>
</BaseLayout>

Card Component

---
export interface Props { title: string; description: string; }
const { title, description } = Astro.props;
---
<div class="ease-card ease-hover-lift" style="max-width: 320px">
  <h2 class="ease-text-lg">{title}</h2>
  <p class="ease-text-sm ease-muted">{description}</p>
  <a href="#" class="ease-btn ease-btn-outline ease-btn-sm">
    Learn More
  </a>
</div>

SSR / Static Output

Astro extracts and inlines CSS during build — importing in any layout ensures styles are included in the final output with no extra configuration.

Scoped Styles

<div class="ease-card custom">
  <slot />
</div>

<style>
  .custom {
    border: 2px solid var(--ease-color-primary);
  }
</style>
