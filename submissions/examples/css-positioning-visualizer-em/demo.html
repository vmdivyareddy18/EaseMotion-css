CSS Positioning Visualizer

An interactive, pure HTML & CSS playground demonstrating core CSS positioning modes and their spatial behaviors.

Default

position: static

Elements render in order as they appear in the normal document flow. The top, right, bottom, left, and z-index properties have no effect.

Self-Relative

position: relative

The element is positioned relative to its normal position. Offsets shift it without affecting the layout of surrounding elements, leaving an empty gap where it originally was.

Target Box (Relative: top: 15px, left: 25px)

Parent-Bound

position: absolute

The element is removed from the normal document flow and positioned relative to its closest positioned ancestor (anything other than static). Other elements behave as if it isn't there.

Target Box (Absolute: top: 10px, right: 10px)

Scroll-Toggle

position: sticky

A hybrid of relative and fixed positioning. The element is treated as relative until it crosses a specified scroll threshold within its container, where it then "sticks" in place.

Scroll down to see the sticky behavior...
Target Box (Sticky: top: 0)
The box stays stuck at the top while scrolling through this container block until the end of its parent layout boundary is reached.
Viewport-Bound

position: fixed

The element is removed from the document flow and positioned relative directly to the viewport window. It remains locked in the exact same place even when the entire page scrolls.
