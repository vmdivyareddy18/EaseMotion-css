Pre-commit Hook

Auto-format EaseMotion CSS files before every commit using Husky + lint-staged

1. Install dependencies

Install Husky and lint-staged as dev dependencies in your project.

2. Configure lint-staged

Add the following to your package.json to run the formatter on staged CSS files.

// package.json
"lint-staged": {
  "*.css": [
    "npx prettier --write",
    "npx stylelint --fix"
  ]
}

3. Create the Husky hook

The .husky/pre-commit hook runs lint-staged automatically before each commit.

#!/usr/bin/env sh
. "$(dirname "$0")/_/husky.sh"

npx lint-staged

4. Initialize Husky

Run npx husky install and npx husky add .husky/pre-commit "npx lint-staged" to set up the hook.

How It Works

When you run git commit, Husky triggers the pre-commit hook, which executes lint-staged. Only staged CSS files are formatted with Prettier and linted with Stylelint. If formatting fails, the commit is aborted, ensuring only clean CSS enters the repository.
