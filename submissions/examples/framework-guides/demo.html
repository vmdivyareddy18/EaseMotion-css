EaseMotion CSS

Seamlessly integrate the animation-first CSS framework into your favorite modern frontend web frameworks.

Quick Start Get EaseMotion CSS

Install the framework package via your package manager of choice to make all utility classes and animation keyframes available in your build process.

npm install easemotion-css

Integration Guides

Select your framework to view tailored setup, imports, and options.

Vite Setup (Vanilla / React / Vue)

For standard Vite builds, import the stylesheet directly inside your main entry point file (usually main.js, main.ts, or index.js) to load the complete animation framework globally.

src/main.js 
// Import Vite's base styles
import './style.css';

// Import EaseMotion compiled CSS
import 'easemotion-css';

// Alternatively, load the minified version
// import 'easemotion-css/easemotion.min.css';

SCSS Imports in Vite

Vite has built-in preprocessor support. If your project uses Sass, install sass dependency and use the modern @use rule to import EaseMotion's variables and mixins directly into your custom stylesheets.

src/style.scss 
// Import the main SCSS variables/mixins
@use "easemotion-css/scss" as ease;

// Use variables inside your custom selectors
.custom-card {
  transition-duration: ease.$duration-normal;
  @include ease.animate-fade-in;
}

Fine-Grained Loading Options

Instead of importing the bundle, you can optimize your build and include only specific modules (e.g. only core animations, custom variables, or structural utilities) depending on your application requirements.

Granular Import Example 
// Import variables first (CSS custom properties)
import 'easemotion-css/core/variables.css';

// Import animations layer
import 'easemotion-css/core/animations.css';

// Import only the structural utilities
import 'easemotion-css/core/utilities.css';

Next.js App Router (13+)

Add EaseMotion globally by importing it in your root layout file. This imports all the animation utility classes and keyframes for use across all server and client components.

app/layout.tsx 
import type { Metadata } from 'next';
import './globals.css';

// Import EaseMotion global styles
import 'easemotion-css/easemotion.min.css';

export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  );
}

Next.js Pages Router

For Pages Router applications, import the global compiled CSS within your custom App file located inside your pages/ directory.

pages/_app.tsx 
import type { AppProps } from 'next/app';
import '../styles/globals.css';

// Import EaseMotion CSS globally
import 'easemotion-css/easemotion.min.css';

export default function MyApp({ Component, pageProps }: AppProps) {
  return <Component {...pageProps} />;
}

SCSS Imports in Next.js Custom Stylesheets

To use SCSS imports in Next.js, configure the Sass options inside your next.config.js file to resolve the path correctly, then import modules inside your layout or component styles.

next.config.js 
const path = require('path');

module.exports = {
  sassOptions: {
    includePaths: [path.join(__dirname, 'node_modules')],
  },
};

Create React App Setup

Import EaseMotion CSS inside your main application file (typically src/index.js or src/App.js) directly below standard CSS resets.

src/index.js 
import React from 'react';
import ReactDOM from 'react-dom/client';
import './index.css';
import App from './App';

// Import EaseMotion styles globally
import 'easemotion-css/easemotion.min.css';

const root = ReactDOM.createRoot(document.getElementById('root'));
root.render(
  <React.StrictMode>
    <App />
  </React.StrictMode>
);

CRA with Sass / SCSS Support

For CRA, install the sass compiler globally in your local dev dependencies. Import EaseMotion's SCSS directly inside any of your modular preprocessed stylesheets.

src/App.scss 
// Import preprocessed variables/mixins
@import "~easemotion-css/scss/index";

.app-header {
  background-color: #282c34;
  // Call an EaseMotion mixin if needed
  @include animate-fade-in;
}

Troubleshooting & FAQs

Quick answers to common issues faced when using CSS frameworks in modern bundlers.

Double check your package import. In some build setups, imports referencing only the package name (e.g. import 'easemotion-css') might not resolve styles correctly if the bundler does not map standard "style" fields in package.json.

Fix: Use the explicit path directly inside your imports: 

import 'easemotion-css/easemotion.min.css';

When compiling SCSS in frameworks, compiler paths might differ depending on configuration. By default, @use 'easemotion-css/scss' works under modern Dart Sass. If you are using webpack (like in older CRA versions), prepend a tilde symbol ~ to signal resolution from node_modules.

Webpack Tip: Use @import "~easemotion-css/scss/index"; to import modules properly when building with older Webpack sass-loaders.

Next.js pre-renders pages on the server. If your animation styles load late, the page might render in its static form before snapping to the animated state, causing a visual jitter.

Fix: Ensure the global stylesheet import in your layout (or _app.js) is positioned before any component or custom stylesheets. This ensures the browser receives layout and animation custom properties as early as possible.

EaseMotion is highly compatible with other frameworks since it encapsulates animations inside dedicated transition classes. However, make sure you don't declare overlapping selector names.

Best Practice: Keep EaseMotion animations and Tailwind utility classes separated. Do not apply animation utility classes to elements that are also utilizing Tailwind transition properties to avoid conflict.
