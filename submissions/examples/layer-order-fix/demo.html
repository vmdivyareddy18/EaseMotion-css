Explicit @layer Order

Fixes #4250 — adds a master layer-order statement to easemotion.css so cascade precedence is deterministic regardless of import order.

Layer Precedence (lowest → highest)

easemotion-base Tokens, resets, variables
↓ overridden by
easemotion-components Buttons, cards, modals…
↓ overridden by
easemotion-utilities Spacing, display, text helpers
↓ overridden by
Consumer styles (unlayered) Always win — no !important needed

The Fix — One Line in easemotion.css

/* Must be the FIRST rule in easemotion.css */
@layer easemotion-base, easemotion-components, easemotion-utilities;

Consumer Override Pattern

/* ✅ Consumer CSS — no @layer wrapper, no !important needed */
.my-brand-btn {
  background: #f38ba8;  /* overrides easemotion-components */
  border-radius: 2px;
}
