🔍 Audit: demo.html Validator

Scans all submissions/examples/ folders for missing or empty demo.html files

Audit Results Preview

— Total
— Passed
— Issues

How It Works

  1. Iterates through every subdirectory in submissions/examples/
  2. Checks that demo.html exists and is non-empty
  3. Reports MISSING or EMPTY for each broken submission
  4. Prints a summary with total count and issue count

Run the Audit

bash submissions/examples/audit-demo-html/audit.sh

Execute from the repository root.

Sample Output

MISSING: my-broken-submission — no demo.html
EMPTY:  another-folder — demo.html is empty
---
Audited: 42 submissions
Issues:  2
