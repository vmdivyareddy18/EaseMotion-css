Badge URL

https://img.shields.io/badge/CSS%20Size-24.8%20KB-6c5ce7

A Shields.io dynamic badge URL. The -6c5ce7 parameter sets the accent colour to match the EaseMotion brand.

Update Flow

1 Release cut triggers CI
2 Build generates minified CSS
3 Shields.io badge updates automatically

Each time a release is published, a GitHub Action extracts the minified CSS size and updates the badge URL parameter.

Why Use a Size Badge?

Dynamic repository badges build trust and improve the developer experience.

Transparency

Users see the exact bundle size before downloading or integrating. No surprises — just clear, honest metrics at a glance.

Performance Awareness

Keeping CSS lean is a shared goal. A visible size badge encourages maintainers to optimise and prevents bundle bloat over time.

Release Monitoring

Each release automatically updates the badge. Teams can track size regressions or improvements directly from the repository README.

Developer Confidence

A professional badge signals quality and maintenance. Developers are more likely to adopt a library that presents its metrics upfront.
