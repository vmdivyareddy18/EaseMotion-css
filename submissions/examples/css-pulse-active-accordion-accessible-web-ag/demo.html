Accessibility FAQ

What is WCAG 2.1 compliance?
The Web Content Accessibility Guidelines (WCAG) 2.1 define how to make web content more accessible to people with disabilities. Level AA compliance is the most commonly targeted standard and covers requirements for contrast ratios, keyboard navigation, focus management, and screen reader compatibility.
Note: This accordion component meets WCAG 2.1 Level AA requirements for color contrast (minimum 4.5:1 for normal text, 3:1 for large text), keyboard operability, and focus indicators.
How does prefers-reduced-motion work?
The prefers-reduced-motion CSS media query detects whether the user has requested minimal animations through their operating system accessibility settings. When enabled, this accordion disables all pulse and reveal animations, replacing them with instant state changes. This helps users with vestibular disorders who experience motion sickness from screen animations.
Why use native details/summary elements?
Native <details> and <summary> elements provide built-in keyboard support (Enter and Space to toggle), automatic ARIA expanded/collapsed state announcements, and correct focus order — all without a single line of JavaScript. This makes the accordion inherently accessible and reduces the surface area for accessibility bugs.
