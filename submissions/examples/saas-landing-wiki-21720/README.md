# Documentation SaaS Landing — Team Wiki (`saas-landing-wiki-21720`)

This proposal provides a complete, copy-paste ready B2B SaaS landing page template designed specifically for Team Knowledge Base and Wiki software (similar to Notion or Slite), utilizing pure HTML/CSS and standard EaseMotion utility classes.

## 📌 Feature Overview

A modern documentation tool is defined by its clean UI, block-based editing experience, and real-time collaboration features. This template achieves a minimalist, focused aesthetic using a monochrome color palette, stark borders, and fluid structural animations to mimic the feeling of a lightweight workspace.

The layout includes:
- A hero section featuring an animated CSS-built block editor mockup with a sidebar and avatar stack.
- A trusted-by integration logo strip.
- A 4-item feature grid highlighting core wiki functionality.
- A real-time collaboration mockup showing threaded comments and a notification toast.
- A clear, transparent pricing layout highlighting individual vs team plans.
- Simple, text-focused testimonials and a comprehensive footer.

### Showcased Classes
The template leverages the following EaseMotion animation and interaction utilities:
- `.ease-slide-up` / `.ease-fade-in` (Applied sequentially to editor 'blocks' to simulate the document loading line-by-line)
- `.ease-bounce-in` (Used on the team avatars in the header to make them pop into view)
- `.ease-pulse` (Used creatively on an absolutely positioned 'cursor' mockup to simulate another user typing)
- `.ease-zoom-in` (Used to reveal the main editor and collaboration mockups with a subtle scaling effect)
- `.ease-hover-lift` (Standard interaction for primary buttons and interactive mockups)
- `.ease-slide-down` (For the sticky navigation bar entrance)

## ⚙️ How to Use

To view this full-page template locally, simply open the `demo.html` file in your web browser. 

The CSS handles the layout, typography, and styling, while standard `ease-*` interaction classes are attached directly to HTML elements to provide the animations. The editor mockup, complete with a simulated slash menu and checked to-do blocks, is built entirely with structural HTML and CSS, requiring zero JavaScript.

*Note: As per the contributing guidelines, this proposal is implemented inside a unique `submissions/examples/saas-landing-wiki-21720/` directory to avoid modifying core files directly and to ensure zero deletions.*
