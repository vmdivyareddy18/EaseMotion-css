🌑 Shadow DOM CSS Variable Tunnel

Fix for ease-* animations inside Web Components — Issue #8408

Root Cause — What Breaks vs What Works
❌ Before Fix — Classes Blocked
/* Shadow root cannot see .ease-fade-in defined in main document */ class MyCard extends HTMLElement { constructor() { super(); this.attachShadow({ mode: 'open' }); this.shadowRoot.innerHTML = ` <div class="ease-fade-in"> ❌ No animation — blocked </div> `; } }
✅ After Fix — Variables Tunnel Through
/* CSS variables pierce shadow boundary */ class MyCard extends HTMLElement { constructor() { super(); this.attachShadow({ mode: 'open' }); this.shadowRoot.innerHTML = ` <style> .fade-in { animation: var(--ease-fade-in); } </style> <div class="fade-in"> ✅ Fades in correctly! </div> `; } }

Alternative — Inject <link> Directly into Shadow Root
// Alternative: inject the whole stylesheet into shadow root // Use when you want full .ease-* class support inside the component class MyCard extends HTMLElement { constructor() { super(); this.attachShadow({ mode: 'open' }); this.shadowRoot.innerHTML = ` <!-- ✅ Inject full stylesheet into shadow root --> <link href="easemotion.css" rel="stylesheet"/> <!-- Now all .ease-* classes work as normal --> <div class="ease-fade-in">✅ Works!</div> <div class="ease-bounce">✅ Works!</div> `; } }

This method loads the full stylesheet per component instance. Use the CSS variable method above for better performance.

All animations respect prefers-reduced-motion via :root variable override.
