Security Policy

Vulnerability disclosure policies, CSS-specific guidance, and Subresource Integrity (SRI) configurations for EaseMotion CSS.

Supported Versions

EaseMotion CSS is a pure CSS framework with zero JavaScript and zero dependencies. The following versions are currently supported with security updates:

Version Status
1.0.x (latest) ✅ Active Support
< 1.0.0 ❌ No longer supported

We recommend all production deployments use the latest minor and patch versions to ensure stability and compatibility.

Scope & Security Concerns

EaseMotion CSS ships as plain .css files only — no JavaScript, no build scripts, no server-side code, and no npm scripts that execute at install time.

The primary security concerns for a CSS framework and how they apply to EaseMotion CSS are mapped below:

Concern Applies? Notes
CSS injection via class names ⚠️ Possible Depends on implementation. Clean/sanitize class name inputs.
XSS via CSS content: values ⚠️ Possible Browser-dependent. Ensure dynamic styling is sanitized.
Malicious @import in published files ✅ Audited All imports are strictly local-only. No remote resources.
Supply chain / dependency attack ✅ Low Risk Zero npm runtime dependencies to be compromised.
CDN tampering ✅ Low Risk Use Subresource Integrity (SRI) hashes in production.

Out-of-Scope Items

The following are considered out of scope for security reports and will not be classified as security vulnerabilities in EaseMotion CSS:

❌

Third-Party CDN / Provider Vulnerabilities

Issues related to CDN hosting providers (such as jsDelivr or cdnjs) tampering with files, unless it stems from a compromise of the maintainer's registry account.

❌

Demos and Examples Only

Security flaws residing in community demos, test suites, or non-production subfolders, unless they directly demonstrate an exploit in the core CSS stylesheet itself.

❌

Browser Bugs / Outdated Clients

Exploits that rely on outdated user browsers, client-side rendering engine bugs, or customized browser extensions that fail to parse CSS standards correctly.

❌

Social Engineering & Phishing

Social engineering campaigns, domain squatted package targets, or phishing emails directed at the project maintainers or community contributors.

Reporting a Vulnerability

⚠️ Do not open a public GitHub issue for security vulnerabilities.

If you discover a security issue — including a malicious payload in npm, a CSS injection vector, or a CDN integrity issue — please report it privately.

Contact the maintainer directly through GitHub's private vulnerability advisory or via email:

📧 Report privately via GitHub's private vulnerability reporting portal

What to include in your report:

1. Description of the vulnerability
2. Steps to reproduce it (include a Proof of Concept if possible)
3. Affected version(s)
4. Potential impact
5. Suggested patch/fix (optional but appreciated)

Response Timeline Expectations:

Timeline Action
Within 48 hours Acknowledgement of your report.
Within 7 days Initial assessment and severity classification.
Within 30 days Patch released (for confirmed vulnerabilities).
After patch Public disclosure with credit to reporter.
Open Private Advisory 🚀

CSS-Specific Security Notes

1. CSS Injection

If your application dynamically constructs EaseMotion CSS class names from user inputs, always sanitize that input first. Avoid constructing strings directly from raw parameters:

// ❌ UNSAFE — never build class names from raw user input
element.className = `ease-${userInput}`;
// ✅ SAFE — use an allowlist of valid class names
const allowed = ['ease-fade-in', 'ease-slide-up', 'ease-btn-primary'];
if (allowed.includes(userClass)) {
  element.className = userClass;
}

2. Subresource Integrity (SRI)

When loading EaseMotion CSS via CDN, verify integrity using SRI hashes to protect your application against CDN compromise or DNS hijacking:

<!-- Generate your SRI hash at: https://www.srihash.org/ -->
<link
  rel="stylesheet"
  href="https://cdn.jsdelivr.net/npm/easemotion-css@1.0.0/easemotion.css"
  integrity="sha384-<YOUR_HASH_HERE>"
  crossorigin="anonymous"
/>
