Delay Test — Critical Case for fill-mode: both

Without fill-mode: both, delayed animations show the element at full opacity during the delay, then jump back to invisible — a double flash. With both, elements stay invisible during the delay.

Delay 100ms .ease-delay-100
Delay 300ms .ease-delay-300
Delay 500ms .ease-delay-500
Delay 700ms .ease-delay-700
Delay 1000ms .ease-delay-1000
Why both and not forwards? forwards only holds the end state. backwards only holds the start state. both does both — ensures elements stay invisible before the animation starts AND stay visible after it ends. For entry animations this is always the correct value.
