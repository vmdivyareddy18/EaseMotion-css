Fix: Fragmented prefers-reduced-motion Implementation

core/base.css only disables scroll-behavior under prefers-reduced-motion: reduce — there's no global animation/transition kill-switch. An audit found 17 component files animating or transitioning something with zero reduced-motion guard at all.

💡 Why this matters: Today, whether a spinning loader or sliding tab respects reduced-motion depends entirely on whether that specific component file remembered to add its own @media block. A single global catch-all rule in core/base.css would protect every current and future component automatically.

The exact patch needed in core/base.css

@media (prefers-reduced-motion: reduce) {
  html {
    scroll-behavior: auto;
  }
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
