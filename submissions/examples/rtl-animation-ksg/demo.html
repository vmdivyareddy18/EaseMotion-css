RTL-Friendly Animations

One animation class. Both directions. Automatically — using dir, :dir(), and CSS logical properties.

Technique A

Slide-in from reading-direction start

Two keyframes (slide-in-from-start-ltr / slide-in-from-start-rtl) are selected by the CSS :dir() pseudo-class, with a [dir] attribute fallback for broader browser support. The same class name works in both contexts — the browser picks the correct animation.

⟶ Left-to-Right (English)

👋 Hello, EaseMotion 
/* Enters from the LEFT (inline-start in LTR) */
:is([dir="ltr"]) .slide-in-directional {
  animation: slide-in-from-start-ltr 600ms ease both;
}

⟵ Right-to-Left (Arabic)

👋 مرحباً، EaseMotion 
/* Enters from the RIGHT (inline-start in RTL) */
:is([dir="rtl"]) .slide-in-directional {
  animation: slide-in-from-start-rtl 600ms ease both;
}

The two keyframes (only the sign of translateX differs):

@keyframes slide-in-from-start-ltr {
  from { opacity: 0; transform: translateX(-36px); }
  to   { opacity: 1; transform: translateX(0); }
}

@keyframes slide-in-from-start-rtl {
  from { opacity: 0; transform: translateX(+36px); }
  to   { opacity: 1; transform: translateX(0); }
}
Technique B

CSS logical properties — zero direction overrides

Logical properties like inset-inline-start, padding-inline, and text-align: start map to the correct physical side automatically based on writing direction. No :dir() selectors needed — just write the rule once.

⟶ LTR underline

Hover to reveal underline 
/* inset-inline-start = LEFT in LTR */
.underline-directional::after {
  inset-inline-start: 0;  /* no LTR/RTL override! */
  width: 0;
  transition: width 300ms ease;
}

⟵ RTL underline

حرك المؤشر لإظهار الخط 
/* inset-inline-start = RIGHT in RTL */
.underline-directional::after {
  inset-inline-start: 0;  /* same rule, flipped! */
  width: 0;
  transition: width 300ms ease;
}

The same inset-inline-start: 0 technique anchors toast notifications to the correct corner in both directions:

Technique C

direction-aware transform-origin: inline-start

transform-origin supports logical keywords. Setting transform-origin: inline-start center makes a scaleX() animation always expand from the start of the reading direction — no separate RTL keyframe needed for this case.

⟶ LTR — expands left → right

.progress-directional {
  transform-origin: inline-start center;
  animation: scaleX(0→1) 1.2s ease both;
}
/* No RTL override needed for transform-origin! */

⟵ RTL — expands right → left

/* Same class, same transform-origin rule */
.progress-directional {
  transform-origin: inline-start center;
  /* inline-start = RIGHT in RTL → expands RTL */
}
