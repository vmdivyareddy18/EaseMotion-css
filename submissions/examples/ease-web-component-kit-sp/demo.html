Attribute → class mapping

How custom element attributes translate to EaseMotion CSS classes.

Element Attribute Values EaseMotion classes
ease-button variant primary, success, danger, outline, ghost .ease-btn-primary … .ease-btn-ghost
ease-button size sm, md, lg .ease-btn-sm / .ease-btn-lg
ease-button animation fade-in, slide-up .ease-fade-in / .ease-slide-up
ease-button hover grow, lift .ease-hover-grow / .ease-hover-lift
ease-card variant shadow, glass, accent, hover .ease-card-shadow … .ease-card-hover
ease-card accent primary, success, danger, warning .ease-card-accent-*
ease-card animation fade-in, slide-up .ease-fade-in / .ease-slide-up

Usage guidance

Why Web Components?

EaseMotion is class-based. Wrapping common patterns in custom elements keeps HTML readable and prevents copy-paste errors across pages.

Light DOM pattern

  • Components render real <button> / <div> nodes.
  • EaseMotion global CSS applies without Shadow DOM barriers.
  • Slot content stays in light DOM for SEO and accessibility.

Extending the kit

  • Add ease-badge, ease-alert, etc. the same way.
  • Map new attributes in observedAttributes.
  • Import the script once per page — no build step required.
