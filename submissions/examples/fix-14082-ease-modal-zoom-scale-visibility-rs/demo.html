Fix for #14082: ease-modal-zoom scale visibility

The .ease-modal-zoom (and other animation variants) defined a transition for opacity, but lacked the actual opacity: 0 and opacity: 1 properties. When applied to standalone custom elements (without the base .ease-modal class), the opacity transition fails. As a result, the element abruptly pops into full visibility and only scales, rather than fading in smoothly.
