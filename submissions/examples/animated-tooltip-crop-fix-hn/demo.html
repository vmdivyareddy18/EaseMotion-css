Tooltip Mobile Responsive Fix

Issue Fixed: In mobile layouts (viewports <= 640px), tooltips situated at the top are automatically flipped to the bottom to prevent cropping off the top edge of the screen. However, the original CSS media query only targeted explicit top tooltips ([data-position="top"]), neglecting default tooltips that have no position attribute.

We resolved this by adding the default selector :not([data-position]) to the responsive media queries.

Testing Instructions:

1. Hover over both buttons in desktop mode. Both tooltips will render **above** the buttons.
2. **Resize your browser viewport width to under 640px** (or toggle mobile view in Developer Tools).
3. Hover over both buttons again. Both tooltips will now correctly render **below** the buttons, avoiding clipping.
