Keyboard-Navigable Accordion

Arrow keys · ARIA · Single/Multi mode · Staggered content reveal

EaseMotion CSS is a zero-dependency, animation-first CSS framework that provides human-readable utility class names. If you can say it in English, you should be able to write it as a class — that is the core philosophy.

Classes like ease-fade-in, ease-slide-up, and ease-center make it easy to style and animate elements without writing custom CSS or JavaScript.

This accordion uses JavaScript for enhanced keyboard navigation and accessibility while keeping the visual design fully CSS-driven.

Keyboard shortcuts:

  • Tab — Move focus between accordion items
  • Enter / Space — Toggle current item
  • ↑ ↓ — Navigate between items
  • Home — Focus first item
  • End — Focus last item

This accordion implements the WAI-ARIA Accordion Pattern:

  • aria-expanded on each trigger button
  • aria-controls connecting trigger to panel
  • aria-labelledby on each panel for context
  • aria-hidden on closed panels
  • role="region" with aria-label on the container
  • Full keyboard navigation (Arrow keys, Home, End)
  • Visible focus indicators via :focus-visible
  • prefers-reduced-motion support

  3. Include demo.html, style.css, and README.md
  4. Open a Pull Request — the maintainer will review and standardize

Compact Variant

Install via npm: npm install easemotion-css, or link directly from CDN.

Add <link rel="stylesheet" href="easemotion.css"> to your HTML head, then use classes like class="ease-fade-in".

Minimal Variant (No Borders)

Pure CSS — no JavaScript required for core styling. Just a single CSS file and you are ready to go.

Motion is a first-class citizen. 40+ keyframes, 20+ animation classes, scroll-triggered reveals, and hover effects.
