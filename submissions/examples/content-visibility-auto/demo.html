What is content-visibility?

content-visibility: auto skips rendering of off-screen elements, dramatically improving initial load and scroll performance.

The content-visibility property is a CSS feature that allows the browser to skip rendering of elements that are not currently visible on screen. When an element has content-visibility: auto, the browser defers layout, painting, and compositing until the element is about to enter the viewport. 

section {
  content-visibility: auto;
  contain-intrinsic-size: 10px 40vh;
}

The contain-intrinsic-size property provides a placeholder size so that scrollbars and layout remain stable even when the content hasn't been rendered yet.

Section 3 — contain-intrinsic-size

Placeholder sizing prevents scroll jank.

When using content-visibility: auto, always pair it with contain-intrinsic-size. This tells the browser the natural size of the element before it renders, preventing layout shifts during scrolling. 

.lazy-section {
  content-visibility: auto;
  contain-intrinsic-size: 10px 40vh;
}
  • 10px — inline-size hint (width)
  • 40vh — block-size hint (height)
  • Adjust based on your content's expected dimensions

Section 4 — Interaction with IntersectionObserver

content-visibility works alongside JS APIs.

content-visibility: auto integrates naturally with IntersectionObserver. When a section comes into view, it triggers rendering automatically — no JavaScript needed for the basic case. You can still use JS for additional effects.

Unlike loading="lazy" on images, content-visibility works on any DOM subtree, making it perfect for long-form articles, dashboards, and feed layouts.

Section 5 — Subtree Visibility

Children inherit the containment context.

When an element has content-visibility: auto, its entire subtree is skipped from rendering. This means deeply nested content like lists, tables, and complex components all benefit from a single declaration on a parent container.

Section 6 — Scrollbar Stability

Without contain-intrinsic-size, scrollbar jumps.

When content-visibility is enabled but contain-intrinsic-size is omitted, the scrollbar length may jump as sections render, creating a poor user experience. Always provide a reasonable size hint.

  • Stable scrollbar — use contain-intrinsic-size
  • Jumpy scrollbar — missing size hint
  • Best practice — set both properties together

Section 7 — Canvas & Media

Works with images, iframes, and canvas elements.

Content-visibility works well with embedded media. Images inside a content-visibility container will also defer loading. This is complementary to the loading="lazy" attribute on images.

For canvas elements and WebGL contexts, deferring rendering until the element is visible can save significant GPU resources.

Section 8 — Accessibility Considerations

Screen readers can still access the full DOM.

Even though off-screen content is not visually rendered, it remains in the DOM and is accessible to screen readers and search engines. The browser still builds the full accessibility tree, so users navigating with assistive technology can tab through all content.

Section 10 — Real-world Usage

Used by Wikipedia, Facebook, and major news sites.

Major websites use content-visibility for long article pages, infinite scroll feeds, and dashboard views. It is particularly effective on mobile devices where CPU and GPU resources are constrained.

Section 11 — Debugging content-visibility

Chrome DevTools highlights skipped content.

In Chrome DevTools, the Elements panel shows a [cv] badge next to elements with content-visibility. The Performance panel records when each section renders, letting you verify the deferred behavior.

Section 12 — Limitations

Not suitable for every layout.

Content-visibility is not recommended for content that must be immediately measurable (e.g., height-calculation-dependent layouts) or for elements that need continuous rendering (animations, video thumbnails in a grid). It also does not work in print media.
