Marquee Accessibility Guidance

Issue #9470 — three accessible patterns for .ease-marquee

Each pattern below shows a working marquee with the correct ARIA attributes for its use case, plus the HTML markup to copy. Use a screen reader (NVDA, JAWS, VoiceOver) or the browser's Accessibility Tree in DevTools to verify how each pattern is announced.
Pattern A — Decorative
Logo strip / brand strip (content repeated elsewhere)
Use aria-hidden="true" when the logos are purely decorative or the same information appears in a visible list elsewhere on the page. The marquee is removed from the accessibility tree entirely — screen readers skip it and there is no duplicate announcement.

<!-- Pattern A: decorative — hidden from screen readers --> <div class="ease-marquee" aria-hidden="true"> <div class="ease-marquee-track"> <img src="react.svg" alt="" /> <img src="node.svg" alt="" /> </div> </div>
Pattern B — Informational
Partner logos / feature highlights (content not elsewhere)
Use aria-label and aria-roledescription when the content is informational but not live or urgent. aria-live="off" on the track means the screen reader reads it once on page load and ignores scroll updates — no repeated announcements.

<!-- Pattern B: informational — labelled, no live updates --> <section class="ease-marquee" aria-label="Our technology partners" aria-roledescription="scrolling banner" > <div class="ease-marquee-track" aria-live="off"> <img src="react.svg" alt="React" /> <img src="node.svg" alt="Node.js" /> </div> </section>
Pattern C — Live ticker
News ticker / live announcements / real-time updates
Use role="log" and aria-live="polite" when the content is a live feed that must be communicated to all users. aria-atomic="false" announces only each new item as it appears — not the entire region. The screen reader waits for idle time before announcing, so it does not interrupt the user.
