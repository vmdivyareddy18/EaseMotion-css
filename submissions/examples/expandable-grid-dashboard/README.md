# Sandbox Component Showcase: Pure-CSS Expandable Dashboard Grid Column Track Matrix

## Overview
A zero-dependency, pure-CSS interactive expandable dashboard grid showcase and fluid container resizing utility widget component isolated within an examples subdirectory. It coordinates horizontal column lane transitions and hidden textual log reveals natively via parent relational state lookups, running smooth layout re-allocations without script utility frameworks.

## 📁 Sandbox Configuration Files
* `demo.html` — Interactive user cockpit hosting distinct column card modules configured with metadata identifiers to check expansion tracks.
* `style.css` — Scoped layout sheet establishing isolated fractional transition bounds and opacity cross-fades linked back to framework tokens.

## 🚀 Key Layout Mechanics
1. **Parent Relational State Interception:** Eliminates the necessity for performance-heavy click listeners or tracking scripts by deploying modern `:has()` selector arrays straight onto the parent grid envelope wrapper (`.alm-expandable-dashboard-grid:has(...)`). Selecting card nodes overrides column allocations natively: `.alm-expandable-dashboard-grid:has(.alm-dash-card-node[data-col="1"]:hover)`.
2. **Fractional Track Re-allocation:** Shifts horizontal boundaries seamlessly by mutating layout fraction variables from standard baseline widths up to extended spatial coordinates (`grid-template-columns: 1fr -> 2fr`). Because adjustments occur at the track declaration layer, peripheral grid boxes shrink proportionally on isolated browser pathways without layout thrashing jumps.
3. **Sandbox Compliance Safeguard:** Fully satisfies repository security policies by placing every asset line strictly inside `submissions/examples/expandable-grid-dashboard/`.
