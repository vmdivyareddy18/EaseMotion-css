Integrate the EaseMotion Platform API

Ship polished, animation-first experiences with enterprise-grade documentation patterns — sticky navigation, semantic callouts, structured API tables, and motion-rich micro-interactions built entirely with HTML and CSS.

Updated Jun 16, 2026 12 min read API v2.4
i

Before you begin

Create a workspace in the EaseMotion console, generate an API key with docs:read scope, and verify your origin domain for browser-based integrations.

Workspace configuration

Define environment variables in a YAML manifest. The platform validates schema on upload and surfaces actionable diffs in the activity panel.

config.yaml YAML 
workspace:
  name: acme-production
  region: us-east-1
  tier: enterprise

api:
  base_url: https://api.easemotion.dev/v2
  timeout_ms: 8000
  retry:
    max_attempts: 3
    backoff: exponential

features:
  motion_tokens: true
  audit_logs: true
  webhooks:
    - events: [deployment.created, docs.published]
      url: https://hooks.acme.io/easemotion
!

Rate limits apply

Enterprise workspaces receive 5,000 requests per minute. Burst traffic above 120% triggers a soft throttle with Retry-After headers.

Authentication

All requests require a bearer token. Rotate keys every 90 days and store secrets in your vault — never commit them to version control.

curl example HTTP 
curl https://api.easemotion.dev/v2/workspaces \
  -H "Authorization: Bearer em_live_7f3c..." \
  -H "Content-Type: application/json"

REST endpoints

Core resources for documentation publishing, search indexing, and workspace analytics.

Method Endpoint Description Scope
GET /v2/articles List published knowledge base articles docs:read
POST /v2/articles Create or draft a new article docs:write
PATCH /v2/search/reindex Trigger semantic index rebuild search:admin
DELETE /v2/webhooks/{id} Remove a webhook subscription webhooks:write

Webhook payloads

Events arrive as signed JSON documents. Verify signatures using your workspace secret before processing downstream automations.

docs.published event JSON 
{
  "id": "evt_9k2mQp1L",
  "type": "docs.published",
  "created_at": "2026-06-16T14:22:11Z",
  "data": {
    "article_id": "art_84Jf2",
    "slug": "platform-api-overview",
    "version": "2.4.0",
    "author": "platform-team@acme.io"
  }
}

Frequently asked questions

How do I migrate from API v1?

Use the guided migration wizard in the console or run em migrate plan to generate a diff of deprecated endpoints and updated payload shapes.

Can I self-host the documentation portal?

Enterprise plans include a static export pipeline. Build artifacts ship as pure HTML/CSS bundles compatible with any CDN or object storage origin.

What browsers are supported?

All modern evergreen browsers with CSS Grid, custom properties, and scroll-driven animation support. Graceful fallbacks preserve layout when advanced motion features are unavailable.
