Overview

Scroll-padding creates breathing room when anchor links snap into view. Without it, fixed headers obscure the top of each section. With scroll-padding, each section starts precisely below the navigation bar, giving every heading room to breathe.

The orange bar at the top of each section is a visual guide — it shows exactly where the scroll-padding offset lands. Notice how the heading text never overlaps the header.

Features

  • Smooth anchor scrolling — CSS handles the jump with scroll-behavior: smooth
  • Responsive offset — scroll-padding-top adapts to header height
  • Accessible by default — Works with keyboard and assistive tech
  • No JavaScript — Pure CSS behavior, zero dependencies

FAQ

What is scroll-padding? It is a CSS property that sets the scroll offset for the scrollport, useful for fixed headers.
How is this different from margin? Margin adds space in the document flow; scroll-padding adjusts the scroll snap/viewport alignment.
Is this accessible? Yes — it respects reduced-motion preferences.
