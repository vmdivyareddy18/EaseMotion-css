IntersectionObserver Utility

Each card below uses observe(el, callback, { once: true }) — they animate in once when scrolled into view, then unobserve.

Usage

observe(el, function(entry) {
  el.classList.add('observer-card--visible');
}, { threshold: 0.2, once: true });
01

Shared Instance

All observed elements share a single IntersectionObserver instance, reducing overhead.

02

WeakMap Tracking

Callbacks are stored in a WeakMap keyed by element, allowing automatic garbage collection.

03

Once Mode

By default, elements unobserve after the first intersection — ideal for one-time entrance animations.

04

Configurable

Supports custom threshold, rootMargin, and once options passed per element.

05

Cleanup

Elements removed from the DOM are automatically cleaned up to prevent memory leaks.
