Unicode Bidi Utilities

CSS unicode-bidi and direction utilities to control text flow directionality. Essential for localization, reversing character lines, and managing mixed-language (bi-directional) text rendering.

What do direction and unicode-bidi do?
- direction sets the base writing direction (LTR or RTL).
- unicode-bidi determines how the browser handles bi-directional text algorithms when mixing LTR languages (like English) and RTL languages (like Arabic or Hebrew).

Bidi Mode Comparisons

Normal Flow .ub-normal

English text containing Arabic: تفاح أحمر (Red Apples).

The default browser state. The text flow is determined strictly by the characters' native directionality rules.

Bidi Override (Reversal) .ub-bidi-override

Reversed English Text

Reverses LTR characters (e.g. English letters) to render right-to-left, printing "Reversed English Text" backward.

Bidirectional Embedding .ub-embed

Hello تفاح 123 World.

Embeds a local direction level (RTL) for inline text, opening a nested scope for bi-directional calculations.

Directional Isolation .ub-isolate

Hello تفاح 123 World.

Isolates inline text direction, preventing the nested RTL text or numbers from visually leaking into the surrounding layout.

Isolated Character Override .ub-isolate-override

Hello Isolate Override World.

Combines character isolation with directional override, reversing local characters without affecting adjacent blocks.

Plaintext Auto-Direction .ub-plaintext

Apple (Starts with LTR)

Ignores the container's dir-ltr and aligns each line based purely on the first letter's directionality.

Helper Writing Directions

Left-To-Right .dir-ltr

Standard English sentences write from left to right. Punctuations appear at the right end.

Initial base direction for Latin/Western scripts.
