🐛 Browser Process Reference Error Fix

Demonstrates graceful handling of invalid em attributes in browser environments

Before Fix ❌

// OLD CODE (throws ReferenceError in browsers)
if (process?.env?.NODE_ENV !== 'production') {
  console.warn('Invalid em value...')
}
// ReferenceError: process is not defined

Optional chaining (?.) doesn't protect against undeclared root identifiers. When process is undefined in browsers, this throws immediately.

After Fix ✅

// NEW CODE (safe in all environments)
if (typeof process === 'undefined' || process.env?.NODE_ENV !== 'production') {
  console.warn('Invalid em value...')
}
// Works gracefully in Node.js AND browsers

Explicit typeof check prevents ReferenceError before accessing process.env.
