Chrome DevTools Profiling Walkthrough

Follow these step-by-step instructions to analyze animation render performance using Chrome DevTools. Identify frame drops, layout thrashing, and paint flashing in your application.

  1. Open DevTools Rendering panel
    Press F12 to open DevTools, then press Ctrl + Shift + P (or Cmd + Shift + P on macOS) to open the Command Menu. Type "Rendering" and select "Show Rendering".
  2. Enable Paint Flashing & Layout Shift Regions
    In the Rendering drawer, check **Paint flashing** and **Layout Shift Regions**. When running the stress test above, you will see green flashes highlight the Unsafe Reflows panel as the browser repaints elements. The Safe GPU panel will show no flashes because the GPU handles positioning directly.
  3. Record a Performance Trace Timeline
    Go to the **Performance** tab, click **Record**, run the test for 3 seconds, and hit **Stop**. Inspect the **Main Thread** timeline. Look for red blocks indicating long frames and see "Recalculate Style" and "Layout" events.

Thread Execution Comparison

Browser Main Thread

Handles HTML parsing, layout geometry calculations, style computations, and page painting. Animating properties like left, top, width, or height blocks this thread, preventing user interactions and dropping frames.

Browser Compositor Thread

Runs asynchronously from the main thread. It delegates layers directly to the GPU compositor. Properties like transform and opacity run entirely on the GPU, allowing elements to animate smoothly even if the main thread is busy.
