# Business Landing — A/B Testing SaaS (`saas-landing-ab-testing-21713`)

This proposal provides a complete, copy-paste ready B2B SaaS landing page template specifically designed for A/B Testing, Feature Flagging, and Experimentation platforms (similar to Optimizely, VWO, or LaunchDarkly), using pure HTML/CSS and EaseMotion utility classes.

## 📌 Feature Overview

An experimentation SaaS needs to look analytical, data-driven, and trustworthy. This template uses a clean, structural aesthetic with a precise monospace/sans-serif typography combination and robust table/chart mockups.

The layout includes:
- A hero section featuring an animated CSS-built A/B test results dashboard mockup (tables and bar charts).
- A logo banner for data-driven companies.
- A 6-card platform feature grid.
- A dark-mode statistical engine section featuring a CSS/SVG Bayesian bell curve visualization.
- An integrations pill grid.
- A MAU-based tiered pricing layout.
- A comprehensive footer.

### Showcased Classes
The template leverages the following EaseMotion animation and interaction utilities:
- `.ease-fade-in` / `.ease-fade-up` (For smooth text entrances, logo grids, and integration cards)
- `.ease-slide-up` (For cascading structural layout animations in pricing, features, and specifically the data table rows in the mockup)
- `.ease-zoom-in` (For drawing attention to the primary mockups and SVG charts)
- `.ease-bounce-in` (For primary trial CTAs)
- `.ease-hover-lift` / `.ease-hover-scale` / `.ease-hover-zoom` (For interactive buttons, feature cards, and the mockups)

## ⚙️ How to Use

To view this full-page template locally, simply open the `demo.html` file in your web browser. 

The CSS handles the structural layout and styling, while standard `ease-*` interaction classes are attached directly to HTML elements to provide the animations without needing custom JavaScript or specific CSS keyframes. The SVG charts and dashboard mockups are built entirely with HTML/CSS.
