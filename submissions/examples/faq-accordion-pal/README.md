# Animated FAQ Accordion Example

## Overview
An animated, pure‑HTML + CSS FAQ accordion that showcases how to build an accessible, responsive, and visually appealing UI component using only the **EaseMotion CSS** utility classes and a tiny amount of custom CSS.

## Features
- No JavaScript – relies on native `<details>`/`<summary>` for accessibility.
- Responsive layout that works on mobile, tablet, and desktop.
- Smooth expand/collapse animation via CSS transitions.
- Card‑based design with rounded corners, subtle shadow, and hover‑grow effect.
- Clean, readable typography and generous spacing.
- Uses EaseMotion utility classes (`ease-center`, `ease-fade-in`, `ease-slide-up`, `ease-delay-*`, `ease-card`, `ease-hover-grow`).

## Technologies Used
- **HTML5**
- **CSS3** (custom properties, flexbox, transitions)
- **EaseMotion CSS** – utility‑first CSS framework providing animation helpers.

## Preview
Open `demo.html` in a browser to see:
1. A centered hero heading.
2. Five FAQ cards that expand/collapse with a smooth slide‑up animation.
3. Hover‑grow effect that subtly enlarges the card and deepens the shadow.
4. Full responsiveness – cards stack and adjust spacing on small screens.

## Usage
1. Clone the repository or copy the `faq-accordion-pal` folder into the `submissions/examples/` directory of the EaseMotion repo.
2. Open `demo.html` in any modern browser (no build step required).
3. Feel free to edit the questions/answers or adjust the CSS variables in `style.css`.

## Notes
- This submission lives entirely under `submissions/examples/faq-accordion-pal/` and **does not modify any core, component, or configuration files** of the EaseMotion framework.
- It is production‑ready and beginner‑friendly, demonstrating how to create interactive UI patterns with only HTML and CSS.
