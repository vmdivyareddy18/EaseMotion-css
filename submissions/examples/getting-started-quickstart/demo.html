Getting Started with EaseMotion CSS

A zero-dependency, animation-first CSS framework. Add motion to your UI in seconds.

CDN Setup npm Install
1

CDN Setup Fastest

Add this <link> to your <head> and you are ready to go:

<link rel="stylesheet"
  href="https://cdn.jsdelivr.net/gh/SAPTARSHI-coder/EaseMotion-css@main/easemotion.min.css" />

Powered by jsDelivr — globally cached, no account needed, live the moment you paste it.

2

npm Install Build Step

For projects using a build tool or package manager:

npm install easemotion-css

Then import in your entry file:

@import "easemotion-css/easemotion.min.css";
3

Your First Animation

Add animation classes directly to any HTML element. Here is a complete page:

<!DOCTYPE html>
<html>
<head>
  <link rel="stylesheet"
    href="https://cdn.jsdelivr.net/gh/SAPTARSHI-coder/EaseMotion-css@main/easemotion.min.css" />
</head>
<body>
  <div class="ease-center">
    <h1 class="ease-fade-in">Hello, EaseMotion!</h1>
    <p class="ease-slide-up ease-delay-200">Smooth motion.</p>
    <button class="ease-btn ease-btn-primary ease-hover-grow">
      Click Me
    </button>
  </div>
</body>
</html>
4

Staggered Sequence

Use ease-delay-* to create staggered entrance sequences:

<div class="ease-slide-up ease-delay-100">First</div>
<div class="ease-slide-up ease-delay-200">Second</div>
<div class="ease-slide-up ease-delay-300">Third</div>
5

Dark Mode Setup

EaseMotion CSS includes built-in dark mode support via the prefers-color-scheme media query. Add the ease-dark class to <html> or let it auto-detect:

<html class="ease-dark">

With an optional toggle button:

<button onclick="document.documentElement.classList.toggle('ease-dark')">
  Toggle Theme
</button>
6

Duration Helpers

Control animation speed with duration utilities:

ClassDuration
ease-duration-fast150ms
ease-duration-medium300ms
ease-duration-slow600ms
 
<div class="ease-fade-in ease-duration-slow">Slow fade</div>
7

Hover Effects

Add interactivity with hover animation classes:

<button class="ease-btn ease-btn-primary ease-hover-grow">
  Scale on hover
</button>

<button class="ease-btn ease-btn-primary ease-hover-shimmer">
  Shimmer on hover
</button>

<div class="ease-card ease-card-hover">
  Card lifts on hover
</div>
8

Scroll-Triggered Reveal

Use ease-reveal with an animation class to trigger on scroll. Include reveal.js before </body>:

<script src="https://cdn.jsdelivr.net/npm/easemotion-css/core/reveal.js"></script>
<div class="ease-reveal ease-slide-up">
  Animates when scrolled into view!
</div>
9

Customization

Override any CSS custom property to theme the framework:

:root {
  --ease-color-primary: #f97316;
  --ease-speed-fast: 100ms;
  --ease-speed-medium: 400ms;
  --ease-radius-md: 1rem;
}

Use @layer to override component styles safely:

@layer easemotion.components {
  .ease-btn-primary {
    background: #f97316;
    border-radius: 2rem;
  }
}
10

Troubleshooting

Animation not playing

Make sure the CDN link is correct and loads before your custom CSS. Check the browser console for 404 errors. Verify you have not accidentally disabled animations in your OS accessibility settings (the framework respects prefers-reduced-motion).

Classes not working

EaseMotion classes start with the ease- prefix (e.g. ease-fade-in). Double-check spelling — a missing hyphen or typo will silently fail. Ensure no other stylesheet is overriding the classes with higher specificity.

Dark mode not toggling

Apply ease-dark on <html>, not <body>. The framework uses CSS variables scoped to the root element. If using a toggle, ensure JavaScript targets document.documentElement.

Scroll reveal not triggering

Confirm reveal.js is included before </body>. The element must have both ease-reveal and an animation class. The reveal script uses IntersectionObserver — check browser support (Chrome 51+, Firefox 55+, Safari 12.1+).

npm import failing

Ensure easemotion-css is in your package.json dependencies (not devDependencies). Run npm install again if the node_modules folder is missing. Verify the import path matches your bundler requirements.

Conflicts with other frameworks

EaseMotion CSS uses @layer cascade layers. If conflicts persist, wrap your custom styles in a layer that loads after easemotion.components. For React/Vue, apply classes directly to the className or :class attribute.
