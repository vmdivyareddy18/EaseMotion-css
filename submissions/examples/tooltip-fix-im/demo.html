ease-tooltip Flicker Fix

Move your mouse rapidly back and forth over the buttons below. The tooltip stays stable and never flickers, because pointer-events: none prevents the tooltip itself from intercepting mouse events and re-triggering the hover state.

Before vs After

Before (buggy)

Tooltip lacks pointer-events: none — rapid mouse movement causes it to intercept events and toggle visibility in a loop, causing visible flicker.

After (fixed)

pointer-events: none on the tooltip means it never receives mouse events itself — only the trigger element controls visibility. No flicker, ever.

css — the fix
/* The critical fix: pointer-events: none on the tooltip itself.
   This prevents the tooltip from intercepting mouse events,
   which was causing a hide/show flicker loop on rapid movement. */

.ease-tooltip-trigger::before,
.ease-tooltip-trigger::after {
  pointer-events: none;
  opacity: 0;
  transition: opacity 0.18s ease, transform 0.18s ease;
}

.ease-tooltip-trigger:hover::before,
.ease-tooltip-trigger:hover::after {
  opacity: 1;
}
Root cause: Without pointer-events: none, when the cursor moves quickly across the trigger's edge, it can briefly land on the tooltip bubble itself. Since the bubble is not a descendant that inherits hover, the browser registers a "mouseleave" on the trigger followed immediately by "mouseenter" — causing opacity to toggle 0→1→0 rapidly, which is the visible flicker. pointer-events: none makes the tooltip completely transparent to the mouse, so only the trigger's hover state matters.
