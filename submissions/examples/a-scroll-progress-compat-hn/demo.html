Scroll Progress Compatibility Fix

Issue Fixed: Native CSS Scroll-driven animations are not supported on Safari < 18 and older Firefox/Chrome. Since the progress bar starts at `scaleX(0)`, it remains permanently hidden on those browsers.

This submission applies CSS feature queries to separate native animations and runs a lightweight scroll fallback listener when unsupported.

Section 1: The CSS Compatibility Gap

CSS scroll-driven animations introduce high performance by linking keyframes directly to viewport scroll timelines. However, because they are a newer standard, millions of active browsers (including older versions of Safari, iOS devices, and standard Firefox installations) do not recognize the animation-timeline: scroll() property.

Section 2: The Progressive Enhancement Fix

By defining transform: scaleX(0) as a static starting style and executing @supports (animation-timeline: scroll()), we isolate the animation binding to browsers that understand it natively. For other browsers, we load a minimal, passive JS scroll listener that updates the CSS scale factor.

Section 3: Optimal Performance

The fallback Javascript code runs efficiently by computing the scroll percentage of the viewport only. The progress bar continues to scroll cleanly on devices of all ranges, satisfying layout expectations across both modern and legacy browsers.

Section 4: Scroll Boundary Reached

Once you reach this point, the scroll bars at the top of the viewport should be fully loaded at 100% width. Scrolling back up will dynamically retract them back to 0.
