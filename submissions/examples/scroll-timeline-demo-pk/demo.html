Reading Progress Bar

The fixed bar at the top of the page fills from 0% to 100% as you scroll. Powered entirely by scroll-timeline — no JS scroll listener needed.

Browser support: Chrome 115+. In unsupported browsers, the progress bar falls back to a static 4px line.

Scroll-Reveal Cards

Each card fades + slides in as it enters the viewport using view-timeline. The animation plays over the "cover" range (0%–100% of the element crossing the viewport).

Implementation

These demos use scroll-timeline and view-timeline — native CSS APIs for scroll-driven animation. Below are the key patterns.

Reading progress bar (scroll-timeline)
@keyframes scroll-progress {
  from { width: 0%; }
  to   { width: 100%; }
}

.scroll-progress-bar {
  position: fixed; top: 0; height: 4px;
  background: var(--ease-color-primary, #6366f1);
  animation: scroll-progress linear forwards;
  scroll-timeline-name: --page-scroll;
  scroll-timeline-axis: block;
  animation-timeline: --page-scroll;
}
Scroll-reveal card (view-timeline)
@keyframes reveal {
  from { opacity: 0; translate: 0 60px; }
  to   { opacity: 1; translate: 0 0; }
}

.reveal-card {
  view-timeline-name: --card;
  view-timeline-axis: block;
  animation: reveal linear forwards;
  animation-timeline: view();
  animation-range: entry 0% entry 100%;
}
Parallax layers
.parallax-slow {
  scroll-timeline-name: --parallax;
  animation: parallax-drift linear;
  animation-timeline: --parallax;
}
@keyframes parallax-drift {
  from { translate: 0 0; }
  to   { translate: 0 -120px; }
}
