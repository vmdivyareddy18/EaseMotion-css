Animation Fill-Mode Documentation

How animation-fill-mode affects EaseMotion animation behavior

The animation-fill-mode CSS property controls how an element's styles are applied before and after an animation plays. Below is a breakdown of each value and when to use it with EaseMotion classes.

Class Fill-Mode Value Behavior Use Case
ease-fade-in both Element starts invisible, ends visible Entrance animations that should stay visible
ease-slide-up both Starts offset & invisible, ends at final position Scroll-reveal or mount animations
ease-bounce none No styles applied outside animation duration Continuous looping emphasis effects
ease-pulse none Snaps back to original state Repeating attention-seeking effects
ease-shake forwards Retains the final keyframe state Error indicators that should stay in final state
ease-rotate-in both Starts rotated + hidden, ends fully visible Element mount / reveal
ease-flip forwards Retains flipped state after animation ends Card flip interactions
ease-scale-in both Starts scaled down + hidden, ends natural size Modal / dialog entrance
