Fix: Undefined --ease-shadow-2xl Variable

components/modals.css references var(--ease-shadow-2xl, 0 25px 50px -12px rgba(0, 0, 0, 0.25)) but --ease-shadow-2xl is never defined in core/variables.css. This demo shows the modal shadow with the variable missing versus with it properly defined per-theme.

💡 Why this matters: Today the modal always renders the hardcoded fallback shadow in every theme. Once --ease-shadow-2xl is defined in core/variables.css for light and dark themes, the modal shadow correctly darkens in dark mode — consistent with every other shadow token (sm/md/lg/xl).

Before (variable undefined — fallback always used)

Modal title

This shadow never changes between light and dark mode because --ease-shadow-2xl is undefined.

After (variable defined — patch from style.css applied)

Modal title

With the patch applied, this shadow correctly darkens to rgba(0,0,0,0.50) in dark mode.

The exact patch needed in core/variables.css

:root {
  --ease-shadow-2xl: 0 25px 50px -12px rgba(0, 0, 0, 0.25);
}

@media (prefers-color-scheme: dark) {
  :root {
    --ease-shadow-2xl: 0 25px 50px -12px rgba(0, 0, 0, 0.50);
  }
}

[data-theme="dark"] {
  --ease-shadow-2xl: 0 25px 50px -12px rgba(0, 0, 0, 0.50);
}
