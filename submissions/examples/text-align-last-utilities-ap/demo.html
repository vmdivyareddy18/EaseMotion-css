Visual Layout Showcase

Observe the behavior of different text-align-last states on justified text paragraphs.

1. Centered Final Line .text-last-center

This paragraph is fully justified, but its final line is centered using the .text-last-center utility class. Centering the last line of a block adds a highly refined, symmetrical finish to abstract summaries, blockquotes, and featured article callouts. It prevents the last line from feeling awkwardly stuck to the left.

2. Justified Final Line .text-last-justify

By applying the .text-last-justify class, the browser forces the last line to stretch and distribute its word spacing so that it touches both the left and right margins, matching the rest of the paragraph. This creates a completely solid, rectangular block of text, which is popular in editorial prints and newspaper layouts.

3. Right-Aligned Final Line .text-last-right

Here, the final line is aligned to the right using .text-last-right. Pushing the last line to the right is highly effective for signature blocks, author bio stamps, or date-stamped summaries at the end of an editorial column, creating a natural separator right before the next content block.

4. Auto/Left Final Line .text-last-auto

This is the default browser behavior with .text-last-auto. Even though the body of the paragraph is fully justified, the final line defaults to the left (or the start of the reading direction). This is the standard presentation for regular reading content, as it maintains flow readability.

Technical Deep-Dive

Important browser specifications, legacy fallbacks, and rendering behaviors.

1. Layout Triggers

The text-align-last property describes alignment for the final line of a block (such as a <p> or <div>) and any line immediately preceding a forced line break (such as a <br> element). It does not affect other lines in the block.

2. Relationship with text-align

If text-align-last is set to its default value of auto, the alignment follows the block's main text-align rule, with one key exception: if text-align is set to justify, the last line is aligned to the start/left (instead of stretching), which prevents short trailing words from stretching excessively. Setting text-align-last: justify; explicitly overrides this safeguard.

3. Browser Compatibility & Safari Fallback

Modern desktop and mobile browsers fully support text-align-last. However, historical Safari versions required the prefixed -webkit-text-align-last rule. Today, native support is universal across Chrome, Firefox, Safari, and Edge.
