Pixel Art & Scaling Algorithms

When a small image (like pixel art) is displayed at a larger size, the browser must interpolate the missing pixels. The image-rendering CSS property controls this interpolation. pixelated uses nearest-neighbor sampling for sharp edges, crisp-edges preserves contrast with anti-aliasing, and auto uses smooth bicubic interpolation that blurs pixel art.

Browser Differences

Chrome treats pixelated and crisp-edges differently: pixelated uses nearest-neighbor and may show minor aliasing at certain scales, while crisp-edges also preserves contrast but with slight smoothing. Firefox treats both identically (nearest-neighbor). Safari supports crisp-edges but not pixelated. The auto value (bicubic) is consistent across all browsers.

/* Pixel art — keep it sharp */ .pixel-art { image-rendering: pixelated; image-rendering: crisp-edges; /* fallback */ } /* Photographic images — smooth interpolation */ .photo { image-rendering: auto; } /* UI icons at small sizes */ .icon { image-rendering: crisp-edges; }

Recommendation

For pixel art, use image-rendering: pixelated with crisp-edges as a fallback. For photos and illustrations, stick with the default auto. For UI icons at non-native sizes, crisp-edges provides a good balance.
