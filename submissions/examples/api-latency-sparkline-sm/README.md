# API Latency Sparkline

1. What does this do? Adds responsive API latency cards with animated sparkline bars, threshold colors, staggered card entry, and hover or focus feedback.
2. How is it used? Apply `.latency-grid` to the wrapper and use `.latency-card` with state classes like `.healthy`, `.warning`, or `.critical` for each service.
3. Why is it useful? It helps teams compare service response trends quickly with purposeful CSS-only motion and readable latency states.
