HTML Dialog Animation Fix

Understand why CSS animations fail to trigger on top-layer <dialog> elements and see the modern CSS @starting-style fix.

🚀 Open Dialog Modals

Compare the instant popup behavior of the standard dialog with the smooth transition of the fixed dialog:

Buggy Modal

This modal appears instantly because the browser moves it to the top-layer with display: block immediately, which bypasses normal entry CSS transitions.

Fixed Modal

This modal scales and fades in smoothly! By using modern CSS @starting-style, we define the starting visual frames of the entry state when the element transitions from display: none.

🛠️ Code Implementation

Use the new allow-discrete transition properties alongside @starting-style to animate elements entering the top-layer natively:

/* Define transition including display and overlay allow-discrete */
dialog {
  opacity: 0;
  transform: scale(0.92);
  transition: 
    opacity 0.4s ease,
    transform 0.4s ease,
    overlay 0.4s ease allow-discrete,
    display 0.4s ease allow-discrete;
}

/* Open State */
dialog[open] {
  opacity: 1;
  transform: scale(1);
}

/* Define starting style rules for entry state */
@starting-style {
  dialog[open] {
    opacity: 0;
    transform: scale(0.92);
  }
}
