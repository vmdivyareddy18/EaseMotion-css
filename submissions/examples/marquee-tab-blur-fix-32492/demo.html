Marquee Tab-Blur Fix

Browsers throttle CSS animations in background tabs. Without a fix, the marquee pauses while the tab is hidden and then jumps forward when you switch back. This submission eliminates that jump by using the Page Visibility API to synchronise animation-play-state with the tab's visibility.

🎨 The CSS

The core pattern: duplicate track content + half-width keyframe + paused state on hover.

.marquee-ticker {
  overflow: hidden; /* clips the scrolling track */
  display: flex;
  white-space: nowrap;
}

.marquee-ticker__track {
  display: inline-flex;
  animation: marquee-scroll 28s linear infinite;
  animation-play-state: running; /* JS overrides this */
  will-change: transform;
}

/* Intentional user pause on hover */
.marquee-ticker:hover .marquee-ticker__track {
  animation-play-state: paused;
}

@keyframes marquee-scroll {
  from { transform: translateX(0); }
  to   { transform: translateX(-50%); } /* two copies → seamless loop */
}

🔧 The JavaScript

A single visibilitychange listener pauses and resumes the animation without losing position.

document.addEventListener("visibilitychange", () => {
  // Query all marquee tracks on the page (supports multiple instances)
  const tracks = document.querySelectorAll(".marquee-ticker__track");

  tracks.forEach(track => {
    track.style.animationPlayState =
      document.hidden ? "paused" : "running";
  });
});
