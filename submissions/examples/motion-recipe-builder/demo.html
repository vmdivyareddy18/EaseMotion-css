Motion Recipe Builder

Combine multiple EaseMotion utility animations into reusable "recipes" using standard CSS Custom Properties. Standardize complex motion patterns across your application while staying true to composable design principles.

🧪 Motion Recipes ♻️ Reusable 🎨 CSS Variables

💡 The Concept: Instead of writing long strings of animation utilities on every element (class="ease-fade-up ease-scale ease-glow"), we define a "Recipe" in CSS using custom properties. A recipe bundles the animations, durations, and easing functions into a single reusable class like .recipe-card-hover.

3. Recipe Variations via CSS Variables

Because the recipes are built using CSS Custom Properties, you can easily tweak durations or easing functions on a per-element basis without writing new keyframes.

🔧 CSS Reference

/* 1. Define the primitive keyframes */
@keyframes ease-fade-up {
  from { opacity: 0; transform: translateY(20px); }
  to   { opacity: 1; transform: translateY(0); }
}
@keyframes ease-scale {
  from { transform: scale(1); }
  to   { transform: scale(1.05); }
}
@keyframes ease-glow {
  from { box-shadow: 0 0 0 transparent; }
  to   { box-shadow: 0 10px 25px rgba(108, 99, 255, 0.4); }
}

/* 2. Create the Recipe Class (Implementing the feature request) */
.ease-recipe-card {
  /* Recipe Parameters (Customizable per instance) */
  --recipe-duration: 300ms;
  --recipe-easing: ease-out;

  /* Base styling */
  background: #1e293b;
  border-radius: 12px;
  padding: 1.5rem;
  transition: all var(--recipe-duration) var(--recipe-easing);
}

/* 3. Apply the Recipe on Hover */
.ease-recipe-card:hover {
  /* Bundling multiple animations/transitions into one recipe! */
  animation:
    ease-fade-up var(--recipe-duration) var(--recipe-easing) forwards,
    ease-scale var(--recipe-duration) var(--recipe-easing) forwards,
    ease-glow var(--recipe-duration) var(--recipe-easing) forwards;
}
