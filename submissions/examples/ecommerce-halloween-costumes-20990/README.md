# App Landing: Seasonal Halloween Costumes E-commerce (`ecommerce-halloween-costumes-20990`)

This proposal provides a complete, copy-paste ready B2C landing page template designed for a fun, seasonal Halloween e-commerce store, built entirely with HTML/CSS and standard EaseMotion utility classes.

## 📌 Feature Overview

A seasonal Halloween store needs to project a bold, playful, and slightly spooky aesthetic. This template achieves a vibrant, high-energy feel using a color palette of deep purples, bright oranges, and slime greens, paired with a spooky display font (`Creepster`) and a chunky sans-serif (`Nunito`).

The layout includes:
- A CSS-animated flying bats overlay.
- An Announcement Bar and sticky Navigation with chunky "ghost" buttons.
- A Hero section featuring abstract CSS ghost/pumpkin shapes and floating animations.
- A "Who are you haunting?" 4-column category grid.
- A "Trending" product grid with "Hot/New" badges and solid shadow styling.
- A Split-layout interactive quiz/guide section.
- A Customer Reviews section.
- A high-contrast Newsletter block designed to look like a framed comic panel.

### Showcased Classes
The template leverages standard EaseMotion animation and interaction utilities, heavily modified locally via CSS to use a playful, bouncy, spring-like `cubic-bezier(0.34, 1.56, 0.64, 1)`:
- `.ease-slide-up` / `.ease-fade-up` (Used for UI elements to pop and bounce into place).
- `.ease-zoom-in` (Applied to product cards with a slight rotation to make them feel chaotic and fun).
- `.ease-float` / `.ease-pulse` (Continuous CSS keyframe animations applied to hero graphics to create ambient movement).
- `.ease-hover-pop` (Custom interaction class for nav links to scale up and bounce on hover).
- `.ease-hover-shake` (Continuous shaking animation triggered on button hover to mimic excitement or fear).

## ⚙️ How to Use

To view this full-page template locally, simply open the `demo.html` file in your web browser. 

The CSS handles the vibrant layout, the solid button shadows, the abstract CSS garment/costume simulations, and the ambient floating/flying animations. The standard `ease-*` interaction classes are attached directly to HTML elements to provide the bouncy entrance animations. The design is fully responsive and requires zero JavaScript.

*Note: As per the contributing guidelines, this proposal is implemented inside a unique `submissions/examples/ecommerce-halloween-costumes-20990/` directory to avoid modifying core files directly and to ensure zero deletions.*
