Text Overflow Utilities

Configure advanced representation of text boundaries when text exceeds the container borders using clip, ellipsis, custom characters, and line-clamp parameters.

Single-Line Truncation Modes

Clip Overflow

overflow-clip

Truncates text at the precise boundary pixel of the content box, leaving characters partially cut off without markers.

This is a long line of text that will be cut off by the clip utility class without showing ellipses.

Ellipsis Overflow

overflow-ellipsis

Appends standard horizontal ellipsis characters ("...") at the end of the line to indicate hidden/overflowing text.

This is a long line of text that will be cut off by the ellipsis utility class to show truncation.

Custom Dash

overflow-string-dash

Appends a custom string marker (such as "--") to represent character truncation (browser support varies, defaults to clip).

This is a long line of text that will be cut off by the custom dash utility class to show a custom string truncation.

Multi-Line Line Clamping

Combine overflow containment with WebKit box orientation parameters to restrict rendering to a specific number of lines before truncating.

Clamp 2 Lines

line-clamp-2

EaseMotion CSS provides clean and predictable utility scales. By applying the multi-line clamp parameters, block text will truncate exactly after two lines have been rendered. This enables developers to create uniform layouts on cards containing descriptions of varying lengths without disrupting the grid vertical flow.
