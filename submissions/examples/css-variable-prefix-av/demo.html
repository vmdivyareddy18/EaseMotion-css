CSS Custom Property Prefixing Refactor

This architectural proposal actively and completely eradicates global CSS namespace pollution by strictly enforcing the --ease-* prefix on every single design token across the entire framework.

❌ Legacy Unscoped Variables

Uses extremely generic, toxic variable names like --primary and --radius. If an enterprise user installs EaseMotion alongside Bootstrap, EaseMotion violently overwrites Bootstrap's global variables, completely destroying the user's existing website layout.

✅ Modern Prefixed Variables

Strictly enforces the --ease-* mathematical namespace (e.g. --ease-primary, --ease-radius). It is now physically and mathematically impossible for EaseMotion to collide with external libraries like Tailwind CSS or Bootstrap.
