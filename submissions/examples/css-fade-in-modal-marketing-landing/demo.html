Customization

Tune it with five CSS variables

Every modal is driven entirely by custom properties — override them globally on :root, or per-modal on the #modal-id selector, as shown in the variations above.

Variable Purpose
--modal-duration Speed of the fade/scale transition (e.g. 0.4s).
--modal-radius Corner rounding of the modal card.
--modal-overlay Background color of the soft, blurred overlay backdrop.
--modal-shadow Elevation shadow applied to the modal card.
--modal-scale Starting scale value the card animates up from (e.g. 0.92).
/* Override per modal instance */ #custom-modal { --modal-duration: 0.4s; --modal-radius: 20px; --modal-overlay: rgba(30,27,75,.55); --modal-shadow: 0 30px 60px rgba(76,29,149,.35); --modal-scale: 0.92; }
Accessibility

Built for every visitor

Marketing modals are notorious for trapping keyboard and screen-reader users. This component avoids that entirely.

⌨ Keyboard Accessible

Every trigger and close control is a native anchor element, fully operable with Tab and Enter.

🔆 Visible Focus States

A high-contrast focus ring appears on every interactive element via :focus-visible.

🏷 Semantic HTML

role="dialog", aria-modal, and aria-labelledby give assistive tech a clear landmark.

🎯 Touch Targets

Buttons and close icons meet a 44px minimum hit area for comfortable mobile tapping.

🌓 High Contrast

Text and CTA colors are tuned to meet WCAG AA contrast against their backgrounds.

🧘 Reduced Motion

prefers-reduced-motion instantly disables scale and fade transitions.
