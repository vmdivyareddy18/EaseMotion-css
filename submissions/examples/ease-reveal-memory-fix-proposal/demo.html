Architectural Proposal: GPU Memory Fix

Resolves issue #17467. This demonstrates how we should refactor .ease-reveal to prevent massive GPU compositor layer bloat on pages with many elements.

❌ Current Architecture

Currently, .ease-reveal applies will-change: transform, opacity permanently upon page load. If you have 500 cards on a page, the browser pre-allocates 500 GPU layers instantly, crashing low-end devices.

Legacy Element

✅ Proposed Architecture

We remove will-change from the base class. Instead, JavaScript (like reveal.js) adds a .ease-reveal-pending class just before the element enters the viewport to trigger the GPU promotion exactly when needed.
