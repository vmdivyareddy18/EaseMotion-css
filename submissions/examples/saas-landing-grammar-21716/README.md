# Grammar & Writing Assistant SaaS Landing (`saas-landing-grammar-21716`)

This proposal provides a complete, copy-paste ready B2B/B2C SaaS landing page template designed specifically for Writing and Grammar Assistant software (similar to Grammarly or ProWritingAid), utilizing pure HTML/CSS and standard EaseMotion utility classes.

## 📌 Feature Overview

A modern writing assistant tool needs to visually demonstrate clarity, inline error correction, and tone detection. This template achieves a clean, professional aesthetic using a crisp sans-serif font, bright green/blue accents for corrections, and custom CSS animations that simulate a real-time editor experience.

The layout includes:
- A high-impact hero section featuring an animated CSS-built document editor mockup with inline error underlines and pop-up correction suggestions.
- A platform integration logo strip.
- A clear Before/After comparison section to demonstrate value.
- A 4-item feature grid highlighting specific writing capabilities.
- A use-cases section featuring an animated "Tone Detection" meter.
- A 3-tier pricing structure with a highlighted "Most Popular" card.
- A comprehensive footer with a final strong Call-to-Action.

### Showcased Classes
The template leverages the following EaseMotion animation and interaction utilities:
- `.ease-pop` (Used on the inline error underlines to draw attention when they are "detected")
- `.ease-slide-up` / `.ease-fade-up` (For smooth, staggered text, cards, and correction popup entrances)
- `.ease-expand` (Simulates the animated filling of the tone detection bars)
- `.ease-pulse` (Used on the "Checking..." status indicator)
- `.ease-spin` (Applied to the overall score circle to simulate calculation)
- `.ease-hover-lift` (Standard interaction for primary buttons and pricing cards)
- `.ease-hover-bg` (Used for interactive correction suggestions in the popups)

## ⚙️ How to Use

To view this full-page template locally, simply open the `demo.html` file in your web browser. 

The CSS handles the layout, typography, and styling, while standard `ease-*` interaction classes are attached directly to HTML elements to provide the animations. The document editor mockup, complete with colored error underlines and hovering correction suggestions, is built entirely with structural HTML and CSS, requiring zero JavaScript.

*Note: As per the contributing guidelines, this proposal is implemented inside a unique `submissions/examples/saas-landing-grammar-21716/` directory to avoid modifying core files directly and to ensure zero deletions.*
