Motion Breakdown

The animation unfolds in three distinct phases, each contributing to the elastic feel.

1

Initial Position

The element starts far off-screen to the left at translateX(-200%). This extreme offset ensures the element is completely invisible before the animation begins.

-200%
2

Overshoot Phase

The element accelerates rapidly to translateX(8%), passing beyond its final resting place. This overshoot is what makes the animation feel energetic and elastic, as if the element has momentum.

+8%
3

Settling Phase

From the overshoot, the element pulls back slightly to translateX(-3%), then settles gracefully at translateX(0). This back-and-forth decay creates the signature elastic settling effect.

0

Linear keyframes inside @keyframes are combined with a cubic-bezier(0.25, 0.5, 0.35, 1.5) easing curve, where the overshoot in the easing’s control points amplifies the elastic feel.

Use Cases

High-energy elastic slide-ins bring attention to important UI elements.

Notifications

Toast alerts and notification banners can slide in with elastic energy to draw immediate user attention without being intrusive.

Dashboard Widgets

Newly loaded dashboard panels can enter with elastic motion to signal fresh data, adding life to data-dense interfaces.

Hero Content

Headlines, call-to-action buttons, and hero banners benefit from elastic reveals that create a memorable first impression.

Promotional Banners

Time-sensitive offers and announcements on e-commerce or marketing sites gain urgency with a fast, elastic entrance.

Keyframes

The @keyframes definition that powers the elastic slide-in effect. 

@keyframes elastic-slide-in {
  0% {
    transform: translateX(-200%);
  }
  70% {
    transform: translateX(8%);
  }
  85% {
    transform: translateX(-3%);
  }
  100% {
    transform: translateX(0);
  }
}
animation elastic-slide-in 0.9s cubic-bezier(0.25, 0.5, 0.35, 1.5) forwards
Easing cubic-bezier(0.25, 0.5, 0.35, 1.5) — the y-value exceeds 1.0 for overshoot feel
