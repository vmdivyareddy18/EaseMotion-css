Naming Convention Guide

Fixes #8389 — documents the canonical naming pattern for all CSS constructs in EaseMotion-css.

1. Utility Classes .ease-{effect}-{variant}

Fade In .ease-fade-in
Slide Up .ease-slide-up
Zoom In .ease-zoom-in
Tada! .ease-tada

2. Keyframe Names ease-kf-{effect}

✅ Correct❌ Never useWhy
ease-kf-fade-in fadeIn Collides with Animate.css
ease-kf-slide-up FadeIn PascalCase — collides with other libs
ease-kf-bounce em-bounce Wrong prefix
ease-kf-tada tada_kf Underscore + reversed order

3. Modifier Classes .ease-{effect}-{modifier}

Fast .ease-tada-fast
Slow .ease-tada-slow
Loop .ease-bounce
Hover me .ease-tada-hover

4. CSS Tokens --ease-{category}-{name}

✅ Correct❌ Never useWhy
--ease-color-primary --primary No prefix — global collision
--ease-duration-normal --easeColorPrimary camelCase
--ease-radius-md --ease_radius_md Underscores
--ease-shadow-focus --em-shadow Wrong prefix
