Tooltip Hover Touch Guard Fix

.ease-tooltip:hover opens tooltip on tap and sticks — missing @media (hover: hover) guard

How to reproduce: Chrome DevTools → Device Toolbar → select any mobile preset → tap the buttons in the Broken section. The tooltip opens and stays open. The Fixed tooltip does not open on tap, but still opens via keyboard focus.
Broken No (hover: hover) guard — tooltip sticks on tap

On touch: tapping either button opens the tooltip and it stays visible until you tap elsewhere. The synthetic :hover from the tap never clears on lift — it is stuck until focus moves.

Fixed Hover guarded + :focus-within preserved for keyboard

On touch: tapping does not open the tooltip — hover is now guarded. On desktop: hovering still shows the tooltip. Via keyboard: pressing Tab to focus the element opens the tooltip via :focus-within — intentionally preserved on all devices.

Live Real .ease-tooltip classes with fix applied

These use the real .ease-tooltip class. style.css overrides the :hover rules with the pointer guard. On touch they will not open on tap. On desktop they behave exactly as before.
