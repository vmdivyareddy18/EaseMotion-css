Multi-Column Text

CSS column-count layouts for magazine-style text with drop caps and column rules

Two Columns

CSS multi-column layout is a powerful yet often overlooked feature of modern CSS. It allows content to flow naturally into multiple columns, similar to newspaper or magazine layouts. The browser automatically handles the distribution of content across columns, ensuring balanced filling and proper reading order.

The column-count property specifies the number of columns. Content flows from one column to the next as it fills each one. Column gaps and rules provide visual separation between columns, creating a polished editorial look without complex grid markup.

Three Columns

Three column layout is ideal for dense text-heavy content like documentation or feature overviews. The narrower column width improves readability by keeping line length optimal for quick scanning of information across multiple categories.

CSS columns work with any HTML content including headings, lists, and images. The column-span property allows certain elements to break out across all columns for visual hierarchy and emphasis within the flow.

Column-fill controls how content is distributed. The default auto fill fills columns sequentially while balance attempts to equalize column heights for a more uniform appearance across the layout.

With Column Rule

Column rules add a visible divider between columns. The column-rule shorthand works like border: width, style, and color. Here we use a dashed purple line that adds visual separation while maintaining the editorial aesthetic of the layout.

The column-gap property controls the space between columns. Combined with column-rule, you can fine-tune the separation to balance readability with visual density for different content types and screen sizes.

Mixed Column Widths

Using column-width instead of column-count creates a responsive multi-column layout. The browser automatically adjusts the number of columns based on available space, ensuring each column maintains a minimum width for readability at any viewport size.

This responsive approach is particularly useful for card grids and article previews where content density should adapt naturally to screen width without media query breakpoints for each column count.
