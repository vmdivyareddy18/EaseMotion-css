modal.js Overflow Restore Fix

body.style.overflow = '' on modal close erases any pre-existing inline overflow value

How to use this demo: Each section simulates the scenario step by step. Click through the steps in order to see what body.style.overflow holds at each stage — and whether it survives the modal closing.
Broken Current behaviour — overflow = '' erases the sidebar value

Scenario: a sidebar toggle sets body.style.overflow = 'hidden' to lock scroll. Then a modal opens (also sets overflow = 'hidden'). When the modal closes, the current code runs body.style.overflow = '' — clearing the sidebar's lock too.

Fixed Proposed fix — saves and restores the previous overflow value

Same scenario. The fix saves body.style.overflow before locking it, and restores that saved value when the modal closes — instead of always resetting to ''.

The fix for core/modal.js
// Save the overflow value BEFORE locking — add this near the top of checkModal()
- body.style.overflow = 'hidden';
+ var previousOverflow = body.style.overflow;
+ body.style.overflow = 'hidden';

// Restore the saved value on close — replace the reset line
- body.style.overflow = '';
+ body.style.overflow = (typeof previousOverflow !== 'undefined') ? previousOverflow : '';
