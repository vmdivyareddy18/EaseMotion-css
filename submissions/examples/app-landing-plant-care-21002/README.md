# App Landing: Plant Care Reminder (`app-landing-plant-care-21002`)

This proposal provides a complete, copy-paste ready B2C landing page designed for a lifestyle/plant care mobile app, built entirely with HTML/CSS and standard EaseMotion utility classes.

## 📌 Feature Overview

A plant care application needs to project growth, organic life, and calm. This template achieves a "lush green" aesthetic using Deep Forest Green (`#2B5C3B`), Soft Mint (`#E8F3E9`), and Terracotta (`#D97B54`) accents. The typography uses the organic serif `Lora` for headings paired with the clean, modern `Outfit` sans-serif for UI elements.

The layout includes:
- A Hero section featuring a detailed CSS mobile app mockup (showing watering reminders) framed by CSS-drawn abstract organic leaves.
- A 3-column features grid outlining smart watering, light meters, and plant ID.
- A split layout showcasing a "Library" with scattered, overlapping CSS plant cards.
- A high-impact "Community" statistics box.
- A clear 2-tier subscription architecture (Sprout vs Jungle tier).
- "Loved by plant parents" review grid.
- Bold App Store download Call-To-Action (CTA) sections.

### Showcased Classes
The template leverages the following EaseMotion animation and interaction utilities, configured with a gentle, organic bezier curve:
- `.ease-fade-up` / `.ease-slide-up` (Configured with a soft, natural `cubic-bezier(0.25, 1, 0.5, 1)` to mimic organic growth)
- `.ease-zoom-in` (Applied to the background leaf blobs and the floating library cards)
- `.ease-slide-left` / `.ease-slide-right` (Used for sequential loading of plant task cards inside the CSS phone mockup)
- `.ease-hover-lift` (Standard soft interaction for buttons and feature cards)
- `.ease-hover-scale` (Applied to the logo and social links)

## ⚙️ How to Use

To view this full-page template locally, simply open the `demo.html` file in your web browser. 

The CSS handles the layout, the phone mockup geometry, the scattered library cards, the organic background leaves, and the typography. The standard `ease-*` interaction classes are attached directly to HTML elements to provide the entrance animations. The design is fully responsive and requires zero JavaScript to execute the entrance and hover effects.
