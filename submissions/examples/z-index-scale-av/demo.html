Z-Index Token Scale Refactor

This architectural proposal actively eliminates catastrophic UI overlap bugs by physically ripping out brittle z-index magic numbers and completely replacing them with a mathematically strict, framework-wide global CSS token scale.

❌ Legacy Magic Numbers

The Navbar dev arbitrarily chose z-index: 999. The Modal dev arbitrarily chose z-index: 100. Now the Navbar violently cuts through the Modal, completely destroying the user interface!

Legacy Navbar (z-index: 999)

Broken Modal (z-index: 100)

I am violently trapped underneath the navbar because of highly dangerous magic numbers!

✅ Modern Z-Index Tokens

Uses a mathematically strict global hierarchy. The Navbar uses var(--ease-z-sticky) and the Modal uses var(--ease-z-modal). The Modal perfectly renders on top, mathematically guaranteed 100% of the time.

Modern Navbar (z-index: var(--ease-z-sticky))

Perfect Modal (z-index: var(--ease-z-modal))

I flawlessly render completely above the navbar natively thanks to the strict token scale!
