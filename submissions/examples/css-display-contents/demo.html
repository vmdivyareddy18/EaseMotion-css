Understanding display: contents

The wrapper cards below are semantically grouped inside an <article> tag, but display: contents forces the browser to ignore the wrapper container visually, allowing the child items to align perfectly to the main Flexbox layout grid.

Card 1

This item aligns properly because its parent container is hidden from the layout engine.

Card 2

Without display: contents, this wrapper would break the main flexbox alignment.
