CSS Cascade Layer Utility Packs

Organize EaseMotion CSS utilities into separate @layer groups (base, utilities, components, animations) to manage specificity and make overriding styles predictable.

🥞 @layer ⚖️ Specificity 🧱 Architecture

💡 What are Cascade Layers? The @layer CSS rule allows you to define explicit layers of specificity. Instead of relying on selector weight (e.g. #id beats .class) or source order, you declare the order of your layers up front. A style in a higher-priority layer will always win against a style in a lower-priority layer, regardless of selector specificity.

1. The Layer Order

We define the layer order up front. In this example, the order is: base → components → utilities → animations. Layers defined later have higher priority. 

/* Declare layer order (highest priority last) */
@layer easemotion.base,
       easemotion.components,
       easemotion.utilities,
       easemotion.animations;

2. Solving the Specificity Problem

In traditional CSS, an ID selector (#unique-button) beats a class selector (.ease-bg-red). With layers, we can put utility classes in a higher-priority layer than components, so utilities always win, even against IDs.

❌ Without Layers

The ID (#traditional-btn) defines a blue background. The utility class (.util-bg-red) tries to make it red. Because IDs have higher specificity, the utility fails.

✅ With @layer

The ID is placed in the components layer. The utility class is placed in the utilities layer. Because utilities is ordered after components, the utility wins!

🔧 CSS Reference

/* 1. Define the order (matches the issue snippet exactly) */
@layer easemotion.base,
       easemotion.components,
       easemotion.utilities,
       easemotion.animations;

/* 2. Base Layer (Resets, typography) */
@layer easemotion.base {
  body { font-family: system-ui, sans-serif; }
}

/* 3. Components Layer (Complex UI patterns) */
@layer easemotion.components {
  .ease-card {
    background: #1e293b;
    border-radius: 12px;
    padding: 1.5rem;
  }
}

/* 4. Utilities Layer (Overrides components) */
@layer easemotion.utilities {
  .ease-bg-purple { background: #7c3aed; }
  .ease-bg-red    { background: #ef4444; }
}

/* 5. Animations Layer (Overrides utilities) */
@layer easemotion.animations {
  .ease-fade {
    animation: fadeIn 1s ease-in-out infinite alternate;
  }
}
