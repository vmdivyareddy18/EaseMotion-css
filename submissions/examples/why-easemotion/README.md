# Why EaseMotion CSS?

EaseMotion CSS is designed to bridge the gap between heavy animation libraries and rigid utility-first frameworks.

## The Problem
Writing custom CSS animations is time-consuming and prone to browser-compatibility issues. Utility-first frameworks offer speed but often bloat the DOM with hundreds of classes.

## Our Approach
- **Animation-First:** Motion is built into the core, not added as an afterthought.
- **Human-Readable:** Class names describe the movement, not the style.
- **Lightweight:** No heavy JavaScript runtimes; it's just pure, optimized CSS.

## Key Differentiators
| Feature | Utility Frameworks | Animation Libraries | EaseMotion CSS |
| :--- | :--- | :--- | :--- |
| **Animation focus** | Low | High | High |
| **Bundle Size** | Large | Heavy JS | Tiny (CSS only) |
| **Learning Curve** | Moderate | Steep | Low |

## Target Audience
- **Portfolio Builders:** Who want high-end motion without JS.
- **Rapid Prototypers:** Who need quick, beautiful UI transitions.
- **Beginners:** Learning the power of CSS keyframes.
