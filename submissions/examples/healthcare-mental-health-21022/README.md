# Healthcare: Mental Health App for Teens (`healthcare-mental-health-21022`)

This proposal provides a complete, copy-paste ready B2C healthcare landing page designed for a youth/teen mental health application, utilizing pure HTML/CSS and standard EaseMotion utility classes.

## 📌 Feature Overview

A teen mental health platform needs to strike a delicate balance: it must feel safe, highly approachable, and non-clinical, while still reassuring parents of its security and legitimacy. This template achieves a supportive, age-appropriate aesthetic using a friendly rounded font (Quicksand), a bright pastel color palette (soft purples, blues, oranges, greens), and soft, bouncy CSS animations.

The layout includes:
- A welcoming hero section featuring abstract CSS blur shapes and a simulated phone mockup with interactive emoji mood-tracking.
- A colorful, rounded 2x2 feature grid.
- A split-layout "How it Works" section featuring a CSS-animated secure connection line between a parent and teen avatar.
- A high-contrast, dark-themed Safety & Privacy box to communicate trust and encryption.
- A dedicated resource banner for parents and school counselors.
- App store download CTAs and a mandatory 988 Crisis lifeline footer block.

### Showcased Classes
The template leverages the following EaseMotion animation and interaction utilities:
- `.ease-fade-up` / `.ease-slide-up` (For gentle entrances of friendly text blocks and feature cards)
- `.ease-zoom-in` (Configured with a custom spring/bounce cubic-bezier to make the phone mockup entrance feel playful and tactile)
- `.ease-hover-scale` (Applied heavily to the mood tracking emojis and App Store buttons to encourage interaction)
- `.ease-hover-lift` (Standard interaction for primary buttons to provide a soft, floating feel)
- `.ease-slide-left` / `.ease-slide-right` (Used in the 'How it Works' section to guide the user's eye through the parent/teen setup flow)

## ⚙️ How to Use

To view this full-page template locally, simply open the `demo.html` file in your web browser. 

The CSS handles the layout, rounded-corner aesthetics, and color-coded feature areas, while standard `ease-*` interaction classes are attached directly to HTML elements to provide the animations. The design is fully responsive and requires zero JavaScript to execute the entrance and hover effects.

*Note: As per the contributing guidelines, this proposal is implemented inside a unique `submissions/examples/healthcare-mental-health-21022/` directory to avoid modifying core files directly and to ensure zero deletions.*
