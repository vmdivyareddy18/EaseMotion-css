Print Stylesheet Guide

How to disable EaseMotion animations for printed output

When printing a page that uses EaseMotion animations, you should disable animations to prevent elements from being cut off mid-animation or appearing invisible. The recommended approach is a dedicated @media print stylesheet that disables all animations and ensures content is fully visible.

💡 Tip: Add your print stylesheet after EaseMotion so it takes precedence, or use the same specificity level. 
/* print.css — include after easemotion.css */
@media print {
  /* Disable all EaseMotion animations */
  [class*="ease-"] {
    animation: none !important;
    transition: none !important;
    transform: none !important;
    opacity: 1 !important;
  }

  /* Ensure hidden elements are visible */
  [class*="ease-fade"],
  [class*="ease-slide"],
  [class*="ease-scale"] {
    opacity: 1 !important;
    transform: none !important;
  }

  /* Remove scroll-driven animations */
  * {
    animation-timeline: none !important;
  }

  /* Hide non-essential decorative elements */
  .ease-bg-dots,
  .ease-bg-noise,
  .ease-bg-grid {
    display: none !important;
  }
}

Recommended approach: Create a separate print.css file and include it with media="print" on the link tag. This keeps your print styles isolated and only loads them when the user prints.

<!-- Include EaseMotion normally -->
<link rel="stylesheet" href="easemotion.min.css">

<!-- Override for print -->
<link rel="stylesheet" href="print.css" media="print">
