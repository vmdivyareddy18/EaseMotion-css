Container Query Animation Breakpoints

Animate based on container size, not viewport. Reusable components adapt to their parent. Chrome 105+

⚡ Requires Chrome 105+. Falls back to viewport-based animation in other browsers.

Narrow Container (< 480px)

The card fades in when inside a narrow parent.

Fade In

Medium Container (481–768px)

The card slides up when inside a medium parent.

Slide Up

Wide Container (> 768px)

The card scales in when inside a wide parent.

Scale In

Interactive: Resize the Container

Drag the handle below to resize this container. The card's animation changes based on container width.

✦

Adaptive Card

Animation changes as the container shrinks or grows.

> 768px → Scale In

Proposed Utilities

core/utilities.css
.ease-container {
  container-type: inline-size;
  container-name: ease;
}

@container ease (max-width: 480px) {
  .ease-container-sm\:fade-in {
    animation: ease-kf-fade-in 0.4s ease both;
  }
}

@container ease (min-width: 481px) and (max-width: 768px) {
  .ease-container-md\:slide-up {
    animation: ease-kf-slide-up 0.4s ease both;
  }
}

@container ease (min-width: 769px) {
  .ease-container-lg\:scale-in {
    animation: ease-kf-zoom-in 0.4s ease both;
  }
}
Usage
<div class="ease-container">
  <div class="ease-container-sm:fade-in
              ease-container-md:slide-up
              ease-container-lg:scale-in">
    Animates based on container width
  </div>
</div>

Viewport Fallback

For browsers without container query support, use regular media queries as fallback:

@supports not (container-type: inline-size) {
  @media (max-width: 480px) {
    .cq-fade-in { animation: ease-kf-fade-in 0.4s ease both; }
  }
  @media (min-width: 481px) and (max-width: 768px) {
    .cq-slide-up { animation: ease-kf-slide-up 0.4s ease both; }
  }
  @media (min-width: 769px) {
    .cq-scale-in { animation: ease-kf-zoom-in 0.4s ease both; }
  }
}
