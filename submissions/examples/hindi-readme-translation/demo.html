English

What is EaseMotion CSS?

EaseMotion CSS is a lightweight, utility-first animation library built entirely in CSS. It gives you a consistent set of motion primitives — hover effects, entrance animations, transitions, and micro-interactions — without any JavaScript dependency.

Every class is prefixed with ease- to keep your stylesheet clean and predictable.

हिन्दी

EaseMotion CSS क्या है?

EaseMotion CSS एक हल्की, यूटिलिटी-फर्स्ट एनिमेशन लाइब्रेरी है जो पूरी तरह CSS में बनी है। यह आपको बिना किसी JavaScript निर्भरता के — hover इफ़ेक्ट, प्रवेश एनिमेशन, ट्रांज़िशन और माइक्रो-इंटरैक्शन — एक सुसंगत सेट प्रदान करती है।

हर क्लास को ease- से प्रीफ़िक्स किया जाता है ताकि आपकी stylesheet साफ़ और अनुमानित बनी रहे।

Philosophy

  • Motion should feel natural, not mechanical.
  • Animations are enhancements, not distractions.
  • Every utility must be composable with others.
  • Zero JavaScript. Pure CSS only.
  • Consistent naming keeps teams aligned.

Installation

Link the stylesheet in your HTML <head>:

<link rel="stylesheet"
  href="easemotion.min.css" />

Or import it in your CSS entry file:

@import "easemotion.css";

English

Quick Usage

Add any ease-* class directly to your HTML element. No configuration needed. 

<button class="ease-hover-grow">
  Hover me
</button>

<div class="ease-fade-in">
  I appear smoothly
</div>

<card class="ease-lift-shadow">
  I rise on hover
</card>
हिन्दी

त्वरित उपयोग

Contributing

All contributions live in submissions/examples/. Modifications to core/ or components/ will be closed without review.

Each submission requires three files:

  • demo.html — self-contained demo
  • style.css — raw CSS for the feature
  • README.md — what, how, and why

Open a Feature Request issue before building anything large. Small animations rarely need pre-approval.

हिन्दी

योगदान (Contributing)

सभी योगदान submissions/examples/ में होते हैं। core/ या components/ में बदलाव बिना समीक्षा के बंद कर दिए जाएंगे।

प्रत्येक submission में तीन फ़ाइलें आवश्यक हैं:

  • demo.html — स्व-निहित डेमो
  • style.css — फ़ीचर के लिए raw CSS
  • README.md — क्या, कैसे, और क्यों

कुछ बड़ा बनाने से पहले एक Feature Request issue खोलें। छोटे एनिमेशन के लिए पूर्व-अनुमोदन की आमतौर पर ज़रूरत नहीं।

English

Naming Rules

You do not need to use the ease- prefix in your submission. Use any class name that communicates intent — .hover-glow, .card-lift, .shimmer-btn.

The maintainer renames everything to ease-kebab-case before merging and replaces hard-coded values with CSS variables.

हिन्दी

नामकरण नियम

आपको अपनी submission में ease- प्रीफ़िक्स नहीं लगानी है। कोई भी ऐसा क्लास नाम उपयोग करें जो इरादा स्पष्ट करे — .hover-glow, .card-lift, .shimmer-btn।

मेंटेनर merge से पहले सब कुछ ease-kebab-case में नाम बदलता है और hard-coded values को CSS variables से बदलता है।
