Scroll Progress Indicator

Pure CSS scroll-driven reader progress tracker

How It Works

Uses CSS animation-timeline: scroll() to link animation progress to the document scroll position. The progress bar width transitions from 0% to 100% as you scroll from top to bottom. No JavaScript needed - the browser compositor handles all animation work, ensuring smooth 60fps performance even on lower-end devices. This is part of the CSS Scroll-Driven Animations specification supported in Chromium-based browsers.

Scroll-Driven Animations

CSS scroll-driven animations allow animations to be tied to the scroll position of a scroll container. Instead of playing based on time, the animation progress is determined by how far the user has scrolled. This feature is part of the CSS Scroll-Driven Animations specification.

Performance Benefits

Since scroll-driven animations run on the compositor thread rather than the main thread, they can maintain 60fps even when JavaScript is busy. This makes them ideal for scroll-linked effects. Unlike JavaScript-based scroll listeners which trigger layout recalculations, CSS scroll animations are fully GPU-accelerated.

Accessibility

The component respects prefers-reduced-motion by showing a full-width static gradient bar when users prefer reduced motion. Always test scroll-based animations with assistive technologies to ensure content remains accessible.

Customization Options

Customize via CSS variables: --sp-height for bar thickness, --sp-start and --sp-end for gradient colors, --sp-z for stacking context. Position the bar at top, bottom, or even vertically along the side edge of the viewport.

Use Cases

Blog posts and articles, documentation pages, e-books and tutorials, presentation slides, and any long-form content where users benefit from knowing their reading position. Progress indicators improve user engagement by providing clear navigation context throughout the page.

Technical Details

The @keyframes animation transitions width from 0% to 100%. The scroll timeline maps animation progress to scroll position. The bar uses position: fixed for viewport-anchored display. Uses transform: scaleX() for GPU-accelerated animation rather than animating width directly.

Browser Support

Chromium 115+ supports scroll-driven animations. Firefox and Safari have the feature in development. The component degrades gracefully in non-supporting browsers by showing a static bar. Use feature detection with @supports (animation-timeline: scroll()).

EaseMotion Integration

This component follows EaseMotion lightweight performant CSS philosophy. It can be combined with other EaseMotion utilities for enhanced effects. The progress bar gradient can match your brand colors through simple CSS variable overrides.
