EaseAccordion

A zero-dependency, highly accessible accordion component animating height from 0 to auto smoothly using pure CSS grid template rows.

Historically, transitioning elements to `height: auto` has been a classic constraint in CSS. Because the browser does not know the exact pixel height of the target content before rendering it, it cannot interpolate or compute transition values between a fixed number (like `0px`) and an intrinsic keyword value (like `auto`).

Instead of transitioning `height` directly, this component sets the wrapper to a CSS Grid container (`display: grid`) and transitions the `grid-template-rows` property: 

.ease-accordion-content {
  display: grid;
  grid-template-rows: 0fr;
  transition: grid-template-rows 300ms ease;
}
.ease-accordion-trigger[aria-expanded="true"] + .ease-accordion-content {
  grid-template-rows: 1fr;
}

By using `0fr` to `1fr` inside a grid container, the browser handles the layout calculations natively and interpolates the row size from 0% to 100% of the child's minimum content height smoothly.

JavaScript is only used to manage accessibility state by toggling the `aria-expanded` boolean attribute (and `.is-open` class fallback) on user click. Unlike traditional solutions, no JavaScript is used to calculate scroll heights, measure DOM elements, or assign hardcoded styling offsets, avoiding layout thrashing.

When the OS accessibility setting `prefers-reduced-motion: reduce` is detected, the transition styling is disabled. The panels will expand and collapse instantly, ensuring a comfortable reading experience.
