Customization

Tune it with five CSS variables

Every modal is driven entirely by custom properties — override them globally on :root, or per-modal on the #modal-id selector, as shown in the variations above.

Variable Purpose
--modal-scale Starting scale value the card animates up from (e.g. 0.9).
--modal-duration Speed of the scale/fade transition (e.g. 0.3s).
--modal-radius Corner rounding of the modal card.
--modal-shadow Elevation shadow applied to the modal card.
--modal-overlay Background color of the soft, blurred overlay backdrop.
/* Override per modal instance */ #custom-modal { --modal-scale: 0.9; --modal-duration: 0.3s; --modal-radius: 18px; --modal-shadow: 0 30px 60px rgba(15,23,42,.25); --modal-overlay: rgba(15,23,42,.45); }
Accessibility

Interfaces that work for everyone

Modal-heavy dashboards are hard to navigate without careful keyboard and screen-reader support. This component gets it right.

⌨ Keyboard Accessible

Every trigger, toggle, and close control is operable with Tab and Enter/Space.

🔆 Visible Focus

A high-contrast focus ring appears on every interactive element via :focus-visible.

🏷 Semantic HTML

role="dialog", aria-modal, and aria-labelledby give assistive tech a clear landmark.

🎯 Large Touch Targets

Buttons, toggles, and action tiles meet a 44px+ minimum hit area for confident tapping.

🌓 High Contrast

Text and CTA colors are tuned to meet WCAG AA contrast against their backgrounds.

🧘 Reduced Motion

prefers-reduced-motion instantly disables scale and fade transitions.
