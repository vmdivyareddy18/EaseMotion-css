column-width — Responsive Multi-Column

Browser calculates column count based on available width and minimum column size.

column-width: 200px — Auto Columns

As the viewport widens, more columns appear automatically. Each column is at least 200px wide. This is the most responsive multi-column approach — no media queries needed for column count.

Column-width tells the browser: "make columns this wide if possible". The browser then fits as many columns as it can into the available space, adjusting automatically when the container size changes.

This property is great for card grids, image galleries, and long-form text that should adapt to screen size without manual breakpoints.

Combine with max-width on the container to prevent excessively wide columns that are hard to read.

column-width: 150px — Denser Columns

With a smaller minimum width of 150px, more columns appear. This creates a denser, more magazine-like layout with shorter line lengths.

Column-width defines the optimal width. The browser may stretch columns slightly to fill space, but they won't shrink below this value.

Combined: columns: 250px 3

The shorthand "columns" property accepts both column-width and column-count. Here we set columns: 250px 3, meaning: aim for 250px columns, but never show more than 3.

This gives you responsive behavior with a hard upper limit on column count, preventing too many skinny columns on very wide screens.

It's the best of both worlds — responsiveness with guardrails.

column-width + column-count = responsive multi-column without media queries.
