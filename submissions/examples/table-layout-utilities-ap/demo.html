Technical Deep-Dive

Understanding rendering performance and layout algorithms under the hood.

1. Automatic Layout (2-Pass Rendering)

In table-layout: auto, the browser must read the content of every single cell in a column before it can determine that column's final width. This requires a two-pass rendering cycle: the first pass measures content sizes, and the second pass positions cells. For large tables with thousands of rows, this can cause significant rendering jank and increase page load times.

2. Fixed Layout (1-Pass Rendering)

In table-layout: fixed, the browser determines column sizes based solely on the table's width and the widths specified in the first row of cells (usually <th> or <col>). It ignores the actual contents of the cells in subsequent rows, allowing a one-pass rendering cycle. The browser can start rendering rows instantly as they stream in, which is ideal for performance on large databases.

3. Requirements for Ellipsis Truncation

To truncate a cell's text using text-overflow: ellipsis;, you must meet three strict conditions:

  • The table layout algorithm must be set to fixed. In automatic tables, columns will simply stretch to fit the text instead of truncating it.
  • The cell or a child container inside the cell must have overflow: hidden; and white-space: nowrap;.
  • The column width must be defined (either explicitly or dynamically allocated by the table width).
