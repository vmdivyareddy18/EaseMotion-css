Bundle Size Tracking

Monitor your CSS bundle sizes and catch unexpected growth before it reaches production.

How it works

  • Installs bundlewatch as a dev dependency
  • Defines size limits for easemotion.css and easemotion.min.css
  • Runs on every PR and fails CI if any bundle exceeds its max size by more than 5KB
  • Comments the bundle size delta directly on the PR

Configuration

{
  "files": [
    {
      "path": "./easemotion.css",
      "maxSize": "170 kB"
    },
    {
      "path": "./easemotion.min.css",
      "maxSize": "30 kB"
    }
  ],
  "ci": {
    "trackBranches": ["main"],
    "defaultCompression": "gzip"
  }
}

The workflow fails if any tracked file exceeds its max size by more than 5KB compared to the main branch.

Built for EaseMotion CSS — keep the framework lightweight.
