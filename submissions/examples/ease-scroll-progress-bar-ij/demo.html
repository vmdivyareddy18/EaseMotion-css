Scroll Progress Bar

A fixed progress bar at the top of the viewport that fills as you scroll. Driven by CSS animation-timeline: scroll(root) with a JavaScript fallback for unsupported browsers.

How It Works

Modern browsers use the native CSS scroll-driven animation API. The bar progresses in sync with your scroll position, running entirely on the compositor thread for jank-free performance.

For browsers that don't support animation-timeline, a lightweight scroll event listener keeps the bar in sync using the same gradient fill.

What is Scroll-Driven Animation?

CSS scroll-driven animations (animation-timeline: scroll()) let you tie animation progress directly to a scroll container's scroll position. The scroll(root) timeline maps the document's scroll range — 0% at the top, 100% at the bottom — directly to the animation's progress.

This means progress bars, parallax effects, and reveal animations can be built with zero JavaScript and zero scroll event listeners.
