Automated Broken Link Checker

Improve documentation quality by automatically checking links, detecting broken URLs, and creating GitHub Issues using GitHub Actions.

Explore Workflow

Overview

This example demonstrates a CI workflow that automatically scans all documentation links on a weekly schedule. Whenever broken links are detected, the workflow generates a report and opens a GitHub Issue so maintainers can fix the documentation quickly.

Automation Workflow

Weekly Schedule Trigger
↓
Checkout Repository
↓
Scan Documentation Links
↓
Detect Broken URLs
↓
Create GitHub Issue Automatically

GitHub Actions Workflow 

name: Broken Link Checker

on:
  schedule:
    - cron: "0 0 * * 0"

jobs:
  check-links:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Check Links
        run: |
          npx lychee docs/index.html

      - name: Create GitHub Issue
        if: failure()
        run: echo "Broken links detected"
