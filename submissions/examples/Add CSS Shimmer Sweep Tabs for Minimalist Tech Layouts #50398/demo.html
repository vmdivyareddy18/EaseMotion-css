Platform architecture and runtime documentation.

GET /v1/users/{id}

Retrieves the details of an existing user account. You need only supply the unique user identifier.

curl https://api.nexus.com/v1/users/usr_123 \
-H "Authorization: Bearer sk_test_89x..."
POST /v1/users

Creates a new user account with the provided parameters.

curl -X POST https://api.nexus.com/v1/users \
-H "Authorization: Bearer sk_test_89x..." \
-d "email=john.doe@example.com"

Real-time Event Streaming

Listen for changes on your Nexus resources via HTTPS callbacks. Webhooks are configured per environment.

Event Type
Description
user.created
Occurs whenever a new user is created.
user.deleted
Occurs whenever a user is permanently removed.
invoice.paid
Occurs when a payment succeeds.

Error Handling

Nexus uses conventional HTTP response codes to indicate the success or failure of an API request.

400
Bad Request

The request was unacceptable, often due to missing a required parameter.

401
Unauthorized

No valid API key provided.

404
Not Found

The requested resource doesn't exist.
