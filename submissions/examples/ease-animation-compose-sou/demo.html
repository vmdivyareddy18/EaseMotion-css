ease-animation-compose

Demonstrates the common bug where applying multiple animation classes on one element only plays the last one — and shows three working fixes using longhand properties, combo classes, or CSS custom property composition.

🔄 Refresh the page to replay all animations.

❌ 1. The Bug — Shorthand Classes Overwrite Each Other

Each class below uses the animation shorthand. When you combine two, the second one wins and the first one is silently dropped. Open DevTools and inspect the computed animation property — you'll see only ONE animation listed.

single class .ease-fade-in-bad
Fades in only
single class .ease-slide-up-bad
Slides up only
BUG: two classes .ease-fade-in-bad .ease-slide-up-bad
Only slide-up plays!

Fade-in is overwritten by slide-up's shorthand.

✅ 2. Fix #1 — Use Longhand Properties

Define animation-name, animation-duration, and animation-timing-function separately. Now both classes contribute their names and BOTH animations play together.

single class .ease-fade-in
Fade in
single class .ease-slide-up
Slide up
two classes — both play! .ease-fade-in .ease-slide-up
Fade + Slide ✨
three classes — all play! .ease-fade-in .ease-slide-up .ease-rotate-in
Fade + Slide + Rotate 🎉

✅ 3. Fix #2 — Predefined Combo Classes

For common combinations, ship explicit combo classes that comma-separate the animations. Predictable, zero magic, easy to read.

.ease-fade-slide-up animation: fade, slide-up;
Fade + Slide combo
.ease-fade-scale-in animation: fade, scale-in;
Fade + Scale combo
.ease-fade-rotate-in animation: fade, rotate-in;
Fade + Rotate combo

✅ 4. Fix #3 — CSS Variable Composition

The most scalable pattern: a base .ease-anim class consumes a custom property --ease-anim-list, and each modifier class appends to that list. Combine as many as you want, in any order.

.ease-anim .add-fade-in
Fade only
+ .add-slide-up
Fade + Slide
+ .add-scale-in
Fade + Scale
all three
Fade + Slide + Rotate

📚 5. Summary — When to Use Which Fix

A quick decision guide:

  • ✅ Fix #1 (Longhand) — Best default for utility frameworks. Write classes once and they compose automatically.
  • ✅ Fix #2 (Combo classes) — Use for common, named presets like .ease-fade-slide-up. Predictable and explicit.
  • ✅ Fix #3 (CSS Variables) — Use for fully dynamic composition. Most powerful, requires every utility to opt-in to the pattern.
  • ❌ Avoid the shorthand (animation: name dur timing;) inside reusable utility classes — it silently overwrites.
