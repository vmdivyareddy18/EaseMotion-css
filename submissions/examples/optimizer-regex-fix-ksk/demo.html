🐛 Bug #40061 — Regex Spacing Fix

extractEaseClasses() and extractEmAttributes() miss valid HTML attributes when spaces surround the = sign. This demo shows the broken vs. fixed behaviour live.

Root Cause Both regex patterns in optimizer.js require = with no surrounding whitespace. HTML allows spaces around =, so <div class = "ease-fade-in"> is silently skipped — causing the build-time optimizer to incorrectly drop valid classes and em animations.

Minimal Fix — 2 lines in optimizer.js

// extractEmAttributes() — line 24
- const re = /\bem=(['"])([^"']+)\1/g;
+ const re = /\bem\s*=\s*(['"])([^"']+)\1/g;

// extractEaseClasses() — line 40
- const classRe = /class=(['"])([^"']+)\1/g;
+ const classRe = /class\s*=\s*(['"])([^"']+)\1/g;
