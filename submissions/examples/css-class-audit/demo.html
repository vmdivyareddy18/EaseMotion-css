CSS Class Name Audit

Detect duplicate CSS class names across submission folders — pure bash, no dependencies.

Why This Matters

With hundreds of CSS examples across multiple folders, the same class name can be defined in different places. When users copy multiple demos into a single project, these duplicates can cause unexpected style conflicts.

This audit script scans every style.css in submissions/, extracts all class selectors, and flags any name that appears in more than one folder.

Usage

bash submissions/examples/css-class-audit/audit-classes.sh

Run from the repo root. No dependencies required.

Sample Output

=== CSS Class Name Audit ===
Scanning style.css files in ./submissions/...

✅ No duplicate class names found across submission folders.

If duplicates exist, the script lists each name and the folders that define it.

How It Works

  1. Finds all style.css files under submissions/
  2. Extracts CSS class selectors (patterns starting with .)
  3. Groups by class name, flags any name appearing in multiple folders
  4. Exits with code 0 if clean, 1 if duplicates found
