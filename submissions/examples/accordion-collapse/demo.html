✦ Native <details> variant — zero JS

🌐 How does the native variant work?
Uses the browser's native <details> element — no JavaScript at all. The content slides in on open via a CSS keyframe. The chevron rotates using the [open] attribute selector. Great for static sites and progressive enhancement.
♿ Is it accessible without JS?
Yes — <details>/<summary> has built-in keyboard support and ARIA semantics provided by the browser. Screen readers announce the expanded/collapsed state automatically.
⚡ Any limitations?
The close animation cannot be triggered with pure CSS on <details> (the element removes open before the animation runs). For animated close on the native variant, a small JS snippet is needed. The JS-toggled variant above handles both open and close smoothly.
