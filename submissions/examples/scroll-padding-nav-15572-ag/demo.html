Scroll-Padding Anchor Navigation

Learn how the scroll-padding CSS property creates elegant, accessible offset scrolling for pages with fixed/sticky headers.

Scroll down or click links above to see it in action

1. The Overlay Problem

When you implement a sticky navigation header, clicking anchor links (like #section) tells the browser to scroll the top of that section to the top of the viewport.

The Bug: The fixed header overlays the first few lines of the target section, blocking headings, subtitles, and introductory paragraphs.

2. Scroll-Padding Solution

By applying scroll-padding-top to the scroll container (usually the html element), you define an inset scroll offset. The browser offsets the landing position by exactly the height of your navigation bar.

  • No complex JavaScript calculation required
  • Smooth, native scrolling preserved
  • Works with all keyboard navigation and screen-readers

3. Clean CSS Implementation

Implementing scroll offsets requires only two CSS declarations on your HTML element:

html {
  scroll-behavior: smooth;
  scroll-padding-top: 80px; /* Matching sticky header height */
}

4. Live Comparison

Try clicking the navigation links at the top. Notice how each section lands precisely 80px below the viewport top, leaving the section heading fully visible beneath the frosted-glass navbar.
