PR Sizing Reference Metrics

        

          

            
              

                Size Label
                Files Changed Range
                Estimated Review Time
                Priority Target
              

            
            
              

                size/XS
                1 - 2 files
                < 5 minutes
                Fast merges (typos, simple tweaks)
              

              

                size/S
                3 - 5 files
                10 - 15 minutes
                Medium reviews (minor refactors, docs pages)
              

              

                size/M
                6 - 10 files
                20 - 30 minutes
                Standard reviews (small features, core variables setup)
              

              

                size/L
                11 - 20 files
                40 - 60 minutes
                Detailed reviews (substantial features, multi-components)
              

              

                size/XL
                20+ files
                > 2 hours
                Complex merges (major refactors, break releases)
              

            
          

        

      


      
      
Detailed Devops Guidelines

      

        

          
1. Review Queue Management

          

            Sizing labels allow maintainers to quickly spot quick wins (XS/S) and schedule larger reviews (L/XL)
            when they have dedicated blocks of time.
          

        


        

          
2. GitHub Script API

          

            Rather than calling heavy custom Docker actions, leveraging actions/github-script executes
            Octokit scripts inside a secure Node context, running in seconds.
