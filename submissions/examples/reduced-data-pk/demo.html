prefers-reduced-data Support

Respect users' bandwidth preferences. Disable decorative animations and heavy visuals on slow or metered connections. Experimental

Full Data Mode

All animations, gradients, and effects active.

🎬 4 animations 🎨 gradients 📦 2.1 KB painted

Reduced Data Mode

Decorative animations and gradients removed.

🎬 0 animations 🎨 flat colors 📦 0.3 KB painted

When It Activates

  • 🧑‍💻 User enables Data Saver in browser (Chrome/Android)
  • 🌍 Slow connection detected by OS
  • 📱 Metered mobile network
  • ⚙️ OS-level reduced data setting

What Gets Disabled

  • ❌ Decorative @keyframes animations
  • ❌ shimmer, ripple, pulse effects
  • ❌ Heavy gradients and box-shadows
  • ❌ Non-essential background images
  • ✅ Content animations (fade-in, slide-up) kept

Proposed Implementation

Add to easemotion.css alongside the existing reduced-motion block:

easemotion.css
@media (prefers-reduced-motion: reduce),
       (prefers-reduced-data: reduce) {
  :root {
    --ease-speed-fast: 0.01ms;
    --ease-speed-medium: 0.01ms;
    --ease-speed-slow: 0.01ms;
    --ease-animation-iterations: 1;
  }
}

@media (prefers-reduced-data: reduce) {
  .ease-shimmer,
  .ease-ripple,
  .ease-pulse,
  .ease-gradient-bg {
    animation: none !important;
    background-image: none !important;
  }
}

Opt-In Override

For users who want animations despite their data preference:

.ease-no-preference-data {
  --ease-reduced-data-override: true;
}

@media (prefers-reduced-data: reduce) {
  .ease-no-preference-data .ease-shimmer {
    animation: ease-shimmer 2s ease infinite !important;
    background-image: linear-gradient(...) !important;
  }
}
