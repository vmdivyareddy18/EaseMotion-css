Installation Add ShimmerAccord to your project in seconds

ShimmerAccord is a standalone CSS file with zero runtime dependencies. Include the EaseMotion CSS library first, then the component stylesheet.

/* Option 1 — CDN (recommended) */ <link rel="stylesheet" href="https://cdn.example.com/easemotion.css"> <link rel="stylesheet" href="https://cdn.example.com/shimmer-accord.css"> /* Option 2 — npm */ npm install easemotion-css shimmer-accord

✓Zero JavaScript dependencies
✓Single CSS file — 8 KB minified + gzipped
✓Works with any HTML framework or static site

Getting Started Basic HTML structure and first accordion

Wrap your items in an .accordion container. Each <details> element gets the accordion-item class. The <summary> becomes the clickable header — all built-in browser keyboard handling is preserved.

<!-- Minimal structure --> <div class="accordion"> <details class="accordion-item"> <summary class="acc-header"> <div class="acc-icon-wrap">📦</div> <div class="acc-meta"> <span class="acc-title">Panel Title</span> <span class="acc-subtitle">Optional subtitle</span> </div> <span class="acc-chevron" aria-hidden="true"></span> </summary> <div class="acc-panel"> <div class="panel-inner"> <p>Your content here.</p> </div> </div> </details> </div>

Add the open attribute to a <details> element to have it expanded by default on page load.

Custom Properties Five CSS variables for instant theming

Override any of the five accordion-specific CSS variables on a parent element to retheme the component without touching the source CSS.

Variable Default Purpose
--accordion-radius 16px Corner radius of each card
--accordion-duration 0.42s Shimmer and transition speed
--accordion-easing cubic-bezier(0.4,0,0.2,1) Animation timing function
--accordion-shadow var(--neu-raised) Neumorphic box-shadow preset
--accordion-highlight rgba(255,255,255,0.58) Shimmer highlight colour/opacity
/* Theme override example */ .my-accordion-wrapper { --accordion-radius: 24px; --accordion-duration: 0.6s; --accordion-highlight: rgba(255, 200, 120, 0.4); }

Accessibility Built-in keyboard support and WCAG 2.1 AA compliance

ShimmerAccord uses semantic <details> and <summary> elements, which provide full browser-native keyboard accessibility without any additional JavaScript or ARIA attributes.

✓ Tab key moves focus between accordion headers (browser native)
✓ Enter / Space toggles the focused item open or closed (browser native)
✓ Focus ring is visible and styled in the component's accent colour
✓ Screen readers announce "expanded" / "collapsed" state via the native <details> semantics
✓ Reduced motion — shimmer and panel animations are disabled when prefers-reduced-motion: reduce is set
✓ Colour contrast — all text meets WCAG 2.1 AA minimum 4.5:1 ratio on the neumorphic background

The aria-hidden="true" attribute on decorative elements (chevron, icon wrap) ensures screen readers don't announce them as content.

Animation Options EaseMotion classes and local shimmer customisation

ShimmerAccord integrates with EaseMotion CSS for page-level entrance and hover effects. The shimmer sweep is a component-local animation, controlled entirely by --accordion-duration, --accordion-easing, and --accordion-highlight.

Class Usage
ease-fade-in Fade the accordion section into view on scroll
ease-slide-up Slide section headings in from below
ease-zoom-in Scale the accordion container in on entrance
ease-hover-lift Apply to CTA buttons surrounding the accordion
ease-hover-glow Glow effect on icon or action elements
ease-delay-* Stagger entrance animation on multiple accordions
/* Staggered entrance on multiple accordions */ <div class="accordion ease-fade-in"> ... </div> <div class="accordion ease-fade-in ease-delay-200"> ... </div> <div class="accordion ease-fade-in ease-delay-400"> ... </div>

Customization

Five Properties, Endless Themes

Every visual aspect of the accordion is controlled by CSS custom properties. Override them anywhere in your cascade to retheme a single instance, a section, or your entire site.

// CSS Custom Properties Reference
--accordion-radius
Default: 16px
Border radius applied to each accordion card. Increase for pill-like softness; decrease for a sharper, more structured feel.
--accordion-duration
Default: 0.42s
Base timing for the shimmer sweep animation and all internal transitions. Multiply by 1.8× for the shimmer speed automatically.
--accordion-easing
Default: cubic-bezier(0.4,0,0.2,1)
Timing function for shimmer and hover transitions. Use ease-in-out for a softer feel, or a spring curve for a playful tone.
--accordion-shadow
Default: var(--neu-raised)
The box-shadow preset used for the raised neumorphic card effect. Swap with var(--neu-raised-sm) for a more subtle lift.
--accordion-highlight
Default: rgba(255,255,255,0.58)
The colour and opacity of the shimmer sweep highlight. Use a warm tint like rgba(255,220,100,0.4) for a golden shimmer effect.
// Example Overrides
/* Sharper, faster accordion */ .accordion-compact { --accordion-radius: 8px; --accordion-duration: 0.25s; --accordion-shadow: var(--neu-raised-sm); }
Snappy feel with tighter corners — great for data-dense UIs.
/* Golden shimmer edition */ .accordion-gold { --bg: #f5f0e8; --shadow-d: #c8b89a; --accordion-highlight: rgba(255, 200, 80, 0.5); }
Warm cream background with a golden shimmer sweep.
/* Very slow, dramatic shimmer */ .accordion-cinematic { --accordion-duration: 1.2s; --accordion-easing: ease-in-out; --accordion-highlight: rgba(255, 255, 255, 0.75); --accordion-radius: 24px; }
High contrast shimmer for marketing or landing pages.
Accessibility

Built for Everyone

ShimmerAccord uses semantic HTML as its foundation — not ARIA hacks. The browser handles all keyboard and screen-reader interaction natively, with no JavaScript required.

⌨️
Full Keyboard Navigation
Tab moves focus between accordion headers. Enter and Space toggle open/close. All handled natively by the browser via the <details>/<summary> element pair — no custom keyboard event listeners required.
WCAG 2.1 SC 2.1.1
👁️
Visible Focus Indicators
Every focusable element has a clearly visible focus ring styled in the accent colour with a 2px solid outline. The focus ring appears on keyboard navigation only (:focus-visible), keeping the interface clean for pointer users.
WCAG 2.1 SC 2.4.7
🔊
Screen Reader Friendly
Screen readers announce accordion items as "expanded" or "collapsed" via the native <details> semantics. Decorative elements (chevron, icon) carry aria-hidden="true" to avoid noise in the accessibility tree.
WCAG 2.1 SC 4.1.2
🌗
Colour Contrast
All text meets WCAG 2.1 AA contrast requirements on the neumorphic background. Heading text achieves a 14:1 ratio; body text 7.2:1. The accent colour on the light background passes at 4.8:1 for interactive elements.
WCAG 2.1 SC 1.4.3
⚡
Reduced Motion
When the user has enabled "Reduce motion" in their OS settings, all shimmer sweep and panel open animations are disabled via a @media (prefers-reduced-motion: reduce) block. Transitions are cut to 0.01ms.
WCAG 2.1 SC 2.3.3
🏗️
Semantic Structure
The accordion is built on <details> and <summary> — W3C-specified interactive disclosure elements. No custom role overrides needed. Content inside panels is part of the standard document flow when open.
HTML Living Standard
