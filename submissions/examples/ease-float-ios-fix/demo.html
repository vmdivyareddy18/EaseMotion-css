Ease-Float iOS Safari Fix

About This Fix

The original .ease-float animation previously caused page jumping on iOS Safari due to mid-animation layer promotion. This fix addresses it by:

  • translateZ(0): Pre-promotes the element to a GPU compositing layer upfront, preventing mid-animation layer shifts
  • backface-visibility: hidden: Prevents flicker on older Safari versions during layer changes
  • Vendor-prefixed keyframes: Ensures the compositor thread handles every frame
  • will-change: transform: Hints the browser to keep this element composited for its lifetime

Note: -webkit-overflow-scrolling: touch only applies to actual scroll containers, not animated elements.
