✅ With calc-size()

Smooth, content-aware height animation using calc-size(max-content, size) and interpolate-size: allow-keywords.

This content uses calc-size() for its height animation. The height transitions smoothly between 0 and max-content regardless of how much content is inside.

Adding more content won't break the animation — the panel adapts naturally because the browser can interpolate between the two sizing values.

Another panel demonstrating that each accordion item works independently. The calc-size() approach handles multiple panels gracefully with no JavaScript height calculations.

❌ Without calc-size()

Uses fixed max-height for animation — content overflows or gets clipped when exceeding the limit.

This panel uses a fixed max-height: 150px for its animation. The height transitions smoothly only if the content fits within 150px.

If the content grows larger than the max-height, it will overflow or get clipped — visible as a jarring cut-off.

With longer content like this paragraph and the ones below, the fixed max-height approach starts to show its limitations.

More text means more overflow. The animation might look okay initially but breaks down with real-world variable content.

Developers often resort to JavaScript measurement or absurdly large max-height values that ruin the animation timing.

This is exactly the problem calc-size() was designed to solve.
