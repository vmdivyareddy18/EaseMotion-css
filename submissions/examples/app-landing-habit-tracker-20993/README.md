# App Landing: Habit Tracker App (`app-landing-habit-tracker-20993`)

This proposal provides a complete, copy-paste ready B2C landing page designed for a mobile habit tracking application, built entirely with HTML/CSS and standard EaseMotion utility classes.

## 📌 Feature Overview

A habit tracking app needs to project focus, motivation, and achievement. This template achieves a premium, "dark-mode first" aesthetic using deep backgrounds (`#09090B`) paired with high-energy neon accents (cyan `#000F0FF` and pink `#FF2E93`). The typography uses `Outfit` for a modern, geometric tech feel.

The layout includes:
- A Hero section featuring a CSS mobile app mockup demonstrating a live habit checklist, date scroller, and streak notifications, surrounded by ambient CSS glowing orbs (`filter: blur()`).
- A 3-column features grid highlighting the methodology with glow hover effects.
- A Count-Up Stats section.
- A "How it Works" split timeline layout paired with a CSS-rendered "GitHub-style" habit heatmap contribution graph.
- A Testimonials grid.
- Bold App Store download Call-To-Action (CTA) section with a subtle gradient border.

### Showcased Classes
The template leverages the following EaseMotion animation and interaction utilities, specifically configured with a deep, smooth `cubic-bezier(0.16, 1, 0.3, 1)` to mimic premium iOS-style application physics:
- `.ease-fade-up` / `.ease-slide-up` (Used for text and card entrances, giving a focused, intentional feel)
- `.ease-zoom-in` (Applied to ambient background glows and stats boxes, combined with an initial blur filter for a fade-in effect)
- `.ease-hover-lift` (Standard interaction for habit cards and download buttons)
- `.ease-hover-glow` (Applied to feature cards and logos to trigger a drop-shadow glow effect on hover)

## ⚙️ How to Use

To view this full-page template locally, simply open the `demo.html` file in your web browser. 

The CSS handles the layout, the dark theme, the phone mockup geometry, the SVG habit rings, and the heatmap grid simulation. The standard `ease-*` interaction classes are attached directly to HTML elements to provide the premium entrance animations. The design is fully responsive and requires zero JavaScript to execute the entrance and hover effects.
