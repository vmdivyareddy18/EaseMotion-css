Loading Spinner Centering Bug

.ease-btn-loading::after uses translate: -50% -50% to center the spinner, but its ease-kf-btn-spin keyframes also set transform: translate(-50%, -50%) rotate(...). Since the standalone translate property and transform are additive, the two -50% offsets stack into -100% -100%, pushing the spinner completely off-center.

Bug reproduced translate + transform both set → offsets add up
Fixed transform alone handles centering + rotation
