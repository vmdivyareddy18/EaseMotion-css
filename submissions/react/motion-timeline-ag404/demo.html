Issue #38757

MotionTimeline

A declarative sequence for heterogeneous EaseMotion steps. Each step can use its own animation type, duration, and gap while the timeline computes cumulative delays from the same semantic rules as the React components.

Normal sequencing is active. The DOM order stays the same.

Equivalent React API

<MotionTimeline delay={200} reverse={false} tag="section" className="hero-sequence">
  <MotionStep type="fade-in" duration={300} gap={100}>
    <HeroTitle />
  </MotionStep>
  <MotionStep type="slide-up" duration={400} gap={150}>
    <HeroDescription />
  </MotionStep>
  <MotionStep type="zoom-in" duration={300}>
    <HeroCTA />
  </MotionStep>
</MotionTimeline>

3-stage heterogeneous timeline

Normal order: title, description, then CTA.

Cumulative delay updates in real time
01

Hero title

type: fade-in

The title appears first, then the timeline advances by its resolved duration plus its gap.

duration: 300ms gap: 100ms delay: 200ms
02

Hero description

type: slide-up

Content uses a different animation, a different duration, and a different gap, but still participates in the same timeline math.

duration: 400ms gap: 150ms delay: 600ms
03

Hero CTA

type: zoom-in

The final step closes the sequence. In reverse mode, it becomes the first step to start, but the DOM order never changes.
